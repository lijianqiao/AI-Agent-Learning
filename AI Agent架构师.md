# AI Agent 架构师学习路线

---

## 前置要求

开始本路线前，需具备以下基础能力：

- **Python 编程**：熟悉异步编程（asyncio）、装饰器、类型注解、虚拟环境管理
- **Docker 基础**：能运行容器、编写简单 Dockerfile、理解网络与卷挂载
- **REST API 理解**：了解 HTTP 请求/响应、JSON 格式、状态码语义
- **基础数学**：线性代数向量运算、概率论基本概念（理解 softmax、余弦相似度等）

## 阶段 1：LLM 原理与调度基石（第 1–4 周）

### 🎯 核心能力

理解大模型本质能力边界，掌握多模型并行的网关架构分配。

### 第 1 周：Transformer & Token 机制

**学习目标**

- Attention 机制与 Tokenization 原理
- Context Window 本质、长文本退化原因与注意力分散效应

**实战**

- 编写：本地简易 Token 统计器、Context 贪婪截断策略模拟器

**验收标准**

- 能清晰解释：KV-Cache 大致原理、为何长上下文性能下降，并出具 Token 预算规划表。

### 第 2 周：推理模式与 Prompt 结构深度演进

**学习目标**

- ReAct 闭环控制、Chain-of-Thought (思维链)
- 业界 Tool Calling (Function Calling) 底层约定参数交互规范与生命周期

**实战**

- 实现：无外部框架加持的 Vanilla Agent 原生循环调用，标准化记录每一次 Reasoning Trace 日志。

**验收标准**

- 完全理解 LangChain 内部封装黑箱机制中对 Prompt 的组装策略。

### 第 3 周：模型网关建设与动态路由 (Routing) 🚀*[进阶强化]*

**学习目标**

- 商业服务 API (GPT-4o/Claude 3.5) vs 端侧开源模型成本 trade-off
- 动态路由架构与统一熔断限流机制
- 本地推理加速方案认知：**vLLM**、**TGI**、**Ollama** 等推理服务框架对比（架构师须知边界，了解适用场景即可）

**实战**

- 集成：**LiteLLM (或任意 LLM 网关框架)** 统一对接多厂商模型接口，建立 API Key 轮询容灾。
- 研发：意图识别探测器，实现**语义路由 (Semantic Routing)**（简单任务跑 Qwen/Llama3 廉价模型，推理繁重跑高配）。

**验收标准**

- 在同等并发效果下，证明通过软路由调度策略使系统总体推理费用压低至少 30%。

### 第 4 周：幻觉发生剖析与防御抑制策略

**学习目标**

- 大模型幻觉产生的数学模糊概率机制、Grounding 本地知识溯源手段

**实战**

- 在纯原生调用基础上，注入验证模块，实现强验证型无中生有过滤系统。

**验收标准**

- 问答系统输出不仅带来源，更必须透传针对此结果的 Confidence Score 系统置信分。

## 阶段 2：生产级 RAG 架构（第 5–8 周）

### 🎯 核心能力

搭建企业级精准、极速检索引擎，抛弃主观黑盒并引入可度量的研发闭环。

### 第 5 周：异构文档洗线、Chunk 策略打样与框架拓宽 🚀*[进阶强化]*

**学习目标**

- Fixed chunking vs Semantic chunking 算法选型、高频元数据注入 (Metadata tagging)

**实战**

- 构建多文档可调整摄入器 (Ingestion Pipeline)
- 框架拓宽：对比 **LangChain** 与专注检索增强的 **LlamaIndex** 在复杂结构文档应对上的异同。

**验收标准**

- 实现一个能应对长篇 PDF 与多级表头 Excel 解析方案且支持 A/B 切换的效果对比实验台。

### 第 6 周：向量数据库纵横与 GraphRAG 认知 🚀*[进阶强化]*

**学习目标**

- 深入掌握 Milvus 或 Weaviate (含 HNSW 索引理念与余弦等距离算法)
- 前瞻探索：了解微软 **GraphRAG (知识图谱与原生向量协同结合技术)**。

**实战**

- 手动调整索引 Top-K 值与建库维度；对图谱的 Node/Edge 建立一次简单关系映射抽取。

**验收标准**

- 基于定制 30 个干扰性搜索问题的验证集中验证纯向量库提取相关召回段落概率 > 85%。

### 第 7 周：Hybrid Search 双轨检索与重排序核心网络

**学习目标**

- BM25 稀疏关键字硬匹配算法与 Dense 向量软匹配融合权重计算
- 结合 Cross-Encoder 互注意力重排序大模型重新校准结果得分

**实战**

- 实现向量软召回库 + 倒排索引库，并在顶部增加 Reranker 重校权重引擎。

**验收标准**

- 针对前一周的刁钻查询干扰测试集，使相关段落命中率跃升 10% 以上。

### 第 8 周：生产级 RAG 系统全貌与 Agentic RAG 进阶 🚀*[进阶强化]*

**学习目标**

- **Agentic RAG** 模式：让 Agent 自主决策何时检索、检索几次、是否需要改写查询，而非单次固定流水线调用，这是 RAG 架构的重要演进方向。

**实战项目**

- 把此前松散系统耦合为一个成型的《高度可调参溯源问答服务集群》，并引入自适应检索决策节点。

**验收标准**

- 完成并发压测探底，在稳定并发下 P95 延迟 < 3s；实现每条产出完全携带 Reference Id 追溯；Agentic RAG 模式在多跳问题上准确率优于单次检索 ≥ 15%。

## 🛑 缓冲期 1（第 9 周）：架构复盘与指标评测体系搭建 🚀*[进阶新增]*

### 第 9 周：从"炼丹"到度量工程化 (Evaluation Metrics)

**核心思想**：依靠"体感"调优必定会撞南墙，必须基于评测分推进敏捷开发。

**学习目标**

- Agent CI/CD Baseline 黄金验证测试集构建方式。

**实战与验收**

- 引入 **Ragas** 或 **TruLens** 第三方严苛打分器；集成算例对上周 RAG 系统全面打分。要求后续任意微调（Chunk变小/模型更换）都可执行自动跑批打分并直接观测到三大指标（上下文相关度、准确性、有助作答率）的起伏漂移。

## 阶段 3：Agent 状态机演化与工作流编排（第 10–13 周）

### 🎯 核心能力

解构长尾复杂决策链，以编程式流程网络降服模型发散式胡言乱语。

### 第 10 周：基于控制流的网络状态机编管 (LangGraph) 与 MCP 协议 🚀*[进阶强化]*

**学习目标**

- 自主图计算拓扑流转模型、节点定义与边条件转换控制理论 (Conditional Edges)。
- **MCP 协议 (Model Context Protocol)**：Anthropic 主导的工具/资源集成标准，已被 LangGraph、Claude、Cursor 等主流 Agent 框架广泛采纳；掌握其 Server/Client 交互规范、Resources/Tools/Prompts 三类原语定义。

**实战**

- 应用 LangGraph 或类似前沿网格化引擎绘制并替换早期的硬编码暴力 Agent 行为循环。
- 基于 MCP 协议封装一个自定义工具服务（如封装本地文件检索能力），并接入 Agent 工具调用链路。

**验收标准**

- 对图状有向无环或条件有步骤流转机制不迷航，杜绝长逻辑状态机发版导致的死锁溢出；能独立实现并部署一个可用的 MCP Server。

### 第 11 周：工作流分治法、并行规划流与 DSPy 参数化优化入门 🚀*[进阶强化]*

**学习目标**

- Planner 与 Executor 定向分离。
- **DSPy** 极客框架核心理念（Prompt 从"魔法提示词调优"转换为类似神经网络参数化优化概念）；掌握 `dspy.Signature`、`dspy.ChainOfThought` 等基础 Module 用法。

> ⚠️ 注意：DSPy 学习曲线较陡，本周重点掌握核心理念与基础 Module 使用；Teleprompter 编译器优化实践放在第 12 周延续。

**实战**

- 先拆包："阅读-总结-对比-草拟"复杂工作解耦分四步。
- 使用 DSPy 的基础 Module 完成至少一个子任务的模块化改造，替换原手写 Prompt。

**验收标准**

- 能用 DSPy 替换至少一个原本手写 Prompt 的子任务模块，并保持相同或更优的输出质量。

### 第 12 周：DSPy 编译优化实践与中断干预、异常边界自稳态建设

**学习目标**

- DSPy Teleprompter 编译器：让少数人工评估案例引导自我打磨核心子任务 Prompt。
- API Error Code、网络闪断，及灾难错误处理预案防雪崩体系。

**实战**

- 引入基础 DSPy Teleprompter 编译器，基于小型标注集自动优化上周的 DSPy 模块。
- 为所有组件套上 Retry (指数退避重试)、超时降级兜底预设选项、以及中途拦截请求权限人工审核审批 (Human-in-the-loop)。

**验收标准**

- DSPy 编译后的模块在评测集上比手工 Prompt 版本准确率提升可量化；异常注入测试下系统不崩溃。

### 第 13 周：上帝视角可视大盘监控体系搭建

**学习目标**

- 追踪链路全日志、分布式事务链路打点系统收集。

**实战**

- 部署并在所有请求挂载郎式神眼级框架 (LangSmith / Phoenix 等)，截留与可视化模型每次 Tool 调用进出参、推理损耗与卡壳阶段点。

**验收标准**

- 可根据唯一 RequestId 在系统库重现完整的一百步复杂流转交互轨迹树图。

## 阶段 4：Memory 系统重装抽象基建（第 14–17 周）

### 🎯 核心能力

跨越多轮对话孤岛与文风健忘边界，构建"比用户更懂用户"的长周期沉浸级记忆中枢。

### 第 14 周：即时操作缓存的 Token 预算管理

**学习目标**

- Token 上限滑动窗口淘汰制、重要短频信息提权算法。

**实战**

- 实现超过上下文 80% 面临 Token OOM 前自动平滑释放无关紧要上下文垃圾不报错的功能。

**验收标准**

- 在模拟长对话压测中，Token 使用率始终保持在阈值内，且关键信息（如用户身份、核心诉求）不丢失。

### 第 15 周：常驻向量存根归档器

**学习目标**

- 大模型背景时间线知识注入、画像库建立。

**实战**

- 在每次闲暇低负载时刻启动定时 Job 轮询系统，把零散小片段组装建立长期行为属性存储库（爱好、公司名等）。

**验收标准**

- 用户画像属性能在跨会话后被准确召回，影响模型回复风格与内容个性化程度。

### 第 16 周：无损级深层语义递归浓缩

**学习目标**

- 极度精炼的信息压缩，提炼图谱层关联实体与属性。

**实战**

- 实现 Recursive Summarization 的迭代摘要法，有效把 5 万字小说精华提纯定格为 2,000 字无缺失矩阵。

**验收标准**

- 压缩后的摘要经人工盲评，关键事件/实体保留率 > 90%，压缩比 > 20:1。

### 第 17 周：金字塔三层全混合记忆抽象体架构

**学习目标**

- 定义高度封装的面向接口调用的顶级聚合器设计模式。

**实战**

- 实现统一记忆聚合器（一调即走：从短期闪存、中期摘要缓存与深层次匹配中快速合成拼盘返回给推理大模型）。

**验收标准**

- 实现 50 轮前后文强跳跃对话压测下，不丢失第二轮及第六轮交代的"我是做什么的""上次报了什么指标"等冷知识能力，做到连续一贯表现。

## 🛑 缓冲期 2（第 18 周）：系统服务彻底微服务解耦化重构 🚀*[进阶新增]*

### 第 18 周：解绑胶水代码，迈向现代高内聚低耦合

**核心思想**：庞然大物即将接通智能集群协作，单体烂框架寸步难行需要打散。

**学习目标**

- API 规范拆解设计、控制反转 (IoC) 与外挂插件化接口独立设计模式运用。

**实战与验收**

- 将所有关于搜索（底层RAG）、查询数据库状态及 Memory 操作均强隔离独立包块。最终使调度控制器的圈复杂度清透洁净，完全屏蔽具体存储技术栈组件，方便单元极测及多人小组协助开垦。

## 阶段 5：多智能体协作系统网络 (Multi-Agent System)（第 19–22 周）

### 🎯 核心能力

让大模型拥有分省分治策略组排编军的能力。

### 第 19 周：群体角色建制网络编排

**学习目标**

- 通过 CrewAI / MetaGPT 或 LangGraph 高阶多角色网络分封等前沿方案拓建分布式团队理念。

**实战**

- 对基础通用型业务建立（产品/研发/测试/运维）四大明确且系统内唯一的权限画像提示词预制舱。

**验收标准**

- 四个角色 Agent 能在协作流水线中各司其职，产出物可追溯到具体角色节点。

### 第 20 周：异步通信大网络、A2A 协议与冲突斡旋共识机制 🚀*[进阶强化]*

**学习目标**

- Publish-Subscribe 事件驱动机制运用、Agent 黑板协查模式、全局共享变量空间设定。
- **A2A 协议 (Agent-to-Agent Protocol)**：Google 2025 年推出的跨 Agent 通信标准，与 MCP 共同构成智能体通信的双轮标准；了解其 Task/Artifact 模型与跨平台互操作场景，掌握与 MCP 的职责边界区分。

**实战**

- 加入由大群节点向核心决策枢纽回报进展、当研发节点和测试节点相互不妥协发生冲突时启动法官裁定模型强力介入表决拍板的机制。

**验收标准**

- 系统在多 Agent 并发写入共享状态时无竞态冲突；冲突裁定节点能在 3 次以内达成最终决策。

### 第 21 周：MAS 群系统全自动化闭环联动

**综合演练**

- 给该微拟真群体输入："在网上搜最近大模型资讯并写五百字博客且审查发布"的高跨度口语指令。观察它从调研、执笔撰写、事实核对到排印出图最终输出的全流水线自动推进度。

**验收标准**

- 全流程无人工干预完成率 > 80%；每个 Agent 的输入/输出均有完整日志追溯。

### 第 22 周：核心壁垒——跨域代码安全沙盒 🚀*[进阶强化]*

**学习目标**

- 容器级动态销毁隔离防线、防止权限越权及敏感系统数据逃逸击穿。

**实战**

- 针对 Agent 自行编写工具测试代码、运行环境脚本高危步骤，嵌入并重构进沙盒云服务 (如 **E2B 沙箱 Sandbox**) 或高度限制配置宿主通信网络的受限 Docker。

**验收标准**

- 即便任一 Agent 在被"误导"运行了死循环宕机或试图拉库植入木马命令时系统皆岿然不动即时报警切断遏制。

---

## 阶段 6：终极工程合规化升级与极客出师战（第 23–26 周）

### 🎯 核心能力

将玩具原型的脆弱与不可控制性升华为能接稳生产高能洪流的企业舰队巨擘。

### 第 23 周：主动防御网、输出过滤阻断与护栏设施 🚀*[进阶强化]*

**学习目标**

- 系统风控合规建设：防御应对 Prompt Injection 提示词提权劫持、PBI 与数据窃取攻击诱导。

**实战**

- 在调度总入口及总出口架设护城墙（引入诸如 **NVIDIA NeMo Guardrails**）。任何关于非业务或企图突破红线套话及篡改系统角色的不合规对话、违禁字段都在请求触发 LLM 前遭前置正则表达式与微小模型直接扑杀。

**验收标准**

- 引入开源提示词攻击恶意大集全做回归测试，阻挡成功抵御超过 95% 防爆底线。

### 第 24 周：架构高载并流策略及缓冲队列整合

**学习目标**

- 当在线推理 QPS 激增，如何保护核心网不崩塌？

**实战**

- 将前台轮询拉取模型改作依靠消息中间件通道 (Kafka/RabbitMQ) 排队执行机制，释放前端锁区，后台常驻线程池异步消峰处理回吐。

**验收标准**
