## 第一部分学习：什么是大模型幻觉？

### 从一个真实案例说起

2023 年，美国律师 Steven Schwartz 用 ChatGPT 辅助写了一份法律简报，引用了六个案件先例。法官发现这六个案件根本不存在——全是 ChatGPT 编造的，完整的案件名称、法院名称、判决日期，没有一个是真的。律师被法院罚款，成为 AI 幻觉最著名的反面教材之一。

**大模型幻觉（Hallucination）** 指的是：模型生成了听起来流畅且有说服力、但实际上与事实不符的内容，包括编造的数据、虚构的引用、错误的逻辑推断。

### 幻觉的分类

| 类型 | 描述 | 例子 |
| --- | --- | --- |
| **事实幻觉** | 编造不存在的事实 | 虚构论文、引用不存在的统计数据 |
| **推理幻觉** | 逻辑链中间跳跃出错 | "A比B快，B比C快，所以A比C慢" |
| **数学幻觉** | 数字计算错误 | 大数字加减法出错 |
| **时间幻觉** | 混淆训练截止日期之后的事件 | 声称某产品"去年刚发布"（但已是3年前） |
| **归因幻觉** | 把观点错误归给某人 | "爱因斯坦说过：……"（他从未说过） |

## 第二部分学习：幻觉的数学概率本质

### 大模型的本质：下一个 Token 的概率分布

大模型本质上是一台**条件概率计算机**：给定前文，预测下一个 Token 的概率分布，选取（或采样）概率最高的那个。

```
P(下一个token | 已有的所有token)
```

每生成一个词，模型都在从一个巨大的词表（约 5 万个 Token）中"投票选举"下一个最可能的词。

### 为什么会生成"假信息"？

**关键矛盾**：模型优化的目标是"生成流畅连贯的文本"，而不是"生成真实的文本"。

举个例子，当你问模型"量子力学领域谁获得了2023年诺贝尔物理学奖？"：

- 模型在训练数据中见过大量"XXX 获得了 20XX 年诺贝尔奖"格式的句子
- 它知道"量子力学"和"诺贝尔物理学奖"经常出现在一起
- 它会生成一个听起来**格式完全正确、名字完全合理**的答案
- 但这个名字可能是从所有量子物理学家名字中"统计拼凑"出来的，并非真实获奖者

**本质：模型在做"格式模仿"，不是在查数据库。**

### 温度参数（Temperature）与幻觉的关系

Temperature 控制采样的"随机性"：

```
Temperature = 0：每次选概率最高的 Token（确定性输出，但容易重复）
Temperature = 1：按概率正常采样（标准设置）
Temperature = 2：放大低概率 Token 的机会（创意更强，幻觉也更多）
```

直观理解：Temperature 就像"投票前喝了多少酒"。

- 完全清醒（0）：每次都投给最有可能的候选人，保守稳定
- 微醺（0.7）：大体理性，偶尔冒险，正常工作
- 大醉（2.0）：随机乱投，创意十足，也可能胡说

**结论：生产环境建议 Temperature 设在 0 ~ 0.3，降低幻觉风险。**

### Top-p 采样与幻觉

Top-p（Nucleus Sampling）：只从累计概率达到 p 的最高概率 Token 集合中采样。

```
top_p = 0.1：只考虑最可能的几个词（保守）
top_p = 0.9：考虑更多候选词（多样）
top_p = 1.0：考虑全部词（最随机）
```

**降低幻觉的经验组合：temperature=0.1, top_p=0.1**（用于需要高准确率的问答场景）

## 第三部分学习：Grounding 本地知识溯源技术

### 什么是 Grounding？

**Grounding（接地气/溯源）** 是指：**强制要求模型的每一个输出都必须能在提供的参考文档中找到来源，禁止模型"自由发挥"。**

把模型从"天才演员（根据剧本自行发挥）"变成"忠实翻译（严格翻译原文）"。

### Grounding 的核心技术：RAG 召回注入

最有效的 Grounding 方法是 **RAG（检索增强生成）**：

```
① 用户提问："我们公司的退货政策是什么？"
        ↓
② 系统从知识库中检索最相关的文档段落
        ↓
③ 将检索到的段落作为"参考资料"注入 Prompt：
   "基于以下文档回答问题，不得超出文档范围：
   [文档内容：退货须在30天内，需保留原始包装...]"
        ↓
④ 模型只能基于给定文档回答，无法编造
        ↓
⑤ 输出答案并附上来源文档 ID
```

### Grounding Prompt 工程模式

```python
GROUNDED_SYSTEM_PROMPT = """
你是一个严格的问答助手。请遵守以下规则：

1. 【只能使用参考文档】：你的回答必须完全基于下方提供的参考文档，
   绝对不能使用参考文档之外的任何知识。

2. 【必须标注来源】：每个关键事实后面必须用 [来源: 文档ID] 格式标注出处。

3. 【不知道就说不知道】：如果参考文档中没有足够信息回答问题，
   请直接回复："参考文档中未找到相关信息，建议咨询专业人员。"
   绝对不能猜测或推断。

4. 【置信度评分】：在回答末尾给出 0-100 的置信度评分，
   评分标准：
   - 90-100：参考文档中有直接明确的原文依据
   - 60-89：参考文档中有间接相关信息，需要一定推断
   - 0-59：信息不充分，回答不确定性较高
"""
```

### 来源追踪的三种层次

| 层次 | 做法 | 精度 |
| --- | --- | --- |
| **文档级** | 标注答案来自哪份文档 | 低 |
| **段落级** | 标注来自文档的哪个段落 | 中 |
| **句子级** | 精确标注支撑答案的原文句子 | 高 |

生产环境推荐至少做到**段落级**溯源，高合规要求场景（医疗、法律）做到**句子级**。

## 第四部分学习：防御体系架构

### 幻觉防御不是一道门，而是一道墙

单靠一种方法无法完全消除幻觉，需要构建**多层防御体系**：

```
用户提问
    ↓
【第1层：输入层防御】
  - 问题澄清：模糊问题先追问，避免模型"自由发挥"
  - 意图检测：识别高幻觉风险问题类型（涉及具体数字、时间、人名）
    ↓
【第2层：检索层防御】
  - RAG 召回：强制用文档接地
  - 相关性过滤：召回相关度 < 阈值时拒绝回答，而非让模型瞎猜
    ↓
【第3层：生成层防御】
  - 低温度参数（0.1-0.2）
  - Grounding Prompt（禁止超出文档范围）
  - CoT 推理（让模型展示推理过程，减少跳跃）
    ↓
【第4层：输出层验证】
  - 来源一致性验证（答案是否真的来自召回的文档）
  - 事实核查模型（用第二个模型验证第一个模型的输出）
  - Confidence Score 计算
    ↓
输出给用户（附来源 + 置信分）
```

### 输出层验证：自我一致性（Self-Consistency）

用同一个问题，采样多次（如5次），看答案是否一致：

- **高度一致**：模型对这个问题很"确定"，幻觉风险低
- **答案分散**：模型在"猜"，幻觉风险高，应降低置信度或拒绝输出

```python
# 采样5次，计算答案一致性
answers = [llm.call(question, temperature=0.7) for _ in range(5)]
# 统计最高频答案和频率
from collections import Counter
freq = Counter(answers)
most_common, count = freq.most_common(1)[0]
consistency_score = count / len(answers)  # 0.0 ~ 1.0
```

## 实战：带 Confidence Score 的强验证问答系统

```python
import json
from openai import OpenAI
from dataclasses import dataclass
from typing import Optional

client = OpenAI()

# ===== 数据结构 =====
@dataclass
class GroundedAnswer:
    """带来源和置信分的问答结果"""
    question: str
    answer: str
    sources: list[dict]        # [{"doc_id": "...", "excerpt": "..."}]
    confidence_score: int      # 0-100
    confidence_reason: str     # 置信度说明
    is_reliable: bool          # 是否可信（置信分 >= 60 视为可信）
    warning: Optional[str]     # 低置信度时的警告信息


# ===== 模拟知识库（真实场景替换为向量数据库检索）=====
KNOWLEDGE_BASE = [
    {
        "doc_id": "policy-001",
        "title": "退换货政策",
        "content": "商品支持购买后30天内无理由退货，退货时商品须保持原包装完好，附带购买凭证。退款将在收到退货后3-5个工作日内原路返还。"
    },
    {
        "doc_id": "policy-002",
        "title": "会员权益说明",
        "content": "黄金会员每月享有10次免费快递权益，白银会员每月享有3次。会员积分满1000分可兑换50元优惠券，积分有效期为自获得之日起12个月。"
    },
    {
        "doc_id": "tech-003",
        "title": "产品技术规格",
        "content": "Model X 处理器为骁龙8 Gen3，RAM 12GB，存储256GB，电池容量5000mAh，支持67W快充，充满电约需45分钟。"
    },
]

def retrieve_docs(query: str, top_k: int = 2) -> list[dict]:
    """
    简化版检索（真实场景使用向量相似度检索）。
    返回与 query 最相关的文档段落。
    """
    # 简化：用关键词匹配模拟检索
    scored = []
    query_words = set(query.lower().split())
    for doc in KNOWLEDGE_BASE:
        content_words = set(doc["content"].lower())
        # 粗糙的关键词重叠评分
        overlap = len(query_words & content_words)
        if overlap > 0 or any(w in doc["content"] for w in query_words):
            scored.append((overlap, doc))
    scored.sort(reverse=True)
    return [doc for _, doc in scored[:top_k]]


# ===== 强验证问答系统 =====
def grounded_qa(question: str) -> GroundedAnswer:
    """
    带来源验证和置信分的问答。
    """
    # 1. 检索相关文档
    docs = retrieve_docs(question)

    if not docs:
        return GroundedAnswer(
            question=question,
            answer="参考文档中未找到相关信息，建议联系客服人员。",
            sources=[],
            confidence_score=0,
            confidence_reason="知识库中无相关文档",
            is_reliable=False,
            warning="⚠️ 知识库未命中，无法提供可靠答案。",
        )

    # 2. 构建带来源的 Prompt
    docs_text = "\n\n".join(
        f"[文档 {d['doc_id']}] {d['title']}\n{d['content']}"
        for d in docs
    )

    system_prompt = f"""你是一个严格的问答助手，必须遵守以下规则：
1. 只能基于下方参考文档回答，禁止使用任何文档外的知识。
2. 每个关键事实必须标注来源，格式：[来源: 文档ID]。
3. 文档中没有的信息，回答"参考文档中未找到此信息"。
4. 最后必须用 JSON 格式输出置信度评估，格式如下：
   {{"confidence_score": 整数0-100, "confidence_reason": "简短理由说明", "source_doc_ids": ["用到的文档ID列表"]}}
   评分标准：90-100分=文档有直接原文依据；60-89分=需要一定推断；0-59分=信息不足

参考文档：
{docs_text}
"""

    # 3. 调用模型（低温度，减少幻觉）
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": question},
        ],
        temperature=0.1,   # 低温度：保守输出
        top_p=0.1,         # 低top_p：减少随机性
    )

    raw_output = response.choices[0].message.content

    # 4. 解析输出（分离答案正文和 JSON 置信度）
    answer_text = raw_output
    confidence_score = 50
    confidence_reason = "默认评分"
    source_ids = []

    try:
        # 找到末尾的 JSON 块
        json_start = raw_output.rfind("{")
        json_end = raw_output.rfind("}") + 1
        if json_start != -1 and json_end > json_start:
            json_str = raw_output[json_start:json_end]
            meta = json.loads(json_str)
            confidence_score = int(meta.get("confidence_score", 50))
            confidence_reason = meta.get("confidence_reason", "")
            source_ids = meta.get("source_doc_ids", [])
            # 正文是 JSON 之前的部分
            answer_text = raw_output[:json_start].strip()
    except (json.JSONDecodeError, ValueError):
        pass  # 解析失败就用默认值

    # 5. 构建来源列表
    sources = [
        {"doc_id": d["doc_id"], "title": d["title"], "excerpt": d["content"][:100] + "..."}
        for d in docs if d["doc_id"] in source_ids
    ] or [
        {"doc_id": d["doc_id"], "title": d["title"], "excerpt": d["content"][:100] + "..."}
        for d in docs
    ]

    # 6. 低置信度警告
    warning = None
    if confidence_score < 60:
        warning = f"⚠️ 置信度较低（{confidence_score}分），建议人工核实或联系客服确认。"

    return GroundedAnswer(
        question=question,
        answer=answer_text,
        sources=sources,
        confidence_score=confidence_score,
        confidence_reason=confidence_reason,
        is_reliable=confidence_score >= 60,
        warning=warning,
    )


def print_answer(result: GroundedAnswer):
    """格式化输出问答结果"""
    print(f"\n{'='*60}")
    print(f"问题：{result.question}")
    print(f"{'='*60}")
    print(f"\n回答：\n{result.answer}")
    print(f"\n来源文档：")
    for s in result.sources:
        print(f"  [{s['doc_id']}] {s['title']}: {s['excerpt']}")
    print(f"\n置信度：{result.confidence_score}/100")
    print(f"评分说明：{result.confidence_reason}")
    print(f"可信状态：{'✅ 可信' if result.is_reliable else '❌ 不可信'}")
    if result.warning:
        print(f"\n{result.warning}")


if __name__ == "__main__":
    test_questions = [
        "退货需要多少天内申请？",            # 文档有直接答案，高置信度
        "黄金会员每月有几次免费快递？",      # 文档有直接答案，高置信度
        "Model X 的电池容量是多少？",        # 文档有直接答案，高置信度
        "这款手机防水等级是多少？",          # 文档中没有此信息，低置信度
    ]

    for q in test_questions:
        result = grounded_qa(q)
        print_answer(result)
```

## 验收交付：带置信分的输出规范

> 根据验收标准，系统输出必须同时携带来源和 Confidence Score。

### 标准输出格式

以下是系统输出的标准 JSON 结构，可作为 API 响应格式：

```json
{
  "question": "退货需要多少天内申请？",
  "answer": "商品支持购买后 30 天内无理由退货 [来源: policy-001]。退款将在收到退货后 3-5 个工作日内原路返还 [来源: policy-001]。",
  "sources": [
    {
      "doc_id": "policy-001",
      "title": "退换货政策",
      "excerpt": "商品支持购买后30天内无理由退货，退货时商品须保持原包装完好..."
    }
  ],
  "confidence_score": 95,
  "confidence_reason": "参考文档中有直接明确的原文依据，答案完全来源于 policy-001",
  "is_reliable": true,
  "warning": null
}
```

### 低置信度示例输出

```json
{
  "question": "这款手机防水等级是多少？",
  "answer": "参考文档中未找到关于防水等级的具体信息。",
  "sources": [],
  "confidence_score": 0,
  "confidence_reason": "知识库中无相关文档覆盖此问题",
  "is_reliable": false,
  "warning": "⚠️ 置信度较低（0分），建议人工核实或联系客服确认。"
}
```

### Confidence Score 评分标准表

| 分段 | 含义 | 处置建议 |
| --- | --- | --- |
| 90–100 | 文档有直接原文依据，答案可靠 | 直接展示给用户 |
| 70–89 | 文档有间接依据，需少量推断 | 展示并提示"来自推断" |
| 50–69 | 信息不充分，答案不确定性较高 | 展示但附加人工核实建议 |
| 0–49 | 文档无相关信息或明显超出范围 | 拒绝直接回答，引导至客服 |
