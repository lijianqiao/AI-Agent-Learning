# 混合检索与重排序

## 学习

**学习目标**

- BM25 稀疏关键字硬匹配算法与 Dense 向量软匹配融合权重计算
- 结合 Cross-Encoder 互注意力重排序大模型重新校准结果得分

**实战**

- 实现向量软召回库 + 倒排索引库，并在顶部增加 Reranker 重校权重引擎。

**验收标准**

- 针对前一周的刁钻查询干扰测试集，使相关段落命中率跃升 10% 以上。

---

## 第一部分学习：为什么需要混合检索？

### 单一检索的各自短板

**纯向量检索**（Dense）擅长语义匹配，但对精确关键词不敏感：
- 搜"订单号 ORD-20240315"，向量检索可能返回一堆包含"订单"的文档，但没有精确匹配这个编号

**纯关键词检索**（Sparse/BM25）擅长精确匹配，但不理解语义：
- 搜"退货流程"，只能匹配包含"退货"和"流程"的文档，搜不到"退换货步骤说明"

### 混合检索的核心思路

**两条腿走路**：同时用 BM25 和向量检索，取两者的并集，再用融合算法统一打分。

```
用户查询
    ├── BM25 稀疏检索 → 结果集 A（关键词精确匹配）
    └── Dense 向量检索 → 结果集 B（语义相似匹配）
            ↓
    融合算法（RRF / 加权求和）
            ↓
    统一排名结果 → Top-K
```

---

## 第二部分学习：BM25 算法

### 什么是 BM25？

**BM25 = Best Matching 25**，是传统搜索引擎（Google、Elasticsearch）的核心算法。它的本质是：**一个词在文档中出现次数越多、在全部文档中出现次数越少，这个词对该文档越重要。**

### 生活类比：判断一篇论文的主题

假设你要判断一篇论文是否关于"量子计算"：

- "的"这个字出现了200次 → 没用，每篇论文都有"的"
- "量子"出现了50次 → 有用，大部分论文不会频繁提到"量子"
- "纠缠"出现了30次 → 很有用，这是量子计算的专业术语

BM25 就是量化了这个直觉：**稀有词权重高，常见词权重低**。

### BM25 的核心公式（直觉版）

```
BM25(query, doc) = Σ IDF(term) × TF_normalized(term, doc)

其中：
  IDF(term) = log(总文档数 / 包含该词的文档数)  ← 越稀有越重要
  TF_normalized = 词频 / (词频 + k1)            ← 词频有上限，避免刷词
```

### BM25 的优势与局限

| 优势 | 局限 |
| --- | --- |
| 精确关键词匹配极强 | 不理解同义词（"退货"≠"退换货"） |
| 计算速度极快（倒排索引） | 不理解语义（"苹果手机"≠"iPhone"） |
| 无需 GPU | 对短查询效果好，长文本退化 |

## 第三部分学习：融合算法 — RRF

### RRF 是什么？

**RRF = Reciprocal Rank Fusion（倒数排名融合）**。核心思想极简：**不关心原始分数，只关心排名。每个结果的得分 = 各路检索中排名的倒数之和。**

```
公式：RRF_score(doc) = Σ 1 / (k + rank_i(doc))
其中 k 通常取 60
```

### 举例

```
BM25 检索结果排名：        向量检索结果排名：
  1. doc_A                   1. doc_C
  2. doc_B                   2. doc_A
  3. doc_C                   3. doc_D

doc_A 的 RRF 分数 = 1/(60+1) + 1/(60+2) = 0.0164 + 0.0161 = 0.0325
doc_B 的 RRF 分数 = 1/(60+2) + 0         = 0.0161
doc_C 的 RRF 分数 = 1/(60+3) + 1/(60+1)  = 0.0159 + 0.0164 = 0.0323
doc_D 的 RRF 分数 = 0        + 1/(60+3)  = 0.0159

最终排名：doc_A > doc_C > doc_B > doc_D
```

**doc_A 在两路检索中都表现不错，RRF 把它排到了第一。**

### RRF 的优点

- 不需要归一化各路检索的分数（BM25 和余弦相似度量纲完全不同）
- 实现简单、效果稳定
- 被 Elasticsearch、Weaviate 等生产系统广泛使用

## 第四部分学习：Cross-Encoder 重排序

### 为什么需要重排序？

初检（BM25 + Dense）速度快但不够精准。它们是**Bi-Encoder**架构：查询和文档**分别**编码，再比较向量距离。

**问题**：查询和文档之间没有直接"对话"，可能遗漏细微的语义关联。

### Cross-Encoder 的原理

把查询和文档**拼在一起**，送入一个模型，让模型直接判断"这个文档和查询有多相关"。

```
Bi-Encoder（初检）：
  查询 → 向量A    文档 → 向量B    距离(A, B) = 相关性分数
  速度快，但精度有限

Cross-Encoder（重排）：
  [查询 + 文档] → 模型 → 直接输出相关性分数
  速度慢（每对都要过一遍模型），但精度高
```

### 生活类比：招聘流程

**初检（Bi-Encoder）**：HR 快速看简历标题，挑出大致匹配的候选人（速度快，可能漏人）

**重排（Cross-Encoder）**：面试官逐个面试候选人，综合评估匹配度（速度慢，但判断准确）

### 两阶段检索架构

```
知识库（100万 Chunk）
        ↓
【第一阶段：粗召回】BM25 + Dense 各召回 Top-20
        ↓
    RRF 融合 → 取 Top-20（去重后）
        ↓
【第二阶段：精排】Cross-Encoder 对 20 条逐一打分
        ↓
    按分数重新排序 → 取 Top-5 送入 LLM
```

## 实战：完整的混合检索 + 重排序管道

```python
"""
混合检索 + RRF 融合 + Cross-Encoder 重排序。
依赖：pip install rank_bm25 chromadb openai sentence-transformers
"""
import numpy as np
from rank_bm25 import BM25Okapi
from openai import OpenAI
from dataclasses import dataclass

client = OpenAI()

# ===== 知识库 =====
DOCS = [
    {"id": "d1", "text": "商品支持购买后30天内无理由退货，退货时须保持原包装完好。"},
    {"id": "d2", "text": "退款将在收到退货后3-5个工作日内原路返还至支付账户。"},
    {"id": "d3", "text": "黄金会员每月享有10次免费快递权益。"},
    {"id": "d4", "text": "会员积分满1000分可兑换50元优惠券。"},
    {"id": "d5", "text": "产品保修期为一年，人为损坏不在保修范围内。"},
    {"id": "d6", "text": "下单后24小时内可免费取消订单。"},
    {"id": "d7", "text": "退换货过程中的运费由买家承担，黄金会员可免运费。"},
    {"id": "d8", "text": "如商品存在质量问题，可在收货后7天内申请换货或维修。"},
]

# ===== 1. BM25 稀疏检索 =====
import jieba
tokenized_docs = [list(jieba.cut(d["text"])) for d in DOCS]
bm25 = BM25Okapi(tokenized_docs)

def bm25_search(query: str, top_k: int = 10) -> list[tuple[str, float]]:
    tokens = list(jieba.cut(query))
    scores = bm25.get_scores(tokens)
    ranked = sorted(enumerate(scores), key=lambda x: x[1], reverse=True)[:top_k]
    return [(DOCS[idx]["id"], score) for idx, score in ranked if score > 0]

# ===== 2. Dense 向量检索 =====
def embed(text: str) -> list[float]:
    resp = client.embeddings.create(model="text-embedding-3-small", input=text)
    return resp.data[0].embedding

doc_embeddings = {d["id"]: np.array(embed(d["text"])) for d in DOCS}

def dense_search(query: str, top_k: int = 10) -> list[tuple[str, float]]:
    q_emb = np.array(embed(query))
    scored = []
    for doc_id, doc_emb in doc_embeddings.items():
        sim = float(np.dot(q_emb, doc_emb) / (np.linalg.norm(q_emb) * np.linalg.norm(doc_emb) + 1e-9))
        scored.append((doc_id, sim))
    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[:top_k]

# ===== 3. RRF 融合 =====
def rrf_fuse(results_list: list[list[tuple[str, float]]], k: int = 60) -> list[tuple[str, float]]:
    scores = {}
    for results in results_list:
        for rank, (doc_id, _) in enumerate(results):
            scores[doc_id] = scores.get(doc_id, 0) + 1.0 / (k + rank + 1)
    ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return ranked

# ===== 4. Cross-Encoder 重排序（用 LLM 模拟）=====
def cross_encoder_rerank(query: str, doc_ids: list[str], top_k: int = 5) -> list[tuple[str, float]]:
    """用 GPT 做轻量级相关性打分（生产环境用 cross-encoder 模型更高效）"""
    doc_map = {d["id"]: d["text"] for d in DOCS}
    scored = []
    for doc_id in doc_ids:
        resp = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {"role": "system", "content": "请评估以下文档与查询的相关性，只输出0-100的整数分。"},
                {"role": "user", "content": f"查询：{query}\n文档：{doc_map[doc_id]}"},
            ],
            temperature=0,
            max_tokens=5,
        )
        try:
            score = int(resp.choices[0].message.content.strip())
        except ValueError:
            score = 50
        scored.append((doc_id, score))
    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[:top_k]

# ===== 完整管道 =====
@dataclass
class SearchResult:
    doc_id: str
    text: str
    bm25_rank: int
    dense_rank: int
    rrf_score: float
    rerank_score: float

def hybrid_search(query: str, rerank_top: int = 5) -> list[SearchResult]:
    # 第一阶段：双路召回
    bm25_results = bm25_search(query, top_k=10)
    dense_results = dense_search(query, top_k=10)

    # RRF 融合
    fused = rrf_fuse([bm25_results, dense_results])
    candidate_ids = [doc_id for doc_id, _ in fused[:10]]

    # 第二阶段：Cross-Encoder 重排
    reranked = cross_encoder_rerank(query, candidate_ids, top_k=rerank_top)

    # 组装结果
    doc_map = {d["id"]: d["text"] for d in DOCS}
    bm25_rank_map = {doc_id: i+1 for i, (doc_id, _) in enumerate(bm25_results)}
    dense_rank_map = {doc_id: i+1 for i, (doc_id, _) in enumerate(dense_results)}
    rrf_map = dict(fused)

    return [
        SearchResult(
            doc_id=doc_id,
            text=doc_map[doc_id],
            bm25_rank=bm25_rank_map.get(doc_id, -1),
            dense_rank=dense_rank_map.get(doc_id, -1),
            rrf_score=round(rrf_map.get(doc_id, 0), 4),
            rerank_score=score,
        )
        for doc_id, score in reranked
    ]


if __name__ == "__main__":
    test_queries = [
        "东西买了不想要了怎么办",     # 语义查询（向量强）
        "退货 包装 30天",             # 关键词查询（BM25强）
        "黄金会员退换货运费谁出",     # 混合型（需要两路协作）
    ]
    for q in test_queries:
        print(f"\n{'='*60}")
        print(f"查询：{q}")
        results = hybrid_search(q)
        for r in results:
            print(f"  [{r.doc_id}] rerank={r.rerank_score} | BM25排名={r.bm25_rank} | Dense排名={r.dense_rank}")
            print(f"    {r.text[:60]}...")
```
