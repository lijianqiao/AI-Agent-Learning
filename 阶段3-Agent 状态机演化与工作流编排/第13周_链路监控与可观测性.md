## 第一部分学习：可观测性三大支柱

### 为什么 Agent 比传统后端更需要可观测性？

传统后端服务的调用链路是**确定性**的——代码写了 A→B→C，它就走 A→B→C，出了 Bug 看堆栈就行。

但 AI Agent 的链路是**非确定性**的——同一个输入，LLM 可能决定先调工具 A 再调工具 B，也可能反过来，甚至跳过某个工具直接给答案。每次 Tool 调用的入参、出参、耗时都不一样。更糟糕的是：

- LLM 推理本身是**黑盒**，你看不到它"在想什么"
- 一次复杂交互可能触发 **50~100 步** Tool 调用嵌套
- Token 消耗直接关联**成本**，延迟直接关联**用户体验**

不装可观测性就上线 Agent，等于蒙眼开赛车——不出事算运气，出了事无从排查。

### 生活类比：医院的三套监控系统

把你的 Agent 想象成一个大型综合医院：

| 医院监控 | 可观测性支柱 | Agent 场景 |
| --- | --- | --- |
| **病历档案**（每个患者的完整就诊记录） | **Traces（链路追踪）** | 一次请求从用户输入到最终回复的完整调用链路树 |
| **仪表盘看板**（今日门诊量、平均等候时间、ICU 占用率） | **Metrics（指标度量）** | Token 总消耗、P99 延迟、工具调用成功率、每日请求量 |
| **值班日志**（护士站每小时记录的异常事件） | **Logs（日志）** | 每个组件的运行时日志——报错、警告、关键状态变更 |

三者缺一不可：

- 只有 **Traces** 没有 Metrics = 你能看每个病人的病历，但不知道今天医院整体运转状况
- 只有 **Metrics** 没有 Traces = 仪表盘显示"平均等候 2 小时"，但你不知道是哪个科室拖后腿
- 只有 **Logs** 没有 Traces = 一堆散落的纸条，无法串成完整故事

### 三大支柱详解

#### Pillar 1: Traces（链路追踪）

Traces 记录的是**一次请求的完整生命周期**——从进入系统到离开系统，经过了哪些组件、每个组件花了多长时间、入参出参是什么。

在 Agent 场景中，一个 Trace 通常包含：

```
Trace: "用户问：帮我分析竞品报告"
├── Span: Planner 拆解任务 (320ms, 450 tokens)
│   ├── Span: LLM 推理 (280ms)
│   └── Span: 解析输出为子任务列表 (40ms)
├── Span: Executor 执行子任务1 - 读取报告 (1200ms)
│   ├── Span: Tool Call - file_reader (800ms)
│   └── Span: LLM 摘要 (400ms, 600 tokens)
├── Span: Executor 执行子任务2 - 对比分析 (900ms)
│   └── Span: LLM 推理 (900ms, 1200 tokens)
└── Span: 生成最终报告 (500ms, 800 tokens)

总耗时: 2920ms | 总 Token: 3050 | 工具调用: 1 次
```

#### Pillar 2: Metrics（指标度量）

Metrics 是**可聚合的数值型时间序列**——不关心单次请求细节，关心整体趋势和分布。

| 指标类别 | 具体指标 | 为什么重要 |
| --- | --- | --- |
| 成本类 | 每次请求平均 Token 消耗 | 直接关联 API 账单 |
| 成本类 | 每日总 Token 消耗趋势 | 预算预警 |
| 延迟类 | 端到端 P50 / P95 / P99 延迟 | 用户体验底线 |
| 延迟类 | 单次 LLM 调用延迟分布 | 定位慢调用 |
| 可靠性 | 工具调用成功率 | 链路健康度 |
| 可靠性 | 重试率 / 降级率 | 异常频率趋势 |
| 质量类 | LLM 输出平均评分（如 RAGAS） | 回答质量趋势 |

#### Pillar 3: Logs（日志）

Logs 是**离散的事件记录**——带时间戳的非结构化或半结构化文本。

```python
# Agent 日志示例
2025-03-15 14:23:01 [INFO]  request_id=abc123 | 收到用户请求: "分析竞品报告"
2025-03-15 14:23:01 [INFO]  request_id=abc123 | Planner 开始拆解任务
2025-03-15 14:23:02 [INFO]  request_id=abc123 | Planner 输出 3 个子任务
2025-03-15 14:23:02 [INFO]  request_id=abc123 | Executor 开始执行子任务1
2025-03-15 14:23:03 [WARN]  request_id=abc123 | Tool file_reader 首次调用超时，触发重试
2025-03-15 14:23:04 [INFO]  request_id=abc123 | Tool file_reader 重试成功 (attempt=2)
2025-03-15 14:23:05 [ERROR] request_id=abc123 | LLM 返回格式异常，触发降级
```

**关键原则：每条日志必须带 `request_id`，否则无法与 Trace 关联。**

### 三大支柱的协作关系

```
         ┌──────────────────────────────────────────────────┐
         │              Metrics Dashboard                    │
         │   "过去 1 小时 P99 延迟从 3s 飙升到 15s"          │
         └────────────────┬─────────────────────────────────┘
                          │ 发现异常，下钻排查
                          ↓
         ┌──────────────────────────────────────────────────┐
         │              Traces Explorer                      │
         │   "慢请求集中在 Tool=web_search 的 Span"          │
         └────────────────┬─────────────────────────────────┘
                          │ 定位具体 Span，查看详情
                          ↓
         ┌──────────────────────────────────────────────────┐
         │              Logs Search                          │
         │   "web_search API 返回 429 Too Many Requests"     │
         └──────────────────────────────────────────────────┘
```

从 Metrics 发现问题 → 用 Traces 定位到具体链路和组件 → 用 Logs 找到根因。这是可观测性排查的经典三板斧。

## 第二部分学习：分布式链路追踪原理（Trace / Span 模型）

### OpenTelemetry 核心概念迁移

OpenTelemetry（简称 OTel）是云原生领域的可观测性事实标准。虽然它最初为微服务设计，但其 Trace/Span 模型完美适配 AI Agent 链路追踪。

### 生活类比：快递物流追踪

你在网上买了一个东西，物流系统会给你一个**运单号（Trace ID）**。打开物流详情你会看到：

```
运单号: SF1234567890   ← Trace ID
├── 揽收 (上海浦东营业部, 09:00-09:15)          ← Span 1
├── 运输 (上海→杭州转运中心, 09:15-12:30)       ← Span 2
│   ├── 装车 (09:15-09:30)                      ← 子 Span 2.1
│   └── 公路运输 (09:30-12:30)                  ← 子 Span 2.2
├── 分拣 (杭州转运中心, 12:30-13:00)             ← Span 3
└── 派送 (杭州西湖营业部, 13:00-14:20)           ← Span 4
    ├── 快递员取件 (13:00-13:10)                 ← 子 Span 4.1
    └── 上门配送 (13:10-14:20)                   ← 子 Span 4.2

总耗时: 5小时20分钟
```

每个环节就是一个 **Span**，所有 Span 串起来就是一个完整的 **Trace**。每个 Span 记录了：开始时间、结束时间、操作名称、状态（成功/失败）、以及一些附加信息（哪个营业部、哪个快递员）。

### 核心数据模型

```
┌─────────────────────────────────────────────────────────────┐
│  Trace                                                       │
│  trace_id: "abc-123-def"                                     │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  Root Span (parent_span_id = None)                     │  │
│  │  span_id: "span-001"                                   │  │
│  │  name: "agent_request"                                 │  │
│  │  start_time: 1710489600.000                            │  │
│  │  end_time: 1710489603.500                              │  │
│  │  attributes: {request_id, user_id, model_name}         │  │
│  │                                                        │  │
│  │  ┌──────────────────────────────────┐                  │  │
│  │  │  Child Span                      │                  │  │
│  │  │  span_id: "span-002"            │                  │  │
│  │  │  parent_span_id: "span-001"     │                  │  │
│  │  │  name: "llm_call"              │                  │  │
│  │  │  attributes: {model, tokens,    │                  │  │
│  │  │    prompt_tokens, completion}    │                  │  │
│  │  └──────────────────────────────────┘                  │  │
│  │                                                        │  │
│  │  ┌──────────────────────────────────┐                  │  │
│  │  │  Child Span                      │                  │  │
│  │  │  span_id: "span-003"            │                  │  │
│  │  │  parent_span_id: "span-001"     │                  │  │
│  │  │  name: "tool_call:web_search"   │                  │  │
│  │  │  attributes: {tool_name, input,  │                  │  │
│  │  │    output, status}               │                  │  │
│  │  └──────────────────────────────────┘                  │  │
│  └────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 关键术语对照表

| OTel 原始概念 | Agent 链路映射 | 说明 |
| --- | --- | --- |
| Trace | 一次完整的用户请求处理 | 从接收输入到返回最终回复 |
| Span | 一个处理步骤 | LLM 调用、Tool 调用、Planner 拆解等 |
| Trace ID | Request ID | 全局唯一标识，贯穿整条链路 |
| Span ID | 步骤唯一标识 | 每个 Span 自己的 ID |
| Parent Span ID | 父步骤 ID | 表达嵌套关系（谁调用了谁） |
| Attributes | 附加元数据 | model_name, token_count, tool_input 等 |
| Events | Span 内的时间点事件 | "开始重试"、"收到 429 错误"等 |
| Status | Span 执行状态 | OK / ERROR / UNSET |

### Span 的四种关键时间属性

```python
# 每个 Span 的时间线
span.start_time  # Span 开始时刻
span.end_time    # Span 结束时刻
span.duration    # = end_time - start_time，本 Span 的总耗时

# 派生指标
self_time = span.duration - sum(child.duration for child in children)
# self_time = 本 Span 自身的处理时间（扣除子 Span 的耗时）
# 如果 self_time 很大但没有子 Span → 瓶颈就在这里
```

**生活类比**：你做一顿饭总共花了 1 小时（duration），其中洗菜 10 分钟、切菜 15 分钟、炒菜 20 分钟（子 Span）。那你"自己"花的时间 = 1h - 45min = 15min，这 15 分钟可能是你在找锅、翻菜谱、发呆——这就是隐藏的瓶颈。

## 第三部分学习：LangSmith vs Arize Phoenix 对比

### 为什么需要专用的 LLM 可观测性平台？

通用 APM 工具（如 Datadog、Jaeger）能做基本的链路追踪，但它们不理解 LLM 调用的语义——不知道什么是 Token、什么是 Prompt、什么是 Tool Call。专用平台在此基础上增加了：

- **Prompt 版本管理**——追踪每次请求用了哪个版本的 Prompt
- **Token 成本核算**——自动按模型定价计算每次调用的花费
- **LLM 输出质量评估**——内置评分器，对回复做自动质量评分
- **Playground 调试**——直接在 UI 上修改 Prompt 并重放请求

### 生活类比：通用体检 vs 专科检查

通用 APM = 去医院做常规体检，量血压、测心率、拍胸片——能发现明显问题，但看不到细节。

LLM 可观测性平台 = 做专科检查——心内科做心脏彩超、神内科做脑电图。针对 LLM 这个"器官"做深度检查，能看到 Token 级别的消耗、Prompt 级别的变化、输出质量的波动。

### 核心对比表

| 维度 | LangSmith | Arize Phoenix |
| --- | --- | --- |
| **厂商** | LangChain 官方 | Arize AI（独立公司） |
| **定位** | LangChain 生态深度绑定 | 框架无关的开放平台 |
| **部署方式** | SaaS 为主（有 Self-hosted） | 开源，本地部署优先 |
| **开源** | 核心闭源，客户端 SDK 开源 | 完全开源（Apache 2.0） |
| **数据隐私** | 数据发送到 LangSmith 云端 | 数据留在本地 |
| **LangChain 集成** | 原生无缝（一行代码开启） | 需手动 Callback 集成 |
| **非 LangChain 支持** | 支持但需额外 SDK | 天然支持任意框架 |
| **Trace 可视化** | 优秀，树状嵌套展示 | 优秀，支持 Trace 树 + 火焰图 |
| **Prompt 管理** | 内置 Prompt Hub | 无内置，需外部管理 |
| **评估能力** | 内置多种 Evaluator | 内置 + 支持 RAGAS 等外部评估 |
| **成本追踪** | 自动按模型定价计算 | 需手动配置价格表 |
| **告警** | 基于规则的告警 | 基于漂移检测的告警 |
| **免费额度** | 每月 5000 次 Trace 免费 | 本地部署完全免费 |
| **适用场景** | LangChain 技术栈、快速接入 | 数据敏感、多框架混用、定制化需求 |

### 选型决策树

```
你的项目用 LangChain 吗？
```

### LangSmith 快速接入示例

```python
"""
LangSmith 接入示例（LangChain 项目）
前提：pip install langsmith langchain langchain-openai
"""
import os

os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-langsmith-api-key"
os.environ["LANGCHAIN_PROJECT"] = "agent-week13-demo"

from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.tools import tool

@tool
def search_database(query: str) -> str:
    """搜索内部数据库"""
    return f"数据库查询结果: 找到与 '{query}' 相关的 3 条记录"

@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression)
        return f"计算结果: {expression} = {result}"
    except Exception as e:
        return f"计算错误: {e}"

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有用的助手，可以搜索数据库和做计算。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_openai_tools_agent(llm, [search_database, calculate], prompt)
executor = AgentExecutor(agent=agent, tools=[search_database, calculate], verbose=True)

result = executor.invoke({"input": "帮我查一下销售数据，然后算一下总销售额 1200+3400+5600"})
print(result["output"])

# 运行后在 https://smith.langchain.com 可看到完整 Trace 树
# 包括每次 LLM 调用的 prompt/completion、每次 Tool 调用的 input/output、耗时、Token 消耗
```

## 第四部分学习：RequestId 全链路追踪设计

### 为什么 RequestId 是可观测性的灵魂？

没有 RequestId 的日志就像没有姓名的病历——一堆检查报告散落在桌上，你不知道哪几张属于同一个病人。

**RequestId 的核心使命**：把分散在不同组件、不同层级的 Traces / Metrics / Logs **串联成一条完整的链路**。

### 生活类比：住院手腕带

住院时护士会给你戴一个手腕带，上面有你的唯一住院号。无论你去做 CT、抽血、还是手术，所有科室都会扫这个手腕带——确保所有检查记录都归档到你的名下。

RequestId 就是 Agent 请求的"手腕带"——从入口生成，一路传递到每个组件、每次 LLM 调用、每次 Tool 调用，最终所有记录都能通过它串起来。

### 设计原则

| 原则 | 要求 | 说明 |
| --- | --- | --- |
| **全局唯一** | UUID v4 或 ULID | 避免碰撞，26 字节 ULID 更推荐（自带时间排序） |
| **入口生成** | 在请求最外层统一生成 | 不要让中间组件各自生成 |
| **全链路透传** | 每个函数调用都携带 | 通过参数显式传递或 Context 隐式传递 |
| **存储关联** | 日志、Trace、Metric 都打上 | 事后检索的唯一入口 |
| **不可变** | 生成后不允许修改 | 保证链路一致性 |

### 两种透传方案对比

| 方案 | 实现方式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 显式传参 | 每个函数签名加 `request_id` 参数 | 简单直观、无魔法 | 侵入性强，改动大 |
| Context 隐式传递 | 用 `contextvars.ContextVar`（Python）或 `threading.local` | 无侵入，函数签名不变 | 需理解上下文机制，异步场景需注意 |

### 基于 contextvars 的 RequestId 透传

```python
"""
使用 Python contextvars 实现 RequestId 隐式透传
适用于异步场景（asyncio 天然支持 contextvars）
"""
import contextvars
import uuid
import logging
from functools import wraps
from datetime import datetime

# 定义上下文变量
_request_id_var: contextvars.ContextVar[str] = contextvars.ContextVar(
    "request_id", default="no-request-id"
)

def get_request_id() -> str:
    return _request_id_var.get()

def set_request_id(request_id: str | None = None) -> str:
    rid = request_id or str(uuid.uuid4())
    _request_id_var.set(rid)
    return rid


class RequestIdFilter(logging.Filter):
    """日志过滤器：自动把 request_id 注入每条日志"""
    def filter(self, record):

        return True

def with_request_id(func):
    """装饰器：为入口函数自动生成 RequestId"""
```

## 第五部分学习：关键指标体系

### Agent 可观测性的黄金指标

传统后端有 RED（Rate / Error / Duration）指标。AI Agent 在此基础上需要额外关注 **Token 成本**和**推理质量**，形成 **REDTQ** 指标体系：

| 指标分类 | 指标名称 | 计算方式 | 告警阈值（参考） |
| --- | --- | --- | --- |
| **R**ate（请求量） | 每分钟请求数 | COUNT(requests) / minute | > 100 RPM 需关注资源 |
| **E**rror（错误率） | 请求失败率 | failed / total × 100% | > 5% 触发告警 |
| **D**uration（延迟） | P95 端到端延迟 | PERCENTILE(duration, 0.95) | > 10s 触发告警 |
| **T**oken（成本） | 每请求平均 Token | SUM(tokens) / COUNT(requests) | > 5000 tokens 需优化 |
| **Q**uality（质量） | 输出评分均值 | AVG(quality_score) | < 0.7 触发告警 |

### 细粒度 Token 追踪

```python
"""
细粒度 Token 消耗追踪
记录每次 LLM 调用的 prompt_tokens / completion_tokens / total_tokens
"""
from dataclasses import dataclass, field
from typing import Optional
import time


@dataclass
class TokenUsage:
    prompt_tokens: int = 0
    completion_tokens: int = 0

    @property
    def total_tokens(self) -> int:
        return self.prompt_tokens + self.completion_tokens

    @property
    def estimated_cost_usd(self) -> float:
        """以 GPT-4o-mini 定价估算（$0.15/1M input, $0.60/1M output）"""
        input_cost = self.prompt_tokens * 0.15 / 1_000_000
        output_cost = self.completion_tokens * 0.60 / 1_000_000
        return input_cost + output_cost


@dataclass
class SpanMetrics:
    span_name: str
    start_time: float = field(default_factory=time.time)
    end_time: Optional[float] = None
    token_usage: TokenUsage = field(default_factory=TokenUsage)
    tool_calls_count: int = 0
    retry_count: int = 0
    status: str = "RUNNING"

    @property
    def duration_ms(self) -> float:
        if self.end_time is None:
            return (time.time() - self.start_time) * 1000
        return (self.end_time - self.start_time) * 1000

    def finish(self, status: str = "OK"):
        self.end_time = time.time()
        self.status = status


@dataclass
class RequestMetrics:
    """一次请求的汇总指标"""
    request_id: str
    spans: list[SpanMetrics] = field(default_factory=list)

    @property
    def total_tokens(self) -> int:
        return sum(s.token_usage.total_tokens for s in self.spans)

    @property
    def total_cost_usd(self) -> float:
        return sum(s.token_usage.estimated_cost_usd for s in self.spans)

    @property
    def total_duration_ms(self) -> float:
        if not self.spans:
            return 0
        return max(s.duration_ms for s in self.spans)

    @property
    def total_tool_calls(self) -> int:
        return sum(s.tool_calls_count for s in self.spans)

    @property
    def total_retries(self) -> int:
        return sum(s.retry_count for s in self.spans)

    def summary(self) -> str:
        lines = [
            f"{'='*60}",
            f"请求指标汇总 (request_id: {self.request_id})",
            f"{'='*60}",
            f"总耗时:       {self.total_duration_ms:.0f} ms",
            f"总 Token:     {self.total_tokens}",
            f"  - Prompt:   {sum(s.token_usage.prompt_tokens for s in self.spans)}",
            f"  - Complete: {sum(s.token_usage.completion_tokens for s in self.spans)}",
            f"预估成本:     ${self.total_cost_usd:.6f}",
            f"工具调用:     {self.total_tool_calls} 次",
            f"重试次数:     {self.total_retries} 次",
            f"{'='*60}",
            f"各 Span 明细:",
        ]
        for s in self.spans:
            lines.append(
                f"  [{s.status}] {s.span_name}: "
                f"{s.duration_ms:.0f}ms, "
                f"{s.token_usage.total_tokens} tokens, "
                f"{s.tool_calls_count} tools, "
                f"{s.retry_count} retries"
            )
        return "\n".join(lines)
```

## 第六部分学习：自研轻量级 Tracer + Trace 树可视化

### 为什么要自研？

LangSmith 和 Phoenix 都很强大，但在以下场景你可能需要自研轻量级方案：

| 场景 | 原因 |
| --- | --- |
| 学习理解原理 | 自己造一遍轮子才真正理解 Trace/Span 模型 |
| 极简嵌入需求 | 不想引入重量级依赖，只需要基本链路记录 |
| 定制化展示 | 需要按业务需求定制 Trace 树的展示格式 |
| 离线环境 | 无法连接外部 SaaS 服务 |

### 生活类比：手写家谱 vs 用族谱软件

Phoenix/LangSmith = 用专业族谱软件，功能齐全但要学习工具。

自研 Tracer = 手写家谱，格式自己定，想怎么画就怎么画。虽然简陋，但你对"辈分关系如何表达"理解得最透彻。

### 完整实现：轻量级 Tracer

```python
"""
自研轻量级 Tracer
支持：嵌套 Span、自动耗时统计、RequestId 贯穿、Trace 树可视化

这是一个完全独立、可直接运行的实现，不依赖任何外部 LLM API。
"""
from __future__ import annotations

import uuid
import time
import json
import contextvars
from dataclasses import dataclass, field, asdict
from typing import Any, Optional
from contextlib import contextmanager
from datetime import datetime


# =====================================================
# 核心数据结构
# =====================================================

@dataclass
class SpanRecord:
    """单个 Span 的完整记录"""
    span_id: str
    parent_span_id: Optional[str]
    trace_id: str
    name: str
    start_time: float
    end_time: Optional[float] = None
    status: str = "RUNNING"
    attributes: dict[str, Any] = field(default_factory=dict)
    events: list[dict[str, Any]] = field(default_factory=list)

    @property
    def duration_ms(self) -> float:
        if self.end_time is None:
            return (time.time() - self.start_time) * 1000
        return (self.end_time - self.start_time) * 1000

    def set_attribute(self, key: str, value: Any):
        self.attributes[key] = value

    def add_event(self, name: str, attributes: dict | None = None):
        self.events.append({
            "name": name,
            "timestamp": time.time(),
            "attributes": attributes or {},
        })

    def finish(self, status: str = "OK"):
        self.end_time = time.time()
        self.status = status


# =====================================================
# Tracer 核心引擎
# =====================================================

_current_span: contextvars.ContextVar[Optional[SpanRecord]] = contextvars.ContextVar(
    "_current_span", default=None
)


class Tracer:
    """轻量级链路追踪器"""

    def __init__(self):
        self._traces: dict[str, list[SpanRecord]] = {}

    def _generate_id(self) -> str:
        return uuid.uuid4().hex[:16]

    @contextmanager
    def start_trace(self, name: str, request_id: str | None = None):
        """启动一个新的 Trace（根 Span）"""
        trace_id = request_id or str(uuid.uuid4())
        span = SpanRecord(
            span_id=self._generate_id(),
            parent_span_id=None,
            trace_id=trace_id,
            name=name,
            start_time=time.time(),
        )
        self._traces.setdefault(trace_id, []).append(span)
        token = _current_span.set(span)
        try:
            yield span
            span.finish("OK")
        except Exception as e:
            span.add_event("exception", {"type": type(e).__name__, "message": str(e)})
            span.finish("ERROR")
            raise
        finally:
            _current_span.reset(token)

    @contextmanager
    def start_span(self, name: str, **attrs):
        """在当前 Trace 内创建子 Span"""
        parent = _current_span.get()
        if parent is None:
            raise RuntimeError("start_span 必须在 start_trace 上下文内调用")

        span = SpanRecord(
            span_id=self._generate_id(),
            parent_span_id=parent.span_id,
            trace_id=parent.trace_id,
            name=name,
            start_time=time.time(),
            attributes=attrs,
        )
        self._traces[parent.trace_id].append(span)
        token = _current_span.set(span)
        try:
            yield span
            span.finish("OK")
        except Exception as e:
            span.add_event("exception", {"type": type(e).__name__, "message": str(e)})
            span.finish("ERROR")
            raise
        finally:
            _current_span.reset(token)

    def get_trace(self, trace_id: str) -> list[SpanRecord]:
        return self._traces.get(trace_id, [])

    def render_trace_tree(self, trace_id: str) -> str:
        """将 Trace 渲染为文本树"""
        spans = self.get_trace(trace_id)
        if not spans:
            return f"Trace {trace_id} 不存在"

        span_map = {s.span_id: s for s in spans}
        children_map: dict[Optional[str], list[SpanRecord]] = {}
        root = None
        for s in spans:
            children_map.setdefault(s.parent_span_id, []).append(s)
            if s.parent_span_id is None:
                root = s

        if root is None:
            return "未找到根 Span"

        lines: list[str] = []
        total_tokens = sum(
            s.attributes.get("tokens", 0) for s in spans
        )
        total_tool_calls = sum(
            1 for s in spans if s.name.startswith("tool:")
        )
        total_duration = root.duration_ms

        lines.append(f"{'='*70}")
        lines.append(f"  Trace ID:  {trace_id}")
        lines.append(f"  总耗时:    {total_duration:.0f} ms")
        lines.append(f"  总 Token:  {total_tokens}")
        lines.append(f"  工具调用:  {total_tool_calls} 次")
        lines.append(f"  Span 数:   {len(spans)}")
        lines.append(f"{'='*70}")
        lines.append("")

        def _render(span: SpanRecord, prefix: str, is_last: bool):
            connector = "└── " if is_last else "├── "
            status_icon = "✅" if span.status == "OK" else "❌" if span.status == "ERROR" else "⏳"

            info_parts = [f"{span.duration_ms:.0f}ms"]
            if "tokens" in span.attributes:
                info_parts.append(f"{span.attributes['tokens']} tokens")
            if "model" in span.attributes:
                info_parts.append(span.attributes["model"])
            if "tool_input" in span.attributes:
                tool_in = str(span.attributes["tool_input"])
                if len(tool_in) > 40:
                    tool_in = tool_in[:40] + "..."
                info_parts.append(f"input={tool_in}")

            info_str = ", ".join(info_parts)
            lines.append(f"{prefix}{connector}{status_icon} {span.name} ({info_str})")

            children = children_map.get(span.span_id, [])
            child_prefix = prefix + ("    " if is_last else "│   ")
            for i, child in enumerate(children):
                _render(child, child_prefix, i == len(children) - 1)

        _render(root, "", True)

        if root.events:
            lines.append("")
            lines.append("事件:")
            for evt in root.events:
                lines.append(f"  - {evt['name']}: {evt.get('attributes', {})}")

        return "\n".join(lines)

    def export_json(self, trace_id: str) -> str:
        """导出 Trace 为 JSON（可供前端 Trace 查看器消费）"""
        spans = self.get_trace(trace_id)
        data = []
        for s in spans:
            data.append({
                "span_id": s.span_id,
                "parent_span_id": s.parent_span_id,
                "trace_id": s.trace_id,
                "name": s.name,
                "start_time": s.start_time,
                "end_time": s.end_time,
                "duration_ms": s.duration_ms,
                "status": s.status,
                "attributes": s.attributes,
                "events": s.events,
            })
        return json.dumps(data, indent=2, ensure_ascii=False)


# =====================================================
# 模拟 Agent 全链路（不依赖外部 API）
# =====================================================

tracer = Tracer()


def simulate_llm_call(prompt: str, delay: float = 0.1) -> dict:

    time.sleep(delay)
    return {
        "content": f"模拟回复: 基于 '{prompt[:30]}...' 的分析结果",
```

### 用 Tracer 复现百步复杂流转

```python
"""
模拟 100 步复杂流转并生成 Trace 树
验收目标：根据唯一 RequestId 重现完整的百步轨迹
"""
import random
import time
import uuid

# 复用上面定义的 Tracer（实际使用时放在同一个文件或 import）
# from tracer import Tracer, tracer

TOOL_NAMES = ["database_query", "web_search", "file_reader", "calculator", "email_sender"]
MODELS = ["gpt-4o-mini", "gpt-4o", "claude-3-haiku"]


def simulate_100_step_agent():
    """模拟包含 100 步操作的复杂 Agent"""
    request_id = f"req-100step-{uuid.uuid4().hex[:8]}"
    step_count = 0

    with tracer.start_trace("complex_agent_100steps", request_id=request_id) as root:
        root.set_attribute("description", "100步复杂流转模拟")

        # 阶段1: Planner 拆解（生成 10 个子任务）
        with tracer.start_span("phase1_planning") as plan_span:
            model = random.choice(MODELS)
            with tracer.start_span("planner_llm", model=model) as llm:
                time.sleep(0.02)
                llm.set_attribute("tokens", random.randint(200, 500))
                step_count += 1
            plan_span.set_attribute("subtask_count", 10)

        # 阶段2: 10 个子任务，每个子任务包含多步
        for task_idx in range(1, 11):
            with tracer.start_span(f"subtask_{task_idx}",
                                   description=f"子任务{task_idx}: 数据分析流程") as task_span:

                # 每个子任务：2~3 个工具调用 + 1~2 次 LLM 推理
                num_tools = random.randint(2, 3)
                num_llm = random.randint(1, 2)

                for tool_idx in range(num_tools):
                    tool_name = random.choice(TOOL_NAMES)
                    with tracer.start_span(f"tool:{tool_name}",
                                           tool_input={"param": f"task{task_idx}_step{tool_idx}"}) as t:
                        delay = random.uniform(0.01, 0.05)
                        time.sleep(delay)
                        if random.random() < 0.05:
                            t.add_event("retry", {"attempt": 2, "reason": "timeout"})
                        t.set_attribute("tool_output", f"result_{task_idx}_{tool_idx}")
                        step_count += 1

                for llm_idx in range(num_llm):
                    model = random.choice(MODELS)
                    with tracer.start_span(f"llm_reasoning_{llm_idx+1}", model=model) as l:
```

### 存储方案对比

| 方案 | 适用规模 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 内存字典 | 开发调试 | 极简、零依赖 | 重启丢失、内存有限 |
| SQLite | 单机小规模 | 文件级持久化、SQL 查询 | 不支持并发写入 |
| PostgreSQL + JSONB | 中等规模 | 结构化 + 半结构化混合查询 | 需运维数据库 |
| Elasticsearch | 大规模全文检索 | 强大的搜索与聚合能力 | 资源消耗大 |
| ClickHouse | 海量时序分析 | 列存压缩、聚合极快 | 不擅长点查 |

### SQLite 持久化示例

```python
"""
将 Trace 数据持久化到 SQLite，支持按 RequestId 检索
"""
import sqlite3
import json
import time
import uuid
from typing import Optional

DB_PATH = "traces.db"


def init_db():
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS spans (
            span_id TEXT PRIMARY KEY,
            trace_id TEXT NOT NULL,
            parent_span_id TEXT,
            name TEXT NOT NULL,
            start_time REAL NOT NULL,
            end_time REAL,
            duration_ms REAL,
            status TEXT DEFAULT 'RUNNING',
            attributes TEXT DEFAULT '{}',
            events TEXT DEFAULT '[]',
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    """)
    conn.execute("CREATE INDEX IF NOT EXISTS idx_trace_id ON spans(trace_id)")
    conn.execute("CREATE INDEX IF NOT EXISTS idx_parent ON spans(parent_span_id)")
    conn.commit()
    return conn


def save_span(conn: sqlite3.Connection, span: dict):
    conn.execute("""
        INSERT OR REPLACE INTO spans
        (span_id, trace_id, parent_span_id, name, start_time, end_time, duration_ms, status, attributes, events)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
    """, (
        span["span_id"],
        span["trace_id"],
        span.get("parent_span_id"),
        span["name"],
        span["start_time"],
        span.get("end_time"),
        span.get("duration_ms", 0),
        span.get("status", "OK"),
        json.dumps(span.get("attributes", {}), ensure_ascii=False),
        json.dumps(span.get("events", []), ensure_ascii=False),
    ))
    conn.commit()


def load_trace(conn: sqlite3.Connection, trace_id: str) -> list[dict]:
    cursor = conn.execute(
        "SELECT * FROM spans WHERE trace_id = ? ORDER BY start_time",
        (trace_id,)
    )
    columns = [desc[0] for desc in cursor.description]
    rows = []
    for row in cursor.fetchall():
        d = dict(zip(columns, row))
        d["attributes"] = json.loads(d["attributes"])
        d["events"] = json.loads(d["events"])
        rows.append(d)
    return rows


def render_tree_from_db(conn: sqlite3.Connection, trace_id: str) -> str:
    """从数据库加载 Trace 并渲染为树"""
    spans = load_trace(conn, trace_id)
    if not spans:
        return f"Trace {trace_id} 不存在"

    children_map: dict[Optional[str], list[dict]] = {}
    root = None
    for s in spans:
        parent = s.get("parent_span_id")
        children_map.setdefault(parent, []).append(s)
        if parent is None:
            root = s

    if root is None:
        return "未找到根 Span"

    lines = [
        f"{'='*60}",
        f"  Trace ID: {trace_id}",
        f"  Span 总数: {len(spans)}",
        f"  总耗时: {root.get('duration_ms', 0):.0f} ms",
        f"{'='*60}",
        "",
    ]

    def _render(span: dict, prefix: str, is_last: bool):
        connector = "└── " if is_last else "├── "
        icon = "✅" if span["status"] == "OK" else "❌"
        dur = span.get("duration_ms", 0)
        attrs = span.get("attributes", {})
        token_info = f", {attrs['tokens']} tokens" if "tokens" in attrs else ""
        lines.append(f"{prefix}{connector}{icon} {span['name']} ({dur:.0f}ms{token_info})")
        children = children_map.get(span["span_id"], [])
        child_prefix = prefix + ("    " if is_last else "│   ")
        for i, child in enumerate(children):
            _render(child, child_prefix, i == len(children) - 1)

    _render(root, "", True)
    return "\n".join(lines)


# ---------- 演示 ----------

conn = init_db()

demo_trace_id = f"req-{uuid.uuid4().hex[:8]}"
demo_spans = [
    {"span_id": "s1", "trace_id": demo_trace_id, "parent_span_id": None,
     "name": "agent_request", "start_time": time.time(), "end_time": time.time() + 2.5,
     "duration_ms": 2500, "status": "OK", "attributes": {"user_input": "分析报告"}},
    {"span_id": "s2", "trace_id": demo_trace_id, "parent_span_id": "s1",
     "name": "planner", "start_time": time.time(), "end_time": time.time() + 0.3,
     "duration_ms": 300, "status": "OK", "attributes": {"tokens": 450, "model": "gpt-4o-mini"}},
    {"span_id": "s3", "trace_id": demo_trace_id, "parent_span_id": "s1",
     "name": "tool:database_query", "start_time": time.time() + 0.3, "end_time": time.time() + 1.1,
     "duration_ms": 800, "status": "OK", "attributes": {"tool_input": "SELECT * FROM sales"}},
    {"span_id": "s4", "trace_id": demo_trace_id, "parent_span_id": "s1",
     "name": "executor_llm", "start_time": time.time() + 1.1, "end_time": time.time() + 2.0,
     "duration_ms": 900, "status": "OK", "attributes": {"tokens": 1200, "model": "gpt-4o"}},
    {"span_id": "s5", "trace_id": demo_trace_id, "parent_span_id": "s1",

     "duration_ms": 500, "status": "OK", "attributes": {"tokens": 800}},
```

## 第八部分学习：生产环境最佳实践

### 采样策略

生产环境不可能 100% 记录所有 Trace（成本和性能都撑不住），需要采样：

| 采样策略 | 说明 | 适用场景 |
| --- | --- | --- |
| 固定比例采样 | 每 N 个请求记录 1 个（如 10%） | 流量大、请求模式均匀 |
| 头部采样 | 在请求入口决定是否采样 | 保证同一 Trace 的所有 Span 要么全采、要么全不采 |
| 尾部采样 | 等 Trace 完成后，根据结果决定是否保留 | 保留错误/慢请求的 Trace，丢弃正常请求 |
| 自适应采样 | 根据实时流量动态调整比例 | 流量波动大的场景 |

**推荐策略**：尾部采样——正常请求采 5%，错误请求采 100%，慢请求（延迟 > P95）采 100%。

### 生活类比：安检抽查

- **固定比例采样** = 每 10 个人查 1 个，简单但可能漏掉"坏人"
- **尾部采样** = 让所有人先过安检门，门响了的（异常请求）100% 开箱检查，没响的抽查 5%

### 性能影响控制

| 措施 | 说明 |
| --- | --- |
| 异步上报 | Span 数据写入缓冲队列，后台线程批量上报 |
| 字段裁剪 | 大字段（如完整 Prompt）只记录前 N 个字符或哈希 |
| 批量写入 | 攒够一批（如 100 条）或定时（如每 5 秒）批量写入存储 |
| 独立存储 | Trace 数据用独立的存储实例，不影响业务数据库 |

```python
def evaluate_alerts(metrics: dict) -> list[dict]:
    """评估告警规则，返回触发的告警列表"""
    triggered = []
    checks = {
        "high_latency": metrics.get("p95_latency_ms", 0) > 10000,
        "high_error_rate": metrics.get("error_rate", 0) > 0.05,
        "token_budget_exceeded": metrics.get("daily_tokens", 0) > 1_000_000,
        "high_retry_rate": metrics.get("retry_rate", 0) > 0.20,
        "tool_call_explosion": metrics.get("avg_tool_calls_per_request", 0) > 20,
    }
    for rule_name, is_triggered in checks.items():
        if is_triggered:
            triggered.append({
                "rule": rule_name,
                **ALERT_RULES[rule_name],
            })
    return triggered
```
