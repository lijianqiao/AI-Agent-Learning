## 第一部分学习：长期记忆 vs 短期记忆的本质区别

### LLM 的"记忆"天然缺陷

所有大语言模型都有一个根本性限制：**Context Window 是它唯一的"记忆"**。一旦对话超出窗口或新开一轮会话，之前所有信息都像从未存在过一样消失了。

这意味着：

1. **单会话内**：模型只能"记住"当前 Context Window 里的内容（短期记忆）
2. **跨会话时**：模型对你是谁、喜欢什么、上次聊了什么，一无所知（没有长期记忆）

对于工具类应用（翻译、写代码）这还能忍，但对于**个人助手、客服、教育辅导**等需要持续关系的场景，这就是致命缺陷——用户不愿意每次开口都重新介绍自己。

### 生活类比：金鱼 vs 管家

**没有长期记忆的 LLM** = 一条金鱼。每 7 秒（每次新会话）它都忘了你是谁。你说"我上次说过我对坚果过敏"，它一脸茫然。

**有长期记忆的 Agent** = 你家的老管家。虽然他不可能记住每一句日常闲聊（短期记忆会遗忘），但他有一本**私人档案本**（长期记忆库），记录了你的关键信息：过敏源、口味偏好、作息习惯、重要日期。每次服务前他都先翻一翻档案本，所以即使隔了三个月没见，他依然知道"先生不吃香菜，喜欢微甜的咖啡"。

### 短期记忆与长期记忆的本质对比

| 维度 | 短期记忆（Context Window） | 长期记忆（外部存储） |
| --- | --- | --- |
| 载体 | 当前对话的 Token 序列 | 外部数据库 / 向量库 / KV 存储 |
| 生命周期 | 单次会话内有效，会话结束即消失 | 持久化存储，跨会话、跨设备 |
| 容量 | 受限于模型窗口（4K-128K Token） | 理论上无上限 |
| 写入方式 | 用户/系统消息自动进入 | 需要主动提取、写入存储 |
| 读取方式 | 模型直接可见 | 需要检索后注入 Context |
| 精度 | 完整保留原文 | 提取时可能丢失细节或引入噪声 |
| 典型内容 | 当前对话上下文、系统指令 | 用户偏好、历史行为、实体属性 |
| 类比 | 你正在进行的对话 | 你的通讯录 + 备忘录 |

### 两者如何协作？

在工程实践中，短期记忆和长期记忆不是二选一，而是**分层互补**：

```
用户发起新会话
       ↓
┌──────────────────────────────────────┐
│          长期记忆检索层              │
│                                      │
│  1. 用 user_id 查询画像库            │
│  2. 获取结构化属性（姓名/职业/偏好）  │
│  3. 向量检索相关历史片段              │
└─────────────┬────────────────────────┘
              ↓
┌──────────────────────────────────────┐
│          System Prompt 注入层        │
│                                      │
│  将画像信息拼接到 System Prompt：     │
│  "该用户是前端工程师，偏好简洁风格，  │
│   上次讨论了 React 性能优化问题"      │
└─────────────┬────────────────────────┘
              ↓
┌──────────────────────────────────────┐
│          短期记忆（Context Window）   │
│                                      │
│  System Prompt（含画像）+ 当前对话    │
│  模型基于完整上下文生成个性化回复      │
└──────────────────────────────────────┘
```

### 关键设计原则

**原则一：长期记忆要"瘦"**

不要试图把所有历史对话都存进长期记忆。长期记忆应该是**高度提炼的结构化属性**——就像管家的档案本只记"先生不吃香菜"，不会逐字抄写你和厨师的对话原文。

**原则二：读取要"快"且"准"**

每次新会话开始时，检索长期记忆的延迟必须够低（百毫秒级），召回的信息必须和当前话题相关——不相关的画像信息注入 Context 只会浪费 Token 还可能误导模型。

**原则三：写入要"异步"**

画像提取是计算密集型任务（需要 LLM 做实体抽取），不能阻塞用户的实时对话。正确做法是**异步后台处理**——对话结束后、在低负载时段批量提取。

## 第二部分学习：用户画像模型设计

### 什么是用户画像？

用户画像（User Profile）是对用户核心特征的结构化描述。在 AI Agent 场景中，画像的目的很单纯：**让模型在下一次对话中表现得"认识你"**。

### 生活类比：酒店 VIP 卡片

高端酒店为 VIP 客人准备的"客户偏好卡"就是一份用户画像：

- **结构化属性**：姓名张三、公司 ABC 科技、职位 CTO、枕头偏好硬枕、房间温度 22°C、不吃海鲜
- **向量化偏好**：上次入住聊过量子计算话题（可以向量化存储以便语义匹配）

下次张三入住，前台一扫卡就知道：准备硬枕、调 22 度、餐厅备注"忌海鲜"——无需张三重复交代。

### 画像模型的双层架构

用户画像由两层组成：**结构化属性层**和**向量化偏好层**。

| 层级 | 存储内容 | 存储方式 | 查询方式 | 适用场景 |
| --- | --- | --- | --- | --- |
| 结构化属性层 | 姓名、职业、公司、偏好标签等确定性属性 | 关系型数据库 / KV Store（Redis, SQLite） | 精确匹配：`WHERE user_id = ?` | "用户叫什么名字？" |
| 向量化偏好层 | 兴趣话题、讨论历史摘要、行为模式等模糊偏好 | 向量数据库（Chroma, Milvus, Qdrant） | 语义检索：与当前话题最相似的 Top-K 偏好 | "用户最近关心什么？" |

### 为什么需要双层而不是只用一层？

| 方案 | 优点 | 缺点 | 适用场景 |
| --- | --- | --- | --- |
| 纯结构化（只用数据库） | 查询快、确定性强 | 无法捕捉"模糊偏好"和语义关联 | 简单场景，属性固定 |
| 纯向量化（只用向量库） | 语义检索能力强 | 精确属性查询不如数据库，召回不稳定 | 开放域聊天机器人 |
| 双层融合（推荐） | 确定性属性秒查 + 模糊偏好语义匹配 | 架构稍复杂，需维护两套存储 | 生产级个人助手 |

### 结构化属性层的 Schema 设计

```python
from dataclasses import dataclass, field
from datetime import datetime
from typing import Optional


@dataclass
class UserProfile:
    """用户画像结构化属性层"""
    user_id: str
    
    # 基础身份
    name: Optional[str] = None
    company: Optional[str] = None
    role: Optional[str] = None           # 如"前端工程师""产品经理"
    industry: Optional[str] = None       # 如"互联网""金融"
    
    # 偏好标签
    interests: list[str] = field(default_factory=list)    # ["React", "性能优化", "TypeScript"]
    communication_style: Optional[str] = None              # "简洁直接" / "详细解释" / "代码为主"
    language_preference: Optional[str] = None              # "中文" / "English"
    
    # 行为统计
    total_sessions: int = 0
    favorite_topics: list[str] = field(default_factory=list)  # 按频次排序的高频话题
    last_active: Optional[datetime] = None
    
    # 元信息
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)
    confidence_scores: dict[str, float] = field(default_factory=dict)  # 每个属性的置信度
```

### 向量化偏好层的设计

向量化偏好层存储的不是固定属性，而是**用户讨论过的话题摘要**——这些摘要被编码为向量后存入向量库，支持语义检索。

```python
@dataclass

    source_session_id: str   # 来源会话 ID
    extracted_at: datetime = field(default_factory=datetime.now)

    # embedding: list[float]  # 在存储层处理，不在数据类中显式持有
```

## 第三部分学习：异步提取——从对话历史中抽取实体/属性

### 为什么要"异步"提取？

画像属性的提取过程需要调用 LLM 来理解对话内容、识别实体和归类属性——这是一个**高延迟、高成本**的操作。如果在用户对话过程中实时执行，会导致：

1. **响应延迟飙升**：用户发一条消息，Agent 要先花 2-3 秒提取画像再回复
2. **成本浪费**：不是每轮对话都包含有价值的画像信息，实时处理会产生大量无效调用
3. **影响用户体验**：用户能感知到明显的卡顿

### 生活类比：餐厅服务员 vs 后厨备料

**实时提取** = 服务员每上一道菜都现跑到菜市场买食材、现洗现切。顾客等到崩溃。

**异步提取** = 后厨在闲时（凌晨）统一采购、洗切备好。上菜时直接从备料区取，又快又稳。

### 异步提取的核心架构

```
实时对话链路（毫秒级响应）          后台提取链路（分钟级延迟）
┌──────────────────────┐          ┌──────────────────────┐
│ 用户消息 → Agent回复  │          │  对话历史消息队列     │
│                      │  写入→   │                      │
│ 不做任何画像提取      │─────────→│  session_id: s123    │
│ 只读取已有画像        │          │  messages: [...]     │
└──────────────────────┘          └──────────┬───────────┘
                                             ↓
                                  ┌──────────────────────┐
                                  │  定时 Job 轮询       │
                                  │  每 10 分钟检查一次   │
                                  │  低负载时段优先处理   │
                                  └──────────┬───────────┘
                                             ↓
                                  ┌──────────────────────┐
                                  │  LLM 实体/属性提取   │
                                  │                      │
                                  │  输入：对话历史       │
                                  │  输出：结构化属性     │
                                  │       + 偏好摘要     │
                                  └──────────┬───────────┘
                                             ↓
                                  ┌──────────────────────┐
                                  │  写入画像库           │
                                  │  结构化 → SQLite      │
                                  │  偏好摘要 → 向量库    │
                                  └──────────────────────┘
```

### 实体/属性提取的 Prompt 设计

提取画像的关键在于一个精心设计的提取 Prompt。它需要做到：

1. **只提取有信心的属性**——宁可不提取也不要提取错误信息
2. **输出结构化 JSON**——方便程序解析
3. **标注置信度**——后续可基于置信度过滤低质量结果

```python
EXTRACTION_PROMPT = """你是一个用户画像分析师。请从以下对话历史中提取用户的个人属性和偏好信息。

## 提取规则
1. 只提取有明确依据的信息，不要推测
2. 对每个提取的属性标注置信度（high/medium/low）
3. 如果对话中没有可提取的信息，返回空结果
4. 区分"用户自述的事实"和"用户讨论的话题"

## 对话历史
{conversation}

## 输出格式（严格 JSON）
{{
    "structured_attributes": [
        {{
            "attribute": "属性名（如 name/company/role/interest）",
            "value": "属性值",
            "confidence": "high/medium/low",
            "evidence": "原文依据"
        }}
    ],
    "preference_summary": "用一两句话概括用户在本次对话中展现的兴趣偏好和讨论话题",
    "has_extractable_info": true/false
}}"""
```

### 提取结果的合并策略

当同一属性被多次提取时（比如用户在不同会话中提到了自己的公司），需要一套合并策略：

| 冲突场景 | 合并策略 | 示例 |
| --- | --- | --- |
| 新旧值相同 | 提升置信度 | 两次都说"在字节工作" → confidence: high |
| 新旧值不同，新值置信度更高 | 用新值覆盖 | 旧: "在腾讯"(medium) → 新: "我已经跳槽到字节了"(high) |
| 新旧值不同，新值置信度更低 | 保留旧值，记录冲突 | 旧: "在字节"(high) → 新: 模型推测"可能在阿里"(low) |
| 列表型属性（如兴趣） | 去重追加 | interests: ["React"] + ["Vue", "React"] → ["React", "Vue"] |

## 第四部分学习：定时 Job 后台轮询机制

### 为什么需要定时 Job？

在第三部分我们明确了画像提取必须异步完成。那么"什么时候执行提取？"就成了一个工程问题。两种典型方案：

| 方案 | 触发方式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 事件驱动 | 会话结束时立即触发提取 | 实时性高 | 高峰期可能雪崩，提取任务互相抢资源 |
| 定时轮询 | 每隔 N 分钟扫描待处理队列 | 负载可控、可选在低峰期运行 | 有一定延迟，画像不是即时更新 |
| 混合策略（推荐） | 重要会话事件触发 + 定时兜底 | 兼顾实时性和稳定性 | 实现略复杂 |

### 生活类比：快递分拣中心

**事件驱动** = 每个快递一到就立刻分拣上车。双十一当天仓库直接瘫痪。

**定时轮询** = 每天固定 3 个时段（凌晨、中午、傍晚）统一分拣。量大了也能有序处理，只是时效稍慢。

**混合策略** = 加急件来了立刻处理，普通件攒一批定时处理。

### 定时 Job 的核心设计

```python
import asyncio
import time
from datetime import datetime, timedelta
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional


class JobStatus(Enum):
    PENDING = "pending"
    PROCESSING = "processing"
    COMPLETED = "completed"
    FAILED = "failed"


@dataclass
class ExtractionJob:
    """一次画像提取任务"""
    job_id: str
    user_id: str
    session_id: str
    messages: list[dict]           # 待提取的对话消息
    status: JobStatus = JobStatus.PENDING
    created_at: datetime = field(default_factory=datetime.now)
    retry_count: int = 0
    max_retries: int = 3
    error_message: Optional[str] = None


class ProfileExtractionScheduler:
    """画像提取定时调度器"""
    
    def __init__(
        self,
        poll_interval_seconds: int = 600,   # 默认 10 分钟轮询一次
        batch_size: int = 10,                # 每批处理 10 个任务
        low_load_hours: tuple = (1, 6),      # 凌晨 1-6 点为低负载时段
        max_concurrent: int = 3              # 最大并发提取数
    ):
        self.poll_interval = poll_interval_seconds
        self.batch_size = batch_size
        self.low_load_hours = low_load_hours
        self.max_concurrent = max_concurrent
        self.job_queue: list[ExtractionJob] = []
        self._running = False
    
    def submit_job(self, user_id: str, session_id: str, messages: list[dict]):
        """会话结束时提交提取任务到队列"""
        job = ExtractionJob(
            job_id=f"job_{user_id}_{session_id}_{int(time.time())}",
            user_id=user_id,
            session_id=session_id,
            messages=messages
        )
        self.job_queue.append(job)
        print(f"[Scheduler] 任务已入队: {job.job_id}, 当前队列长度: {len(self.job_queue)}")
        return job.job_id
    
    def _is_low_load_period(self) -> bool:
        """判断当前是否处于低负载时段"""
        current_hour = datetime.now().hour
        start, end = self.low_load_hours
        return start <= current_hour < end
    
    def _get_effective_batch_size(self) -> int:
        """低负载时段处理更多任务"""
        if self._is_low_load_period():
            return self.batch_size * 3   # 低峰期三倍吞吐
        return self.batch_size
    
    def _get_pending_jobs(self) -> list[ExtractionJob]:
        """获取待处理任务"""
        pending = [j for j in self.job_queue if j.status == JobStatus.PENDING]
        batch_size = self._get_effective_batch_size()
        return pending[:batch_size]
    
    async def _process_single_job(self, job: ExtractionJob, extractor):
        """处理单个提取任务"""
        job.status = JobStatus.PROCESSING
        try:
            result = await extractor.extract(job.user_id, job.messages)
            if result:
                await extractor.save_to_profile(job.user_id, result)
            job.status = JobStatus.COMPLETED
            print(f"[Scheduler] 任务完成: {job.job_id}")
        except Exception as e:
            job.retry_count += 1
            job.error_message = str(e)
            if job.retry_count >= job.max_retries:
                job.status = JobStatus.FAILED
                print(f"[Scheduler] 任务失败（已达最大重试）: {job.job_id}, 错误: {e}")
            else:
                job.status = JobStatus.PENDING   # 退回队列等待下次重试
                print(f"[Scheduler] 任务重试 {job.retry_count}/{job.max_retries}: {job.job_id}")
    
    async def _poll_and_process(self, extractor):
        """单次轮询：取任务 → 并发处理"""
        jobs = self._get_pending_jobs()
        if not jobs:
            return
        
        period = "低负载" if self._is_low_load_period() else "常规"
        print(f"[Scheduler] {period}时段，本轮处理 {len(jobs)} 个任务")
        
        semaphore = asyncio.Semaphore(self.max_concurrent)
        
        async def _limited_process(job):
            async with semaphore:
                await self._process_single_job(job, extractor)
        
        await asyncio.gather(*[_limited_process(j) for j in jobs])
    
    async def start(self, extractor):
        """启动定时轮询主循环"""
        self._running = True
        print(f"[Scheduler] 调度器启动，轮询间隔: {self.poll_interval}s")
        while self._running:
            await self._poll_and_process(extractor)
            await asyncio.sleep(self.poll_interval)
    
    def stop(self):
        """停止调度器"""
        self._running = False
        print("[Scheduler] 调度器已停止")
    
    def get_stats(self) -> dict:
        """获取调度器统计信息"""
        from collections import Counter
        status_counts = Counter(j.status.value for j in self.job_queue)
        return {
            "total_jobs": len(self.job_queue),
            "pending": status_counts.get("pending", 0),
            "processing": status_counts.get("processing", 0),
            "completed": status_counts.get("completed", 0),
            "failed": status_counts.get("failed", 0),
            "is_low_load": self._is_low_load_period()
        }
```

## 第五部分学习：跨会话画像存储与召回

### 存储方案选型

跨会话的关键在于**持久化**。对话可以关闭，但画像必须留存。

| 存储方案 | 适用场景 | 优点 | 缺点 |
| --- | --- | --- | --- |
| SQLite | 单机/原型开发 | 零配置、嵌入式、够用 | 不支持高并发写入 |
| PostgreSQL | 生产环境 | 强一致性、JSONB 灵活 | 需要运维 |
| Redis | 高频读取的热数据 | 毫秒级读取、TTL 自动过期 | 持久化需额外配置 |
| ChromaDB / Qdrant | 向量化偏好层 | 原生向量检索 | 不适合存结构化属性 |
| SQLite + ChromaDB（推荐） | 原型到中等规模 | 结构化 + 向量双层兼顾 | 大规模需替换 |

### 生活类比：图书馆的双系统

**结构化存储（SQLite）** = 图书馆的借阅卡目录柜。按作者名、ISBN 精确查找，一翻就到。

**向量存储（ChromaDB）** = 图书馆的"相关推荐"系统。你说"我想看关于时间旅行的小说"，它能语义匹配到《时间简史》旁边陈列的科幻小说——不是精确查找，而是模糊关联。

### 画像召回的注入策略

存储只是一半，另一半是**如何把画像信息注入到新会话中**。


### 注入时的 Token 预算管理

画像信息不能无限占用 Context Window，需要设定 Token 预算：

| 信息类型 | 建议 Token 预算 | 优先级 |
| --- | --- | --- |
| 用户基础身份（姓名/职业/公司） | 50-80 Tokens | P0（必须注入） |
| 沟通风格偏好 | 20-40 Tokens | P0（必须注入） |
| 相关话题偏好（Top-3） | 100-200 Tokens | P1（有则注入） |
| 历史行为统计 | 30-50 Tokens | P2（低优先级） |
| 合计 | 200-370 Tokens | — |

在 4K Token 窗口的模型中，画像信息约占 5-10% 的 Context——这是一个合理的比例。

```python
"""
用户画像提取与召回系统
依赖：pip install openai chromadb

# ==================== 第一层：数据模型 ====================

@dataclass
class UserProfile:
    """结构化用户画像"""
    user_id: str
    name: Optional[str] = None
    company: Optional[str] = None
    role: Optional[str] = None
    industry: Optional[str] = None
    interests: list[str] = field(default_factory=list)
    communication_style: Optional[str] = None
    total_sessions: int = 0
    created_at: str = field(default_factory=lambda: datetime.now().isoformat())
    updated_at: str = field(default_factory=lambda: datetime.now().isoformat())


@dataclass
class ExtractionResult:
    """LLM 提取结果"""
    structured_attributes: list[dict]    # [{"attribute": "name", "value": "张三", "confidence": "high"}]
    preference_summary: str              # "用户讨论了 React 性能优化..."
    has_extractable_info: bool


# ==================== 第二层：结构化存储（SQLite） ====================

class ProfileStore:
    """结构化画像存储"""
    
    def __init__(self, db_path: str = "user_profiles.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.row_factory = sqlite3.Row
        self._init_tables()
    
    def _init_tables(self):
        self.conn.executescript("""
            CREATE TABLE IF NOT EXISTS profiles (
                user_id TEXT PRIMARY KEY,
                name TEXT,
                company TEXT,
                role TEXT,
                industry TEXT,
                interests TEXT DEFAULT '[]',
                communication_style TEXT,
                total_sessions INTEGER DEFAULT 0,
                created_at TEXT,
                updated_at TEXT
            );
            
            CREATE TABLE IF NOT EXISTS extraction_log (
                log_id TEXT PRIMARY KEY,
                user_id TEXT,
                session_id TEXT,
                extracted_at TEXT,
                attributes_json TEXT,
                preference_summary TEXT
            );
        """)
        self.conn.commit()
    
    def get_profile(self, user_id: str) -> Optional[UserProfile]:
        """获取用户画像"""
        row = self.conn.execute(
            "SELECT * FROM profiles WHERE user_id = ?", (user_id,)
        ).fetchone()
        if not row:
            return None
        return UserProfile(
            user_id=row["user_id"],
            name=row["name"],
            company=row["company"],
            role=row["role"],
            industry=row["industry"],
            interests=json.loads(row["interests"]) if row["interests"] else [],
            communication_style=row["communication_style"],
            total_sessions=row["total_sessions"],
            created_at=row["created_at"],
            updated_at=row["updated_at"]
        )
    
    def upsert_profile(self, profile: UserProfile):
        """创建或更新画像"""
        profile.updated_at = datetime.now().isoformat()
        self.conn.execute("""
            INSERT INTO profiles (user_id, name, company, role, industry, 
                                  interests, communication_style, total_sessions,
                                  created_at, updated_at)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
            ON CONFLICT(user_id) DO UPDATE SET
                name = COALESCE(excluded.name, profiles.name),
                company = COALESCE(excluded.company, profiles.company),
                role = COALESCE(excluded.role, profiles.role),
                industry = COALESCE(excluded.industry, profiles.industry),
                interests = excluded.interests,
                communication_style = COALESCE(excluded.communication_style, profiles.communication_style),
                total_sessions = excluded.total_sessions,
                updated_at = excluded.updated_at
        """, (
            profile.user_id, profile.name, profile.company, profile.role,
            profile.industry, json.dumps(profile.interests, ensure_ascii=False),
            profile.communication_style, profile.total_sessions,
            profile.created_at, profile.updated_at
        ))
        self.conn.commit()
    
    def log_extraction(self, user_id: str, session_id: str, 
                       attributes: list[dict], summary: str):
        """记录提取日志"""
        self.conn.execute("""
            INSERT INTO extraction_log (log_id, user_id, session_id, 
                                        extracted_at, attributes_json, preference_summary)
            VALUES (?, ?, ?, ?, ?, ?)
        """, (
            str(uuid.uuid4()), user_id, session_id,
            datetime.now().isoformat(), json.dumps(attributes, ensure_ascii=False), summary
        ))
        self.conn.commit()
    
    def close(self):
        self.conn.close()


# ==================== 第三层：向量偏好存储（ChromaDB） ====================

class PreferenceVectorStore:
    """向量化偏好存储"""
    
    def __init__(self, persist_dir: str = "./chroma_profiles"):
        import chromadb
        self.client = chromadb.PersistentClient(path=persist_dir)
        self.collection = self.client.get_or_create_collection(
            name="user_preferences",
            metadata={"hnsw:space": "cosine"}
        )
    
    def add_preference(self, user_id: str, session_id: str, summary: str):
        """添加一条偏好摘要"""
        chunk_id = f"{user_id}_{session_id}_{uuid.uuid4().hex[:8]}"
        self.collection.add(
            ids=[chunk_id],
            documents=[summary],
            metadatas=[{
                "user_id": user_id,
                "session_id": session_id,
                "extracted_at": datetime.now().isoformat()
            }]
        )
    
    def search_preferences(self, user_id: str, query: str, top_k: int = 3) -> list[str]:
        """按语义检索用户偏好"""
        results = self.collection.query(
            query_texts=[query],
            n_results=top_k * 2,
            where={"user_id": user_id}
        )
        if not results["documents"] or not results["documents"][0]:
            return []
        return results["documents"][0][:top_k]
    
    def get_recent_preferences(self, user_id: str, limit: int = 3) -> list[str]:
        """获取用户最近的偏好摘要"""
        results = self.collection.get(
            where={"user_id": user_id},
            limit=limit
        )
        return results["documents"] if results["documents"] else []


# ==================== 第四层：LLM 画像提取器 ====================

class ProfileExtractor:
    """基于 LLM 的画像提取器"""
    
    EXTRACTION_PROMPT = """你是一个用户画像分析师。请从以下对话历史中提取用户的个人属性和偏好信息。

## 提取规则
1. 只提取有明确依据的信息，不要推测
2. 对每个提取的属性标注置信度（high/medium/low）
3. 如果对话中没有可提取的信息，返回 has_extractable_info: false
4. 区分"用户自述的事实"和"用户讨论的话题"

## 可提取的属性类型
- name: 用户姓名
- company: 所在公司
- role: 职业角色
- industry: 所在行业
- interest: 兴趣爱好或技术方向
- communication_style: 沟通风格偏好（简洁/详细/代码为主）

## 对话历史
{conversation}

## 输出格式（严格 JSON，不要输出其他内容）
{{
    "structured_attributes": [
        {{
            "attribute": "属性名",
            "value": "属性值",
            "confidence": "high/medium/low",
            "evidence": "原文依据"
        }}
    ],
    "preference_summary": "一两句话概括用户在本次对话中的兴趣和讨论话题",
    "has_extractable_info": true
}}"""
    
    def __init__(self, api_key: str, base_url: str = "https://api.openai.com/v1",
                 model: str = "gpt-4o-mini"):
        from openai import AsyncOpenAI
        self.client = AsyncOpenAI(api_key=api_key, base_url=base_url)
        self.model = model
    
    async def extract(self, user_id: str, messages: list[dict]) -> Optional[ExtractionResult]:
        """从对话历史中提取画像信息"""
        conversation = self._format_conversation(messages)
        prompt = self.EXTRACTION_PROMPT.format(conversation=conversation)
        
        try:
            response = await self.client.chat.completions.create(
                model=self.model,
                messages=[{"role": "user", "content": prompt}],
                temperature=0.1,
                response_format={"type": "json_object"}
            )
            raw = response.choices[0].message.content
            data = json.loads(raw)
            
            return ExtractionResult(
                structured_attributes=data.get("structured_attributes", []),
                preference_summary=data.get("preference_summary", ""),
                has_extractable_info=data.get("has_extractable_info", False)
            )
        except Exception as e:
            print(f"[Extractor] 提取失败: {e}")
            return None
    
    def _format_conversation(self, messages: list[dict]) -> str:
        lines = []
        for msg in messages:
            role = "用户" if msg["role"] == "user" else "助手"
            lines.append(f"{role}: {msg['content']}")
        return "\n".join(lines)
    
    async def save_to_profile(self, user_id: str, result: ExtractionResult):
        """将提取结果保存到画像（由调度器调用）"""
        pass  # 实际保存逻辑在 ProfileManager 中统一处理


# ==================== 第五层：画像管理器（统一门面） ====================

class ProfileManager:
    """画像管理器 —— 串联提取、存储、召回的统一入口"""
    
    def __init__(self, profile_store: ProfileStore,
                 vector_store: PreferenceVectorStore,
                 extractor: ProfileExtractor):
        self.profile_store = profile_store
        self.vector_store = vector_store
        self.extractor = extractor
    
    # ---------- 写入侧 ----------
    
    async def process_extraction(self, user_id: str, session_id: str,
                                  messages: list[dict]):
        """完整的提取 + 合并 + 存储流程"""
        result = await self.extractor.extract(user_id, messages)
        if not result or not result.has_extractable_info:
            print(f"[ProfileManager] 用户 {user_id} 本次对话无可提取信息")
            return
        
        existing = self.profile_store.get_profile(user_id)
        if not existing:
            existing = UserProfile(user_id=user_id)
        
        self._merge_attributes(existing, result.structured_attributes)
        existing.total_sessions += 1
        self.profile_store.upsert_profile(existing)
        
        if result.preference_summary:
            self.vector_store.add_preference(user_id, session_id, result.preference_summary)
        
        self.profile_store.log_extraction(
            user_id, session_id, result.structured_attributes, result.preference_summary
        )
        print(f"[ProfileManager] 用户 {user_id} 画像已更新")
    
    def _merge_attributes(self, profile: UserProfile, attributes: list[dict]):
        """将提取结果合并到现有画像"""
        confidence_rank = {"high": 3, "medium": 2, "low": 1}
        
        for attr in attributes:
            name = attr["attribute"]
            value = attr["value"]
            confidence = attr.get("confidence", "medium")
            
            if confidence == "low":
                continue
            
            if name == "name" and (not profile.name or confidence == "high"):
                profile.name = value
            elif name == "company" and (not profile.company or confidence == "high"):
                profile.company = value
            elif name == "role" and (not profile.role or confidence == "high"):
                profile.role = value
            elif name == "industry" and (not profile.industry or confidence == "high"):
                profile.industry = value
            elif name == "interest":
                if value not in profile.interests:
                    profile.interests.append(value)
            elif name == "communication_style":
                if not profile.communication_style or confidence == "high":
                    profile.communication_style = value
    
    # ---------- 读取侧 ----------
    
    def recall_profile(self, user_id: str, current_query: Optional[str] = None) -> dict:
        """召回画像信息，组装为可注入 System Prompt 的内容"""
        profile = self.profile_store.get_profile(user_id)
        
        if current_query:
            preferences = self.vector_store.search_preferences(user_id, current_query, top_k=3)
        else:
            preferences = self.vector_store.get_recent_preferences(user_id, limit=3)
        
        return {
            "profile": profile,
            "relevant_preferences": preferences
        }
    
    def build_system_prompt(self, user_id: str, 
                            current_query: Optional[str] = None,
                            base_prompt: str = "你是一个智能助手。") -> str:
        """构建包含画像信息的个性化 System Prompt"""
        recall = self.recall_profile(user_id, current_query)
        profile = recall["profile"]
        preferences = recall["relevant_preferences"]
        
        if not profile:
            return base_prompt
        
        parts = [base_prompt, "\n\n## 用户背景信息"]
        
        if profile.name:
            parts.append(f"- 用户姓名：{profile.name}")
        if profile.role:
            role_info = f"- 职业角色：{profile.role}"
            if profile.company:
                role_info += f"，就职于 {profile.company}"
            if profile.industry:
                role_info += f"（{profile.industry}行业）"
            parts.append(role_info)
        if profile.interests:
            parts.append(f"- 兴趣方向：{', '.join(profile.interests)}")
        if profile.communication_style:
            parts.append(f"- 沟通风格偏好：{profile.communication_style}")
        
        if preferences:
            parts.append("\n## 相关历史话题")
            for i, pref in enumerate(preferences, 1):
                parts.append(f"{i}. {pref}")
        
        parts.append("\n请根据以上用户背景调整回复风格和内容深度。")
        
        return "\n".join(parts)
```
