# 递归摘要压缩

## 学习

**学习目标**

- 极度精炼的信息压缩：掌握 Recursive Summarization 递归摘要算法，将超长文本逐层浓缩为高保真精华
- 提炼图谱层关联实体与属性：从摘要中自动抽取实体、关系、属性，构建知识图谱级别的结构化信息

**实战**

- 实现 Recursive Summarization 的迭代摘要法，有效把 5 万字小说精华提纯定格为 2,000 字无缺失矩阵
- 构建实体/关系抽取器，从每层摘要中同步提取图谱信息，确保压缩后的语义网络完整

**验收标准**

- 压缩后的摘要经人工盲评，关键事件/实体保留率 > 90%，压缩比 > 20:1

## 第一部分学习：为什么需要递归摘要

### 单次摘要的天花板

当你把一篇 5 万字的小说丢给 LLM 说"帮我总结一下"，会遇到三道硬墙：

**硬墙一：上下文窗口装不下**

即便是 128K tokens 的模型，5 万字中文约 7-8 万 tokens，一次性塞不进去。就算塞进去，"迷失在中间"效应（Lost in the Middle）会让模型严重忽略中间段落的关键信息。

**硬墙二：注意力分散导致遗漏**

大模型的注意力是有限资源。输入越长，每个 token 分到的注意力越少。实验表明，当输入超过 8K tokens 时，末尾生成的摘要平均遗漏 30%-50% 的关键事件。

**硬墙三：压缩比受限**

一次调用从 5 万字压到 2,000 字，压缩比 25:1。这要求模型在一次推理中完成极端信息取舍，效果远不如分步骤逐层蒸馏。

### 生活类比：酿酒蒸馏

**单次摘要** = 把一桶葡萄汁直接暴晒，想一步得到浓缩精华——大部分风味物质在高温下被破坏了。

**递归摘要** = 多次低温蒸馏。第一次蒸馏去掉大量水分，第二次进一步浓缩，第三次提取精华。每一步都温和精准，最终得到高纯度酒精，风味完整保留。

### 单次 vs 递归摘要效果对比

| 维度 | 单次摘要 | 递归摘要 |
| --- | --- | --- |
| 输入限制 | 受上下文窗口硬约束 | 分块处理，无长度上限 |
| 关键信息保留率 | 50%-70%（长文本时急剧下降） | 85%-95%（逐层蒸馏保真） |
| 压缩比极限 | 约 10:1 开始质量崩塌 | 可达 25:1+ 且保真 |
| 实体遗漏 | 严重（尤其中间段落） | 轻微（每层独立关注局部） |
| 计算成本 | 1 次 LLM 调用 | 多次调用（但每次输入短，总 token 可控） |
| 适用场景 | 短文档（< 5,000 字） | 长文档、书籍、报告（万字以上） |

## 第二部分学习：Recursive Summarization 算法原理

### 核心思想：分而治之，逐层蒸馏

递归摘要的本质是一个**信息蒸馏塔**——原料从底部灌入，每经过一层就浓缩一次，直到顶部输出高纯度精华。

```
                    ┌─────────────┐
                    │  最终摘要    │  ← 2,000 字（Layer 3）
                    │  (精华定格)  │
                    └──────┬──────┘
                           ↑ 合并 + 摘要
                ┌──────────┴──────────┐
                │                     │
          ┌─────┴─────┐         ┌─────┴─────┐
          │ 中间摘要 A │         │ 中间摘要 B │  ← ~5,000 字（Layer 2）
          └─────┬─────┘         └─────┬─────┘
                ↑ 各自摘要              ↑ 各自摘要
        ┌───────┴───────┐      ┌───────┴───────┐
        │               │      │               │
   ┌────┴────┐   ┌────┴────┐ ┌────┴────┐  ┌────┴────┐
   │ 块摘要1 │   │ 块摘要2 │ │ 块摘要3 │  │ 块摘要4 │  ← ~12,000 字（Layer 1）
   └────┬────┘   └────┬────┘ └────┬────┘  └────┬────┘
        ↑ 摘要         ↑          ↑             ↑
   ┌────┴────┐   ┌────┴────┐ ┌────┴────┐  ┌────┴────┐
   │ 原文块1 │   │ 原文块2 │ │ 原文块3 │  │ 原文块4 │  ← 50,000 字（Layer 0）
   └─────────┘   └─────────┘ └─────────┘  └─────────┘
```

### 两种主流架构

#### 架构一：Map-Reduce 式

先 Map（独立压缩每个块），再 Reduce（合并所有块摘要后再压缩）。最经典、最容易并行化。

```
Map 阶段：
  块1 → 摘要1    （可并行）
  块2 → 摘要2    （可并行）
  块3 → 摘要3    （可并行）
  块4 → 摘要4    （可并行）

Reduce 阶段：
  [摘要1 + 摘要2 + 摘要3 + 摘要4] → 合并摘要

  如果合并后仍超长 → 再次 Map-Reduce（递归）
```

#### 架构二：层级式（Hierarchical）

每次将相邻 N 个块分组，组内合并摘要，逐层上卷。更像一棵自底向上生长的树。

```
Layer 0:  [块1] [块2] [块3] [块4] [块5] [块6] [块7] [块8]
              ↓         ↓         ↓         ↓
Layer 1:  [摘要12]  [摘要34]  [摘要56]  [摘要78]
              ↓                   ↓
Layer 2:  [摘要1234]         [摘要5678]
                    ↓
Layer 3:       [最终摘要]
```

### 两种架构对比

| 维度 | Map-Reduce 式 | 层级式（Hierarchical） |
| --- | --- | --- |
| 并行度 | Map 阶段高度并行 | 同层内可并行，层间串行 |
| 上下文连贯性 | Reduce 阶段可看到全局 | 仅相邻块感知彼此，跨组信息需等上层融合 |
| 递归层数 | 通常 2-3 层 | log_N 层（块数越多层数越深） |
| 实现复杂度 | 简单 | 中等（需管理树状结构） |
| 信息损失模式 | Reduce 时易丢细节（信息被两次压缩） | 跨组边界信息易丢失 |
| 适用场景 | 段落间相对独立（论文、报告） | 叙事连贯性强（小说、纪实） |
| 类比 | 多人分头读章节，最后一人汇总 | 两两讨论合并笔记，逐轮汇聚 |

### 关键设计参数

| 参数 | 含义 | 推荐值 | 调参思路 |
| --- | --- | --- | --- |
| `chunk_size` | 每个原文块的大小（字符数） | 3,000-5,000 字 | 太小丢上下文，太大压缩难 |
| `chunk_overlap` | 相邻块重叠字符数 | chunk_size 的 10%-15% | 防止在叙事关键节点切断 |
| `compression_ratio` | 每层目标压缩比 | 3:1 ~ 5:1 | 单层压得太猛会丢信息 |
| `max_summary_length` | 每层摘要的最大长度 | chunk_size / compression_ratio | 确保 LLM 输出可控 |
| `merge_group_size` | 层级式每组合并的块数 | 2-4 | 太大则合并时输入过长 |
| `final_target_length` | 最终摘要目标长度 | 根据需求（如 2,000 字） | 反推需要递归几层 |

### 递归层数估算公式

```
原文总长 L，每层压缩比 r，最终目标长度 T

所需层数 n = ⌈log_r(L / T)⌉

示例：L = 50,000 字, r = 4, T = 2,000 字
n = ⌈log_4(50000 / 2000)⌉ = ⌈log_4(25)⌉ = ⌈2.32⌉ = 3 层
```

## 第三部分学习：实体/关系抽取——从文本到图谱

### 为什么摘要不够，还要抽实体？

摘要是"自然语言压缩"，人读得懂但机器不好用。如果你想让下游系统（知识图谱、问答引擎、推荐系统）直接使用压缩结果，就需要**结构化**的实体和关系。

### 生活类比：读书笔记 vs 人物关系图

**摘要** = 你写的读书笔记，一段段文字描述故事梗概。

**实体抽取** = 你在笔记旁边画的人物关系图——谁是谁、谁和谁有什么关系、在哪里发生了什么。笔记丢了还能从关系图还原大致情节，但关系图丢了，笔记里的"他""那个人"就没人能对上号。

### 三元组：知识图谱的最小单元

```
(实体A, 关系, 实体B)

例：
  (林黛玉, 爱慕, 贾宝玉)
  (贾宝玉, 居住于, 怡红院)
  (王熙凤, 管理, 荣国府)
  (刘姥姥, 三进, 大观园)
```

### 抽取的三个层次

| 层次 | 抽取内容 | 示例 | 用途 |
| --- | --- | --- | --- |
| 实体识别（NER） | 人名、地名、组织、时间 | "贾宝玉"→人物，"大观园"→地点 | 建立节点 |
| 关系抽取（RE） | 实体间的动作/关联 | (贾宝玉, 居住于, 大观园) | 建立边 |
| 属性抽取（AE） | 实体的固有特征 | 贾宝玉.性格=叛逆, 贾宝玉.别名=怡红公子 | 丰富节点信息 |

### 在递归摘要中嵌入实体抽取

关键洞察：**不是等摘要完成后再抽实体，而是在每一层摘要时同步抽取**。这样做有两个好处：

1. **实体累积**：底层抽到的细粒度实体不会因为上层压缩而丢失
2. **关系验证**：如果上层摘要提到了某个关系，但底层实体库里没有对应实体，说明上层摘要可能出现了幻觉

```
Layer 0（原文块）
  ├── 摘要 → Layer 1 摘要
  └── 实体抽取 → 实体集合 E0 = {贾宝玉, 林黛玉, 大观园, ...}

Layer 1（块摘要合并）
  ├── 摘要 → Layer 2 摘要
  └── 实体抽取 → 实体集合 E1 ⊆ E0（只保留高频/关键实体）

Layer 2（中间摘要合并）
  ├── 摘要 → 最终摘要
  └── 实体抽取 → 实体集合 E2 ⊆ E1（核心实体）

最终输出：
  - 最终摘要文本（2,000 字）
  - 全局实体图谱 E_all = E0 ∪ E1 ∪ E2（去重合并）
  - 每个实体标注"出现层级"和"频率"
```

## 第四部分学习：信息保真度评估方法

### 压缩了这么多，怎么知道没丢关键信息？

这是递归摘要的核心质量问题。我们需要一套系统性的评估方法，而不是拍脑袋说"看起来挺好的"。

### 生活类比：行李压缩收纳

你把一整柜衣服用真空压缩袋压成一小箱，到了目的地打开检查：

- **覆盖率检查**：每件衣服都在吗？（关键事件是否被保留）
- **完好度检查**：衣服有没有被压坏？（信息是否被扭曲）
- **可用性检查**：穿出去还能见人吗？（摘要读起来连贯吗）

### 四大评估维度

| 评估维度 | 度量方式 | 计算方法 | 及格线 |
| --- | --- | --- | --- |
| 关键事件保留率 | 人工标注 N 个关键事件，检查摘要命中几个 | 命中数 / N | ≥ 90% |
| 实体保留率 | 原文核心实体集 vs 摘要中提及的实体集 | \|E_摘要 ∩ E_原文\| / \|E_原文\| | ≥ 90% |
| 事实一致性（Faithfulness） | 摘要中每句话是否可从原文推导，无幻觉 | LLM-as-Judge 或 NLI 模型判断 | ≥ 95% |
| 连贯性（Coherence） | 摘要是否逻辑通顺、无断层 | 人工评分（1-5 分）或 LLM 评分 | ≥ 4.0/5.0 |

### 自动化评估方法

#### 方法一：ROUGE 系列（词粒度）

基于 n-gram 重叠计算摘要与参考文本的相似度。简单快速但粒度粗。

```
ROUGE-1：单词级别重叠
ROUGE-2：二元组级别重叠
ROUGE-L：最长公共子序列
```

#### 方法二：BERTScore（语义粒度）

用 BERT 编码原文和摘要，计算 token 级别的语义相似度。能捕捉同义替换。

#### 方法三：LLM-as-Judge（推理粒度）

让另一个 LLM 作为裁判，逐条判断摘要的事实一致性。最灵活但成本最高。

#### 方法四：实体覆盖率（图谱粒度）

比较原文实体集和摘要实体集的交集比例。最直观，可自动化。

### 评估方法对比

| 方法 | 评估维度 | 优点 | 缺点 | 成本 |
| --- | --- | --- | --- | --- |
| ROUGE | 词面重叠 | 快速、无需模型 | 无法识别同义改写 | 极低 |
| BERTScore | 语义相似度 | 识别同义表达 | 对事实错误不敏感 | 中 |
| LLM-as-Judge | 事实一致性 | 最接近人类判断 | 贵、自身也可能出错 | 高 |
| 实体覆盖率 | 结构完整性 | 客观、可追溯 | 只看实体不看叙事 | 低 |

**最佳实践：组合使用**——用实体覆盖率做快速初筛，用 LLM-as-Judge 做精细验证。

## 第五部分学习：压缩比与保留率的 Trade-off

### 核心矛盾：压得越狠，丢得越多

这是信息论的基本约束——有损压缩必然伴随信息损失。我们的目标不是零损失（那就是原文），而是在可接受的损失范围内，最大化压缩比。

### 生活类比：浓缩果汁

- **1:1 还原果汁**：口感完美，但体积毫无缩减
- **3:1 浓缩**：口感略有差异，但携带方便
- **20:1 超浓缩**：需要精心工艺才能保留风味，否则就是糖水
- **100:1 果汁粉**：已经不是果汁了，只剩关键化学成分

### 压缩比 vs 保留率曲线

```
保留率
100%|●
    |  ●
 90%|    ●──────────── 目标线（≥90%）
    |      ●
 80%|        ●
    |          ●
 70%|            ●
    |              ●
 60%|                ●
    |                  ●
 50%|                    ●
    +--+--+--+--+--+--+--+--→ 压缩比
    1:1 5:1 10:1 15:1 20:1 25:1 30:1

    典型曲线：保留率随压缩比增大而下降，
    但递归摘要通过逐层蒸馏可以把曲线右移（同等压缩比下保留率更高）
```

### 影响 Trade-off 的关键因素

| 因素 | 对保留率的影响 | 对压缩比的影响 | 调优建议 |
| --- | --- | --- | --- |
| 每层压缩比 | 单层压得猛 → 保留率暴跌 | 层数减少 → 总计算量少 | 每层 3:1~5:1，温和蒸馏 |
| 递归层数 | 层数多 → 信息经过更多次筛选 | 层数多 → 最终可达更高压缩比 | 按公式估算，通常 2-4 层 |
| Chunk 重叠率 | 重叠大 → 边界信息保全 | 重叠大 → 冗余增加 | 10%-15% |
| 实体注入 | 强制保留关键实体 → 保留率提升 | 实体占用摘要篇幅 | 核心实体必须注入 Prompt |
| 模型能力 | 强模型压缩更精准 | 与压缩比无关 | 生产用 GPT-4 级别 |

### 实用策略：保留率优先的动态压缩

核心思路：不追求固定压缩比，而是**让每层压缩到"保留率不低于阈值"为止**。

```
while 当前文本长度 > 目标长度:
    对每个块做摘要，目标压缩比 = 4:1
    评估保留率
    if 保留率 < 90%:
        降低压缩比为 3:1，重新摘要
    合并所有摘要，进入下一层
```

## 第六部分学习：实战代码——递归摘要引擎 + 实体抽取器

### 完整代码：RecursiveSummarizer

```python
"""
递归摘要引擎 + 实体抽取器
将超长文本逐层压缩为高保真摘要，同步提取图谱级实体与关系
"""

import os
import json
import math
import hashlib
from typing import Optional
from dataclasses import dataclass, field
from openai import OpenAI


# ============================================================
# 数据结构
# ============================================================

@dataclass
class Entity:
    name: str
    entity_type: str          # 人物 / 地点 / 组织 / 事件 / 物品
    attributes: dict = field(default_factory=dict)
    frequency: int = 1
    first_seen_layer: int = 0

    @property
    def id(self) -> str:
        return hashlib.md5(f"{self.name}:{self.entity_type}".encode()).hexdigest()[:8]


@dataclass
class Relation:
    source: str       # 实体名
    relation: str     # 关系类型
    target: str       # 实体名
    evidence: str     # 原文依据片段
    layer: int = 0


@dataclass
class LayerResult:
    layer: int
    chunks: list[str]
    summaries: list[str]
    merged_text: str
    entities: list[Entity]
    relations: list[Relation]
    compression_ratio: float
    char_count_before: int
    char_count_after: int


@dataclass
class RecursiveSummaryResult:
    final_summary: str
    layers: list[LayerResult]
    all_entities: dict[str, Entity]   # name -> Entity（去重合并）
    all_relations: list[Relation]
    total_compression_ratio: float
    original_length: int
    final_length: int


# ============================================================
# 文本分块器
# ============================================================

def split_into_chunks(
    text: str,
    chunk_size: int = 4000,
    overlap: int = 400
) -> list[str]:
    """将长文本按字符数切分为带重叠的块"""
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))

        if end < len(text):
            for sep in ["。", "！", "？", ".", "\n\n", "\n"]:
                last_sep = text[start:end].rfind(sep)
                if last_sep > chunk_size * 0.5:
                    end = start + last_sep + len(sep)
                    break

        chunks.append(text[start:end].strip())
        start = end - overlap if end < len(text) else end

    return [c for c in chunks if c]


# ============================================================
# LLM 调用封装
# ============================================================

class LLMClient:
    def __init__(self, model: str = "gpt-4o-mini", base_url: Optional[str] = None):
        self.client = OpenAI(
            api_key=os.getenv("OPENAI_API_KEY", "your-api-key"),
            base_url=base_url or os.getenv("OPENAI_BASE_URL"),
        )
        self.model = model

    def call(self, system_prompt: str, user_prompt: str, max_tokens: int = 4096) -> str:
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt},
            ],
            max_tokens=max_tokens,
            temperature=0.3,
        )
        return response.choices[0].message.content.strip()


# ============================================================
# 摘要生成器
# ============================================================

SUMMARIZE_SYSTEM_PROMPT = """你是一位专业的信息压缩专家。你的任务是将给定文本精炼为高保真摘要。

要求：
1. 保留所有关键事件、人物、地点、时间节点
2. 保留因果链条和转折点
3. 删除冗余描写、重复信息、过渡性语句
4. 摘要长度控制在原文的 {ratio_hint} 左右
5. 不得添加原文没有的信息（禁止幻觉）
6. 使用简洁精准的语言"""

SUMMARIZE_USER_PROMPT = """请将以下文本压缩为高保真摘要，目标长度约 {target_length} 字：

---原文开始---
{text}
---原文结束---

输出要求：直接输出摘要文本，不要任何前缀说明。"""


def summarize_chunk(
    llm: LLMClient,
    text: str,
    target_length: int,
    ratio_hint: str = "1/4"
) -> str:
    """对单个文本块做摘要"""
    system = SUMMARIZE_SYSTEM_PROMPT.format(ratio_hint=ratio_hint)
    user = SUMMARIZE_USER_PROMPT.format(
        target_length=target_length,
        text=text,
    )
    return llm.call(system, user, max_tokens=target_length * 2)

# 实体/关系抽取器
# ============================================================

EXTRACT_SYSTEM_PROMPT = """你是一位知识图谱构建专家。从给定文本中抽取实体和关系。

输出严格的 JSON 格式：
{
  "entities": [
    {"name": "实体名", "type": "人物|地点|组织|事件|物品|时间", "attributes": {"key": "value"}}
  ],
  "relations": [
    {"source": "实体A", "relation": "关系描述", "target": "实体B", "evidence": "原文依据片段"}
  ]
}

规则：
1. 只抽取明确出现在文本中的实体和关系
2. 实体名使用文中最常见的称呼
3. 每条关系必须附带原文依据
4. 不要推测或捏造"""

EXTRACT_USER_PROMPT = """请从以下文本中抽取实体和关系：

{text}

输出 JSON："""


def extract_entities_and_relations(
    llm: LLMClient,
    text: str,
    layer: int
) -> tuple[list[Entity], list[Relation]]:
    """从文本中抽取实体和关系"""
    result = llm.call(EXTRACT_SYSTEM_PROMPT, EXTRACT_USER_PROMPT.format(text=text))

    try:
        cleaned = result.strip()
        if cleaned.startswith("```"):
            cleaned = cleaned.split("\n", 1)[1].rsplit("```", 1)[0]
        data = json.loads(cleaned)
    except json.JSONDecodeError:
        return [], []

    entities = [
        Entity(
            name=e["name"],
            entity_type=e.get("type", "未知"),
            attributes=e.get("attributes", {}),
            first_seen_layer=layer,
        )
        for e in data.get("entities", [])
    ]

    relations = [
        Relation(
            source=r["source"],
            relation=r["relation"],
            target=r["target"],
            evidence=r.get("evidence", ""),
            layer=layer,
        )
        for r in data.get("relations", [])
    ]

    return entities, relations

# 实体合并器

def merge_entities(
    existing: dict[str, Entity],
    new_entities: list[Entity]
) -> dict[str, Entity]:
    """合并新抽取的实体到全局实体字典，去重并累加频率"""
    for entity in new_entities:
        if entity.name in existing:
            existing[entity.name].frequency += 1
            existing[entity.name].attributes.update(entity.attributes)
        else:
            existing[entity.name] = entity
    return existing

# 核心引擎：递归摘要器

class RecursiveSummarizer:
    def __init__(
        self,
        llm: LLMClient,
        chunk_size: int = 4000,
        chunk_overlap: int = 400,
        compression_ratio_per_layer: float = 4.0,
        final_target_length: int = 2000,
        extract_entities: bool = True,
    ):
        self.llm = llm
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap
        self.compression_ratio = compression_ratio_per_layer
        self.final_target_length = final_target_length
        self.extract_entities = extract_entities

    def estimate_layers(self, text_length: int) -> int:
        """估算所需递归层数"""
        if text_length <= self.final_target_length:
            return 0
        return math.ceil(
            math.log(text_length / self.final_target_length)
            / math.log(self.compression_ratio)
        )

    def _summarize_layer(
        self,
        text: str,
        layer: int,
        target_length: int,
    ) -> LayerResult:
        """执行单层摘要 + 实体抽取"""
        chunks = split_into_chunks(text, self.chunk_size, self.chunk_overlap)
        char_count_before = len(text)

        chunk_target = max(
            int(len(text) / len(chunks) / self.compression_ratio),
            200,
        )

        summaries = []
        layer_entities = []
        layer_relations = []

        for i, chunk in enumerate(chunks):
            print(f"  [Layer {layer}] 处理块 {i+1}/{len(chunks)}，"
                  f"长度 {len(chunk)} 字 → 目标 {chunk_target} 字")

            summary = summarize_chunk(
                self.llm, chunk, chunk_target,
                ratio_hint=f"1/{int(self.compression_ratio)}"
            )
            summaries.append(summary)

            if self.extract_entities:
                entities, relations = extract_entities_and_relations(
                    self.llm, chunk, layer
                )
                layer_entities.extend(entities)
                layer_relations.extend(relations)

        merged_text = "\n\n".join(summaries)
        char_count_after = len(merged_text)

        return LayerResult(
            layer=layer,
            chunks=chunks,
            summaries=summaries,
            merged_text=merged_text,
            entities=layer_entities,
            relations=layer_relations,
            compression_ratio=char_count_before / max(char_count_after, 1),
            char_count_before=char_count_before,
            char_count_after=char_count_after,
        )

    def summarize(self, text: str) -> RecursiveSummaryResult:
        """执行完整的递归摘要流程"""
        original_length = len(text)
        estimated_layers = self.estimate_layers(original_length)

        print(f"原文长度: {original_length} 字")
        print(f"目标长度: {self.final_target_length} 字")
        print(f"估计层数: {estimated_layers}")
        print(f"目标压缩比: {original_length / self.final_target_length:.1f}:1")
        print("=" * 60)

        all_entities: dict[str, Entity] = {}
        all_relations: list[Relation] = []
        layers: list[LayerResult] = []
        current_text = text

        layer_num = 0
        while len(current_text) > self.final_target_length * 1.5:
            print(f"\n--- Layer {layer_num} ---")
            print(f"当前文本长度: {len(current_text)} 字")

            layer_result = self._summarize_layer(
                current_text,
                layer=layer_num,
                target_length=self.final_target_length,
            )
            layers.append(layer_result)

            all_entities = merge_entities(all_entities, layer_result.entities)
            all_relations.extend(layer_result.relations)

            print(f"  压缩后: {layer_result.char_count_after} 字 "
                  f"(本层压缩比 {layer_result.compression_ratio:.1f}:1)")

            current_text = layer_result.merged_text
            layer_num += 1

            if layer_num > estimated_layers + 2:
                print("⚠ 超过预估层数，强制停止")
                break

        if len(current_text) > self.final_target_length * 1.2:
            print(f"\n--- 最终精炼 ---")
            final_summary = summarize_chunk(
                self.llm, current_text, self.final_target_length,
                ratio_hint=f"约{self.final_target_length}字"
            )
        else:
            final_summary = current_text

        final_length = len(final_summary)
        total_ratio = original_length / max(final_length, 1)

        print(f"\n{'=' * 60}")
        print(f"最终摘要长度: {final_length} 字")
        print(f"总压缩比: {total_ratio:.1f}:1")
        print(f"提取实体数: {len(all_entities)}")
        print(f"提取关系数: {len(all_relations)}")

        return RecursiveSummaryResult(
            final_summary=final_summary,
            layers=layers,
            all_entities=all_entities,
            all_relations=all_relations,
            total_compression_ratio=total_ratio,
            original_length=original_length,
            final_length=final_length,
        )

# 保真度评估器
# ============================================================

class FidelityEvaluator:
    """评估递归摘要的信息保真度"""

    def __init__(self, llm: LLMClient):
        self.llm = llm

    def evaluate_entity_retention(
        self,
        original_entities: dict[str, Entity],
        summary: str,
    ) -> dict:
        """实体保留率：检查原文核心实体在摘要中出现的比例"""
        core_entities = [
            e for e in original_entities.values() if e.frequency >= 2
        ]

        if not core_entities:
            core_entities = list(original_entities.values())

        retained = [e for e in core_entities if e.name in summary]
        missed = [e for e in core_entities if e.name not in summary]

        retention_rate = len(retained) / max(len(core_entities), 1)

        return {
            "total_core_entities": len(core_entities),
            "retained_count": len(retained),
            "missed_count": len(missed),
            "retention_rate": retention_rate,
            "retained_names": [e.name for e in retained],
            "missed_names": [e.name for e in missed],
            "pass": retention_rate >= 0.9,
        }

    def evaluate_key_events(
        self,
        key_events: list[str],
        summary: str,
    ) -> dict:
        """关键事件保留率：用 LLM 判断每个关键事件是否在摘要中有体现"""
        results = []
        for event in key_events:
            prompt = f"""判断以下摘要是否包含了这个关键事件的信息。

关键事件：{event}

摘要：
{summary}

请只回答 YES 或 NO，不要解释。"""

            answer = self.llm.call(
                "你是一个严格的事实核查员。", prompt, max_tokens=10
            )
            results.append({
                "event": event,
                "retained": "YES" in answer.upper(),
            })

        retained_count = sum(1 for r in results if r["retained"])
        retention_rate = retained_count / max(len(key_events), 1)

        return {
            "total_events": len(key_events),
            "retained_count": retained_count,
            "retention_rate": retention_rate,
            "details": results,
            "pass": retention_rate >= 0.9,
        }

    def evaluate_faithfulness(self, original_text: str, summary: str) -> dict:
        """事实一致性：检查摘要是否引入了原文中不存在的信息"""
        prompt = f"""请逐句检查以下摘要，判断每句话是否能从原文中找到依据。

原文（节选前 3000 字）：
{original_text[:3000]}

摘要：
{summary}

请以 JSON 格式输出：
{{"faithful_sentences": 数量, "unfaithful_sentences": 数量, "total_sentences": 数量, "issues": ["问题描述"]}}"""

        result = self.llm.call("你是严格的事实一致性审核员。", prompt)
        try:
            cleaned = result.strip()
            if cleaned.startswith("```"):
                cleaned = cleaned.split("\n", 1)[1].rsplit("```", 1)[0]
            data = json.loads(cleaned)
            total = data.get("total_sentences", 1)
            faithful = data.get("faithful_sentences", total)
            rate = faithful / max(total, 1)
            return {
                "faithfulness_rate": rate,
                "total_sentences": total,
                "faithful_sentences": faithful,
                "issues": data.get("issues", []),
                "pass": rate >= 0.95,
            }
        except json.JSONDecodeError:
            return {"faithfulness_rate": None, "error": "解析失败", "pass": False}

    def full_evaluation(
        self,
        result: RecursiveSummaryResult,
        key_events: list[str],
        original_text: str,
    ) -> dict:
        """运行完整评估套件"""
        print("\n" + "=" * 60)
        print("信息保真度评估报告")
        print("=" * 60)

        entity_eval = self.evaluate_entity_retention(
            result.all_entities, result.final_summary
        )
        print(f"\n【实体保留率】{entity_eval['retention_rate']:.1%}"
              f" ({'✅ 通过' if entity_eval['pass'] else '❌ 未通过'})")
        if entity_eval["missed_names"]:
            print(f"  遗漏实体: {', '.join(entity_eval['missed_names'])}")

        event_eval = self.evaluate_key_events(key_events, result.final_summary)
        print(f"\n【关键事件保留率】{event_eval['retention_rate']:.1%}"
              f" ({'✅ 通过' if event_eval['pass'] else '❌ 未通过'})")
        for detail in event_eval["details"]:
            status = "✅" if detail["retained"] else "❌"
            print(f"  {status} {detail['event']}")

        faith_eval = self.evaluate_faithfulness(original_text, result.final_summary)
        if faith_eval.get("faithfulness_rate") is not None:
            print(f"\n【事实一致性】{faith_eval['faithfulness_rate']:.1%}"
                  f" ({'✅ 通过' if faith_eval['pass'] else '❌ 未通过'})")

        compression_pass = result.total_compression_ratio >= 20
        print(f"\n【压缩比】{result.total_compression_ratio:.1f}:1"
              f" ({'✅ ≥20:1' if compression_pass else '❌ <20:1'})")

        all_pass = (
            entity_eval["pass"]
            and event_eval["pass"]
            and faith_eval.get("pass", False)
            and compression_pass
        )

        print(f"\n{'=' * 60}")
        print(f"综合结果: {'✅ 全部通过' if all_pass else '❌ 存在未达标项'}")
        print(f"{'=' * 60}")

        return {
            "entity_retention": entity_eval,
            "key_event_retention": event_eval,
            "faithfulness": faith_eval,
            "compression_ratio": result.total_compression_ratio,
            "compression_pass": compression_pass,
            "all_pass": all_pass,
        }

### 关键设计决策说明

| 设计决策 | 选择 | 原因 |
| --- | --- | --- |
| 分块时尊重句子边界 | 在 `.`、`。`、`\n` 处断开 | 避免切断语义完整的句子 |
| 每层同步抽取实体 | Map 阶段同时做摘要和抽取 | 底层细粒度实体不会丢失 |
| 实体按频率去重 | 高频实体 = 核心实体 | 自动识别主要角色/地点 |
| 动态层数而非固定层数 | `while len > target * 1.5` | 适应不同长度的输入文本 |
| LLM-as-Judge 评估事件保留 | 逐事件独立判断 | 比 ROUGE 更准确地捕捉语义级别的保留 |

## 第七部分学习：进阶优化策略

### 策略一：实体锚定 Prompt

在每层摘要时，将上一层提取的核心实体列表注入 Prompt，强制 LLM 在摘要中保留这些实体：

```python
ENTITY_ANCHORED_PROMPT = """请将以下文本压缩为摘要。

【必须保留的核心实体】（这些实体不可遗漏）：
{entity_list}

---原文---
{text}

要求：摘要中必须提及以上所有核心实体，目标长度约 {target_length} 字。"""
```

### 策略二：重叠区域去重

相邻块的重叠部分可能导致同一事件被两个块各摘要一次。解决方案：

```python
def deduplicate_overlapping_summaries(summaries: list[str], llm: LLMClient) -> str:
    """合并摘要时去除重叠区域产生的重复信息"""
    merged = "\n\n".join(summaries)
    prompt = f"""以下是多段摘要的拼接，可能存在重复描述同一事件的情况。
请合并去重，保留所有不重复的信息，删除重复部分。

{merged}

输出合并后的无重复文本："""
    return llm.call("你是文本去重专家。", prompt)
```

### 策略三：分层压缩比自适应

对信息密度高的块（如情节高潮）使用更低的压缩比，对信息密度低的块（如环境描写）使用更高的压缩比：

| 块类型 | 信息密度判断 | 压缩比 |
| --- | --- | --- |
| 事件密集段（对话、冲突） | 实体数/字数 > 阈值 | 2:1 ~ 3:1（温和压缩） |
| 过渡段（场景描写、心理活动） | 实体数/字数 < 阈值 | 5:1 ~ 8:1（激进压缩） |
| 重复段（回忆、重述） | 与已有摘要高度相似 | 10:1+（极度压缩） |

### 策略四：摘要质量自检回路

每层摘要生成后，自动进行质量自检，不合格则重新生成：
