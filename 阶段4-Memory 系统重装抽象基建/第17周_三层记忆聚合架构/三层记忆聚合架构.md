# 三层记忆聚合架构

## 学习

**学习目标**

- 定义高度封装的面向接口调用的顶级聚合器设计模式：把短期闪存、中期摘要缓存与长期深层匹配三层记忆抽象为统一接口，一次调用即可获得完整上下文拼盘
- 掌握 Token 预算在三层间的动态分配策略，确保有限窗口下信息密度最大化

> 本周从"记忆怎么存"升级到"记忆怎么聚合调度"——核心命题不再是单一记忆层的读写，而是如何让三层记忆像金字塔一样协作，自动拼装出最适合当前推理的上下文。

**实战**

- 实现统一记忆聚合器 `MemoryAggregator`：一调即走，从短期闪存、中期摘要缓存与长期深层匹配中快速合成拼盘返回给推理大模型
- 50 轮跳跃对话压测，验证冷知识保持能力

**验收标准**

- 实现 50 轮前后文强跳跃对话压测下，不丢失第 2 轮及第 6 轮交代的"我是做什么的""上次报了什么指标"等冷知识能力，做到连续一贯表现

## 第一部分学习：金字塔三层记忆架构

### 为什么需要三层记忆？

回忆前面几周我们构建的 Agent——它要么把所有历史消息塞进 Prompt（Token 炸裂），要么只保留最近 N 轮（丢失关键信息）。这就像一个只有"短期记忆"的人，别人十分钟前交代的事就忘了。

现实中人类的记忆是分层协作的：

- **感觉记忆 / 工作记忆**（秒级）：你正在读的这句话
- **短期记忆**（分钟~小时）：今天开会讨论了什么
- **长期记忆**（天~年）：你的名字、职业、去年项目的关键指标

AI Agent 同样需要这种分层——不同"新鲜度"和"重要度"的信息用不同的存储策略和召回策略来管理。

### 生活类比：厨房的三层储物体系

想象你是一个厨师在做饭：

- **灶台台面（短期闪存）**：正在切的菜、刚倒好的调料——伸手就能拿到，但空间极小，只放当前在用的东西
- **冰箱（中期摘要缓存）**：今天买的食材、昨天剩的汤——需要开门拿，稍慢，但容量大得多，保鲜期几天
- **地下室储藏间（长期深层匹配）**：腊肉、陈年老酒、去年收的干货——需要下楼翻找，最慢，但能存几年的东西

做菜时你不可能把地下室所有东西都搬到灶台上（Token 窗口装不下），而是：**先看灶台有没有 → 再开冰箱找 → 最后才下地下室翻**。这就是三层记忆的协作逻辑。

### 三层架构定义

```
                    ┌────────────────────────────┐
                    │     推理大模型 (LLM)        │
                    │   接收拼装好的上下文拼盘     │
                    └────────────┬───────────────┘
                                 ↑
                    ┌────────────┴───────────────┐
                    │    MemoryAggregator 聚合器   │
                    │    统一接口 · 一调即走        │
                    └──┬──────────┬──────────┬───┘
                       ↓          ↓          ↓
              ┌──────────┐ ┌───────────┐ ┌───────────┐
              │ 短期闪存  │ │ 中期摘要   │ │ 长期深层   │
              │ (Flash)  │ │ (Summary) │ │ (Deep)    │
              │          │ │           │ │           │
              │ 最近K轮  │ │ 滑窗摘要   │ │ 向量/关键  │
              │ 原始消息  │ │ 压缩缓存   │ │ 词语义匹配 │
              └──────────┘ └───────────┘ └───────────┘
                  快·小         中·中          慢·大
```

### 三层记忆详细对比

| 维度 | 短期闪存 (Flash) | 中期摘要缓存 (Summary) | 长期深层匹配 (Deep) |
| --- | --- | --- | --- |
| 存什么 | 最近 K 轮的原始 user/assistant 消息 | 滑窗周期内对话的压缩摘要 | 所有历史对话的向量嵌入 + 关键事实条目 |
| 时间跨度 | 当前会话最近 3~5 轮 | 当前会话的第 1 轮到倒数第 K 轮 | 跨会话、跨天、长期 |
| 存储介质 | 内存列表（list） | 内存字典 / Redis | 向量数据库（Chroma / Qdrant） |
| 读取速度 | 极快（O(1) 直接切片） | 快（O(1) 字典查找） | 较慢（向量 ANN 检索） |
| Token 密度 | 低（原始消息冗余多） | 高（摘要压缩，信息密度大） | 中（检索结果需二次筛选） |
| 典型大小 | 500~2000 tokens | 300~800 tokens | 检索 top-k 条，200~1000 tokens |
| 类比 | 灶台台面 | 冰箱 | 地下室储藏间 |

### 为什么不直接全用长期记忆？

一个常见的疑问：既然向量数据库什么都能存、什么都能搜，为什么还要分三层？

| 方案 | 优点 | 致命缺点 |
| --- | --- | --- |
| 只用短期（最近 K 轮） | 零延迟，格式完整 | 早期信息全部丢失 |
| 只用长期（全量向量检索） | 理论上不丢信息 | 最近几轮的上下文连贯性被检索打散；延迟高；嵌入模型对"刚说的上一句"反而匹配不准 |
| 三层分级协作 | 近期信息连贯 + 中期压缩高效 + 远期精准召回 | 需要聚合器设计（本周要解决的问题） |

**核心结论**：三层分级的本质是**用空间换时间、用压缩换容量、用检索换覆盖**——三者缺一不可。

## 第二部分学习：统一聚合器接口设计

### 面向接口编程的必要性

三层记忆各自有不同的存储介质、不同的读写 API、不同的返回格式。如果调用方（推理链路）直接跟三层实现耦合，代码会变成这样：

```python
# ❌ 调用方直接耦合三层实现
def build_context(query, chat_history, vector_db, summary_cache):
    recent = chat_history[-6:]  # 短期
    summary = summary_cache.get("current_session")  # 中期
    docs = vector_db.similarity_search(query, k=3)  # 长期
    # 手动拼接，硬编码 Token 分配...
    context = format_recent(recent) + "\n" + summary + "\n" + format_docs(docs)
    return context
```

问题跟前面几周讲的一样——**改一层的实现就要改调用方，测试困难，复用为零**。

### 生活类比：USB 接口标准

USB 口就是一个"统一聚合接口"：

- 你不关心接的是鼠标、键盘还是硬盘（短期/中期/长期的具体实现）
- 你只需要知道"插进去就能用"（调用 `retrieve()` 就能拿到结果）
- 设备厂商可以各自升级内部芯片，只要 USB 接口不变，你的电脑不用改任何东西

我们要做的就是给三层记忆定义这样一个"USB 接口"。

### 策略模式（Strategy Pattern）在记忆聚合中的应用

策略模式的核心思想：**把算法族封装成可互换的策略对象，让调用方不依赖具体实现**。

```
              ┌──────────────────────────┐
              │   MemoryAggregator       │
              │                          │
              │   layers: List[Layer]    │
              │   budget: TokenBudget    │
              │                          │
              │   + retrieve(query)      │ ← 调用方只认识这一个方法
              └────────────┬─────────────┘
                           │ 内部遍历
              ┌────────────┼────────────┐
              ↓            ↓            ↓
     ┌──────────────┐ ┌──────────┐ ┌──────────┐
     │ FlashMemory  │ │ Summary  │ │ DeepMatch│
     │ (实现 Layer) │ │ (实现    │ │ (实现    │
     │              │ │  Layer)  │ │  Layer)  │
     │ + retrieve() │ │+retrieve│ │+retrieve│
     └──────────────┘ └──────────┘ └──────────┘
```

| 设计模式概念 | 在记忆聚合中的角色 | 好处 |
| --- | --- | --- |
| 抽象接口 `MemoryLayer` | 定义 `retrieve(query, budget) → str` 方法签名 | 统一调用协议 |
| 具体策略 `FlashMemory` | 实现短期闪存的 retrieve 逻辑 | 可独立替换、测试 |
| 具体策略 `SummaryCache` | 实现中期摘要的 retrieve 逻辑 | 可独立替换、测试 |
| 具体策略 `DeepMatch` | 实现长期匹配的 retrieve 逻辑 | 可独立替换、测试 |
| 上下文 `MemoryAggregator` | 持有三层策略，按预算分配依次调用 | 调用方一行代码搞定 |

### 接口定义代码

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass


@dataclass
class RetrievalResult:
    """单层记忆的检索结果"""
    text: str
    token_count: int
    source: str        # "flash" / "summary" / "deep"
    confidence: float  # 0.0 ~ 1.0


class MemoryLayer(ABC):
    """记忆层统一接口——所有层必须实现这个协议"""

    @abstractmethod
    def retrieve(self, query: str, max_tokens: int) -> RetrievalResult:
        """根据查询和 Token 预算返回该层的记忆片段"""
        ...

    @abstractmethod
    def store(self, messages: list[dict]) -> None:
        """将新消息写入该层存储"""
        ...

    @property
    @abstractmethod
    def layer_name(self) -> str:
        """层名称标识"""
        ...
```

这段代码定义了"USB 接口规范"——任何新的记忆层只要实现 `MemoryLayer`，就能无缝接入聚合器。

## 第三部分学习：三层协作流程——查询时如何合成上下文

### 协作流程全景

当用户发来一条新消息时，聚合器需要在毫秒级完成以下流程：

```
用户消息: "我上次报的那个DAU指标是多少来着？"
                        ↓
          ┌─────────────────────────────────┐
          │  Step 1: 短期闪存快照            │
          │  从最近 K 轮原始消息中查找        │
          │  → 命中？直接返回原始上下文        │
          │  → 未命中？继续往下                │
          └──────────────┬──────────────────┘
                         ↓
          ┌─────────────────────────────────┐
          │  Step 2: 中期摘要缓存命中        │
          │  在滑窗摘要中搜索关键词/语义      │
          │  → 命中？返回摘要片段             │
          │  → 未命中？继续往下                │
          └──────────────┬──────────────────┘
                         ↓
          ┌─────────────────────────────────┐
          │  Step 3: 长期深层匹配            │
          │  用向量检索 + 关键词过滤          │
          │  → 返回 top-k 最相关的历史片段    │
          └──────────────┬──────────────────┘
                         ↓
          ┌─────────────────────────────────┐
          │  Step 4: Token 预算裁剪 & 拼装   │
          │  按优先级分配预算，截断超限部分    │
          │  → 输出最终上下文拼盘              │
          └─────────────────────────────────┘
```

### 生活类比：医院看病的分诊流程

你去医院看病，医生不会一上来就给你做全身 CT：

1. **先问最近情况**（短期闪存）："这两天哪里不舒服？" ——如果症状明显，直接开药
2. **再看病历摘要**（中期摘要）："上个月体检报告说你血压偏高" ——如果相关，参考这个背景
3. **最后调历史档案**（长期深层）："三年前你做过心脏支架手术" ——只在需要时才去翻老档案

**越近的信息越先用，越远的信息越精准检索**——这就是三层协作的核心逻辑。

### 协作策略对比

| 策略 | 描述 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 瀑布式（Waterfall） | 从短期到长期依次检索，命中就停 | 最快，短期命中时零额外开销 | 可能遗漏长期重要信息 |
| 并行全查（Parallel） | 三层同时检索，合并结果 | 信息最全面 | 延迟取决于最慢的层（向量检索） |
| 预算驱动（Budget-First） | 先分配预算，三层各自在预算内检索 | Token 控制最精确 | 实现复杂度较高 |
| 混合模式（本周采用） | 短期直取 + 中期/长期并行 + 预算裁剪 | 兼顾速度、覆盖、Token 控制 | 需要仔细调参 |

本周我们采用**混合模式**：短期闪存直接切片（零延迟），中期和长期并行检索（用线程池），最后按预算统一裁剪拼装。

### 混合模式的详细时序

```
时间轴 →

T0: 收到用户消息
    │
T1: ├── 短期闪存：直接切片最近 K 轮 ──→ flash_result (< 1ms)
    │
    ├── 中期摘要：关键词/语义搜索 ────→ summary_result (~5ms)
    │        ↕ 并行
    ├── 长期深层：向量 ANN 检索 ──────→ deep_result (~20ms)
    │
T2: 三层结果全部到齐
    │
T3: Token 预算裁剪器：
    │   flash_result  → 分配 40% 预算 → 截断/保留
    │   summary_result → 分配 30% 预算 → 截断/保留
    │   deep_result   → 分配 30% 预算 → 截断/保留
    │
T4: 拼装上下文拼盘 → 返回给 LLM
```

## 第四部分学习：Token 预算在三层间的分配策略

### 为什么需要预算分配？

LLM 的上下文窗口是有限的（4K / 8K / 128K），而系统 Prompt、工具描述、用户当前消息都要占位。留给"历史记忆上下文"的空间往往只有总窗口的 40%~60%。

如果三层记忆各自"想拿多少拿多少"，结果一定是 Token 超限 → 被截断 → 关键信息丢失。所以需要一个**预算分配器**统一调度。

### 生活类比：家庭月度预算

你家每月收入 10000 元（Token 总预算），要分给：

- **房租/房贷（系统 Prompt + 工具描述）**：固定支出，不可压缩 → 3000 元
- **伙食费（短期闪存）**：最基本的日常需求，优先保障 → 2800 元
- **教育/医疗（中期摘要）**：重要但可调节 → 2100 元
- **储蓄/投资（长期深层）**：有余力才分配 → 2100 元

如果某个月医疗突发大笔支出（中期摘要里有很关键的信息需要更多 Token），就从储蓄里挪一部分过来——这就是**动态预算调整**。

### 预算分配策略对比

| 策略 | 分配方式 | 适用场景 | 缺点 |
| --- | --- | --- | --- |
| 固定比例 | Flash 40% / Summary 30% / Deep 30% | 对话模式稳定、不需动态调整 | 无法应对突发长上下文 |
| 优先级贪心 | Flash 先吃饱 → Summary 吃剩余 → Deep 吃残余 | 强调近期上下文连贯性 | 长期信息容易被饿死 |
| 置信度加权 | 按每层返回结果的 confidence 动态分配 | 信息密度最大化 | 需要每层返回置信度分数 |
| 混合自适应（本周采用） | 基础比例 + 置信度微调 + 保底最小预算 | 生产环境推荐 | 参数需要根据业务调优 |

### 混合自适应预算分配算法

```python
@dataclass
class TokenBudget:
    """Token 预算管理器"""
    total: int          # 总预算（记忆部分可用的 Token 数）
    flash_ratio: float = 0.40   # 短期基础比例
    summary_ratio: float = 0.30 # 中期基础比例
    deep_ratio: float = 0.30    # 长期基础比例
    min_per_layer: int = 100    # 每层保底最小 Token

    def allocate(
        self, flash_conf: float, summary_conf: float, deep_conf: float
    ) -> dict[str, int]:
        """根据基础比例 + 置信度微调计算各层实际预算

        Args:
            flash_conf: 短期层置信度 (0~1)
            summary_conf: 中期层置信度 (0~1)
            deep_conf: 长期层置信度 (0~1)

        Returns:
            {"flash": N, "summary": N, "deep": N}
        """
        weights = {
            "flash": self.flash_ratio * (0.5 + 0.5 * flash_conf),
            "summary": self.summary_ratio * (0.5 + 0.5 * summary_conf),
            "deep": self.deep_ratio * (0.5 + 0.5 * deep_conf),
        }

        total_weight = sum(weights.values())
        allocated = {}
        for layer, w in weights.items():
            raw = int(self.total * w / total_weight)
            allocated[layer] = max(raw, self.min_per_layer)

        overflow = sum(allocated.values()) - self.total
        if overflow > 0:
            largest = max(allocated, key=allocated.get)
            allocated[largest] -= overflow

        return allocated
```

### 预算分配示例演算

假设总预算 2000 tokens，三层置信度分别为 flash=0.9, summary=0.6, deep=0.8：

```
基础权重:
  flash   = 0.40 × (0.5 + 0.5×0.9) = 0.40 × 0.95 = 0.380
  summary = 0.30 × (0.5 + 0.5×0.6) = 0.30 × 0.80 = 0.240
  deep    = 0.30 × (0.5 + 0.5×0.8) = 0.30 × 0.90 = 0.270

归一化:
  总权重 = 0.380 + 0.240 + 0.270 = 0.890
  flash   = 2000 × 0.380/0.890 ≈ 854 tokens
  summary = 2000 × 0.240/0.890 ≈ 539 tokens
  deep    = 2000 × 0.270/0.890 ≈ 607 tokens

合计 = 2000 tokens ✓
```

短期闪存置信度高（最近几轮确实相关），所以自动多分了预算；中期摘要置信度低（摘要里没找到太相关的内容），预算相应缩减。

## 第五部分学习：冷知识保持能力

### 什么是"冷知识"？

在多轮对话中，有些信息**只在早期提过一次，之后长时间不被提及，但突然在很后面的某轮又需要用到**。这类信息我们称为"冷知识"。

典型例子：

- 第 2 轮用户说"我是做数据分析的"→ 第 35 轮问"你能根据我的工作背景推荐学习路径吗？"
- 第 6 轮用户说"上次 DAU 报了 12 万"→ 第 42 轮问"帮我跟上次的指标做个对比"

### 生活类比：老朋友的记忆

你跟一个朋友半年前吃饭时提到"最近换了新工作，在XX公司做产品经理"。半年后你再聊天，如果他还记得这件事，你会觉得"这朋友靠谱，有心"；如果他问"你现在在哪上班来着？"你会觉得对方根本不在乎你。

对 AI Agent 来说，**冷知识保持能力直接决定用户信任度和体验**。

### 冷知识丢失的三大原因

| 原因 | 机制 | 发生在哪一层 |
| --- | --- | --- |
| 滑窗截断 | 只保留最近 K 轮，早期信息直接丢弃 | 短期闪存 |
| 摘要压缩损失 | 摘要时把"不重要"的细节丢掉了（但其实后面会用到） | 中期摘要 |
| 检索不命中 | 用户的新提问和早期信息的语义距离太远，向量检索召回不到 | 长期深层匹配 |

### 冷知识保持的解决方案

#### 方案一：关键事实提取 + 永久存储

在每轮对话后，用轻量 LLM 提取"关键事实"（Key Facts），单独存入一个不会被摘要压缩或滑窗淘汰的永久存储。

```python
FACT_EXTRACTION_PROMPT = """从以下对话中提取用户透露的关键个人事实。
只提取具体的、可能在未来对话中有用的信息。
```

```python
def count_tokens(text: str, model: str = "gpt-4") -> int:
    """使用 tiktoken 计算 Token 数"""
    try:
        enc = tiktoken.encoding_for_model(model)
    except KeyError:
        enc = tiktoken.get_encoding("cl100k_base")
    return len(enc.encode(text))


def truncate_to_tokens(text: str, max_tokens: int, model: str = "gpt-4") -> str:
    """将文本截断到指定 Token 数"""
    try:
        enc = tiktoken.encoding_for_model(model)
    except KeyError:
        enc = tiktoken.get_encoding("cl100k_base")
    tokens = enc.encode(text)
    if len(tokens) <= max_tokens:
        return text
    return enc.decode(tokens[:max_tokens])

@dataclass
class RetrievalResult:
    """单层记忆的检索结果"""
    text: str
    token_count: int
    source: str
    confidence: float


@dataclass
class KeyFact:
    """关键事实条目"""
    category: str
    content: str
    round_number: int
    timestamp: float = field(default_factory=time.time)

    @property
    def uid(self) -> str:
        raw = f"{self.category}:{self.content}"
        return hashlib.md5(raw.encode()).hexdigest()[:12]

# Token 预算管理器
# ============================================================

@dataclass
class TokenBudget:
    total: int
    flash_ratio: float = 0.40
    summary_ratio: float = 0.30
    deep_ratio: float = 0.30
    min_per_layer: int = 100

    def allocate(
        self,
        flash_conf: float = 0.5,
        summary_conf: float = 0.5,
        deep_conf: float = 0.5,
    ) -> dict[str, int]:
        weights = {
            "flash": self.flash_ratio * (0.5 + 0.5 * flash_conf),
            "summary": self.summary_ratio * (0.5 + 0.5 * summary_conf),
            "deep": self.deep_ratio * (0.5 + 0.5 * deep_conf),
        }
        total_weight = sum(weights.values()) or 1.0
        allocated = {}
        for layer, w in weights.items():
            raw = int(self.total * w / total_weight)
            allocated[layer] = max(raw, self.min_per_layer)

        overflow = sum(allocated.values()) - self.total
        if overflow > 0:
            largest = max(allocated, key=allocated.get)
            allocated[largest] = max(allocated[largest] - overflow, self.min_per_layer)

        return allocated

class MemoryLayer(ABC):

    @abstractmethod
    def retrieve(self, query: str, max_tokens: int) -> RetrievalResult:
        ...

    @abstractmethod
    def store(self, messages: list[dict], round_number: int) -> None:
        ...

    @property
    @abstractmethod
    def layer_name(self) -> str:
        ...


# ============================================================
# 第一层：短期闪存 (FlashMemory)
# ============================================================

class FlashMemory(MemoryLayer):
    """保留最近 K 轮的原始消息"""

    def __init__(self, max_rounds: int = 5):
        self._max_rounds = max_rounds
        self._history: list[dict] = []

    @property
    def layer_name(self) -> str:
        return "flash"

    def store(self, messages: list[dict], round_number: int) -> None:
        for msg in messages:
            self._history.append({**msg, "_round": round_number})
        cutoff = max(0, round_number - self._max_rounds + 1)
        self._history = [m for m in self._history if m["_round"] >= cutoff]

    def retrieve(self, query: str, max_tokens: int) -> RetrievalResult:
        lines = []
        for msg in self._history:
            role = msg.get("role", "unknown")
            content = msg.get("content", "")
            lines.append(f"[{role}] {content}")
        raw_text = "\n".join(lines)
        truncated = truncate_to_tokens(raw_text, max_tokens)
        token_count = count_tokens(truncated)
        confidence = min(1.0, token_count / max(max_tokens, 1))
        return RetrievalResult(
            text=truncated,
            token_count=token_count,
            source="flash",
            confidence=confidence,
        )

# 第二层：中期摘要缓存 (SummaryCache)
# ============================================================

class SummaryCache(MemoryLayer):
    """对滑出短期窗口的对话做增量摘要压缩"""

    def __init__(self, summarize_fn=None):
        self._summaries: list[dict] = []
        self._buffer: list[dict] = []
        self._summarize_fn = summarize_fn or self._default_summarize

    @property
    def layer_name(self) -> str:
        return "summary"

    @staticmethod
    def _default_summarize(messages: list[dict]) -> str:
        lines = []
        for msg in messages:
            lines.append(f"{msg.get('role','?')}: {msg.get('content','')}")
        combined = "\n".join(lines)
        if len(combined) > 300:
            combined = combined[:300] + "..."
        return f"[摘要] {combined}"

    def store(self, messages: list[dict], round_number: int) -> None:
        for msg in messages:
            self._buffer.append({**msg, "_round": round_number})
        if len(self._buffer) >= 6:
            to_summarize = self._buffer[:6]
            self._buffer = self._buffer[6:]
            summary_text = self._summarize_fn(to_summarize)
            rounds = sorted(set(m["_round"] for m in to_summarize))
            self._summaries.append({
                "text": summary_text,
                "rounds": rounds,
                "token_count": count_tokens(summary_text),
            })

    def retrieve(self, query: str, max_tokens: int) -> RetrievalResult:
        if not self._summaries:
            return RetrievalResult(text="", token_count=0, source="summary", confidence=0.0)

        query_lower = query.lower()
        scored = []
        for s in self._summaries:
            text_lower = s["text"].lower()
            keyword_hits = sum(1 for word in query_lower.split() if word in text_lower)
            score = keyword_hits / max(len(query_lower.split()), 1)
            scored.append((score, s))

        scored.sort(key=lambda x: x[0], reverse=True)

        result_parts = []
        used_tokens = 0
        for score, s in scored:
            if used_tokens + s["token_count"] > max_tokens:
                remaining = max_tokens - used_tokens
                if remaining > 20:
                    result_parts.append(truncate_to_tokens(s["text"], remaining))
                break
            result_parts.append(s["text"])
            used_tokens += s["token_count"]

        full_text = "\n".join(result_parts)
        token_count = count_tokens(full_text)
        confidence = scored[0][0] if scored else 0.0
        return RetrievalResult(
            text=full_text,
            token_count=token_count,
            source="summary",
            confidence=min(1.0, confidence),
        )

# 第三层：长期深层匹配 (DeepMatchMemory)
# ============================================================

class DeepMatchMemory(MemoryLayer):
    """基于向量 + 关键词的长期记忆检索

    生产环境应接入 Chroma / Qdrant 等向量数据库。
    此处为教学演示，使用简化的 TF 余弦相似度实现。
    """

    def __init__(self, top_k: int = 5):
        self._top_k = top_k
        self._documents: list[dict] = []

    @property
    def layer_name(self) -> str:
        return "deep"

    def store(self, messages: list[dict], round_number: int) -> None:
        for msg in messages:
            content = msg.get("content", "")
            if not content.strip():
                continue
            self._documents.append({
                "content": content,
                "role": msg.get("role", "unknown"),
                "round": round_number,
                "tokens": content.lower().split(),
            })

    def _simple_similarity(self, query_tokens: list[str], doc_tokens: list[str]) -> float:
        if not query_tokens or not doc_tokens:
            return 0.0
        q_set = set(query_tokens)
        d_set = set(doc_tokens)
        intersection = q_set & d_set
        if not intersection:
            return 0.0
        return len(intersection) / (len(q_set) ** 0.5 * len(d_set) ** 0.5)

    def retrieve(self, query: str, max_tokens: int) -> RetrievalResult:
        if not self._documents:
            return RetrievalResult(text="", token_count=0, source="deep", confidence=0.0)

        query_tokens = query.lower().split()
        scored = []
        for doc in self._documents:
            sim = self._simple_similarity(query_tokens, doc["tokens"])
            scored.append((sim, doc))

        scored.sort(key=lambda x: x[0], reverse=True)
        top = scored[: self._top_k]

        result_parts = []
        used_tokens = 0
        max_conf = 0.0
        for sim, doc in top:
            if sim <= 0:
                continue
            line = f"[Round {doc['round']}][{doc['role']}] {doc['content']}"
            line_tokens = count_tokens(line)
            if used_tokens + line_tokens > max_tokens:
                remaining = max_tokens - used_tokens
                if remaining > 20:
                    result_parts.append(truncate_to_tokens(line, remaining))
                break
            result_parts.append(line)
            used_tokens += line_tokens
            max_conf = max(max_conf, sim)

        full_text = "\n".join(result_parts)
        return RetrievalResult(
            text=full_text,
            token_count=count_tokens(full_text),
            source="deep",
            confidence=min(1.0, max_conf),
        )

# 关键事实存储 (FactStore) —— 冷知识保护
# ============================================================

class FactStore:
    """永久存储关键事实，不受滑窗和摘要压缩影响"""

    def __init__(self):
        self._facts: dict[str, KeyFact] = {}

    def add(self, fact: KeyFact) -> None:
        self._facts[fact.uid] = fact

    def add_batch(self, facts: list[KeyFact]) -> None:
        for f in facts:
            self.add(f)

    def search(self, query: str, max_results: int = 5) -> list[KeyFact]:
        query_lower = query.lower()
        scored = []
        for fact in self._facts.values():
            text = f"{fact.category} {fact.content}".lower()
            hits = sum(1 for w in query_lower.split() if w in text)
            if hits > 0:
                scored.append((hits, fact))
        scored.sort(key=lambda x: x[0], reverse=True)
        return [f for _, f in scored[:max_results]]

    def format_facts(self, facts: list[KeyFact], max_tokens: int) -> str:
        lines = []
        for f in facts:
            lines.append(f"[{f.category}](第{f.round_number}轮提及) {f.content}")
        text = "\n".join(lines)
        return truncate_to_tokens(text, max_tokens)

    @property
    def all_facts(self) -> list[KeyFact]:
        return list(self._facts.values())

# 关键事实提取器（模拟 LLM 提取，生产环境替换为真实 LLM 调用）
# ============================================================

class FactExtractor:
    """从对话中提取关键事实

    生产环境下这里调用 LLM；教学演示使用规则匹配。
    """

    PATTERNS = [
        (r"我是做(.+?)的", "职业"),
        (r"我在(.+?)工作", "工作单位"),
        (r"我叫(.+?)(?:[，。,.]|$)", "姓名"),
        (r"(?:DAU|dau|日活).*?(\d[\d,.万亿]+)", "指标-DAU"),
        (r"(?:MAU|mau|月活).*?(\d[\d,.万亿]+)", "指标-MAU"),
        (r"(?:收入|营收|GMV|gmv).*?(\d[\d,.万亿]+)", "指标-收入"),
        (r"(?:转化率|CVR|cvr).*?(\d[\d,.%]+)", "指标-转化率"),
        (r"上次.*?报.*?(\d[\d,.万亿%]+)", "历史指标"),
        (r"(?:用|使用|偏好|喜欢).*?(Python|Java|Go|Rust|SQL|Excel)", "工具偏好"),
        (r"(?:目标|KPI|OKR).*?是(.+?)(?:[，。,.]|$)", "目标"),
    ]

    def extract(self, messages: list[dict], round_number: int) -> list[KeyFact]:
        facts = []
        for msg in messages:
            if msg.get("role") != "user":
                continue
            content = msg.get("content", "")
            for pattern, category in self.PATTERNS:
                match = re.search(pattern, content)
                if match:
                    value = match.group(1).strip()
                    facts.append(KeyFact(
                        category=category,
                        content=f"{content[:60]}... → {value}" if len(content) > 60 else f"{content} → {value}",
                        round_number=round_number,
                    ))
        return facts

# 核心：MemoryAggregator 统一聚合器
# ============================================================

class MemoryAggregator:
    """三层记忆统一聚合器 —— 一调即走

    Usage:
        aggregator = MemoryAggregator(token_budget=2000)

        # 每轮对话后存储
        aggregator.store(messages=[
            {"role": "user", "content": "..."},
            {"role": "assistant", "content": "..."},
        ], round_number=1)

        # 需要上下文时一调即走
        context = aggregator.retrieve("用户的新问题")
    """

    def __init__(
        self,
        token_budget: int = 2000,
        flash_rounds: int = 5,
        deep_top_k: int = 5,
        fact_budget_ratio: float = 0.15,
        summarize_fn=None,
    ):
        self._budget = TokenBudget(total=token_budget)
        self._fact_budget_ratio = fact_budget_ratio
        self._flash = FlashMemory(max_rounds=flash_rounds)
        self._summary = SummaryCache(summarize_fn=summarize_fn)
        self._deep = DeepMatchMemory(top_k=deep_top_k)
        self._fact_store = FactStore()
        self._fact_extractor = FactExtractor()
        self._layers: list[MemoryLayer] = [self._flash, self._summary, self._deep]
        self._round_counter = 0
        self._lock = threading.Lock()

    @property
    def fact_store(self) -> FactStore:
        return self._fact_store

    def store(self, messages: list[dict], round_number: Optional[int] = None) -> None:
        """存储一轮对话到三层记忆 + 提取关键事实"""
        with self._lock:
            if round_number is None:
                self._round_counter += 1
                round_number = self._round_counter
            else:
                self._round_counter = max(self._round_counter, round_number)

            for layer in self._layers:
                layer.store(messages, round_number)

            facts = self._fact_extractor.extract(messages, round_number)
            self._fact_store.add_batch(facts)

    def retrieve(self, query: str) -> str:
        """一调即走：从三层记忆 + 事实存储中合成上下文拼盘"""

        fact_budget = int(self._budget.total * self._fact_budget_ratio)
        layer_budget_total = self._budget.total - fact_budget

        temp_budget = TokenBudget(
            total=layer_budget_total,
            flash_ratio=self._budget.flash_ratio,
            summary_ratio=self._budget.summary_ratio,
            deep_ratio=self._budget.deep_ratio,
            min_per_layer=self._budget.min_per_layer,
        )

        # 并行检索中期和长期，短期直取
        results: dict[str, RetrievalResult] = {}

        flash_result = self._flash.retrieve(query, max_tokens=int(layer_budget_total * 0.5))
        results["flash"] = flash_result

        with ThreadPoolExecutor(max_workers=2) as pool:
            futures = {
                pool.submit(
                    self._summary.retrieve, query, int(layer_budget_total * 0.4)
                ): "summary",
                pool.submit(
                    self._deep.retrieve, query, int(layer_budget_total * 0.4)
                ): "deep",
            }
            for future in as_completed(futures):
                layer_name = futures[future]
                results[layer_name] = future.result()

        allocation = temp_budget.allocate(
            flash_conf=results["flash"].confidence,
            summary_conf=results["summary"].confidence,
            deep_conf=results["deep"].confidence,
        )

        parts = []

        flash_text = truncate_to_tokens(results["flash"].text, allocation["flash"])
        if flash_text.strip():
            parts.append(f"=== 近期对话 ===\n{flash_text}")

        summary_text = truncate_to_tokens(results["summary"].text, allocation["summary"])
        if summary_text.strip():
            parts.append(f"=== 历史摘要 ===\n{summary_text}")

        deep_text = truncate_to_tokens(results["deep"].text, allocation["deep"])
        if deep_text.strip():
            parts.append(f"=== 深层记忆匹配 ===\n{deep_text}")

        relevant_facts = self._fact_store.search(query, max_results=10)
        if relevant_facts:
            fact_text = self._fact_store.format_facts(relevant_facts, fact_budget)
            if fact_text.strip():
                parts.append(f"=== 关键事实 ===\n{fact_text}")

        context = "\n\n".join(parts)

        total_tokens = count_tokens(context)
        if total_tokens > self._budget.total:
            context = truncate_to_tokens(context, self._budget.total)

        return context

    def get_stats(self) -> dict:
        """返回聚合器当前状态统计"""
        return {
            "total_rounds": self._round_counter,
            "flash_messages": len(self._flash._history),
            "summary_segments": len(self._summary._summaries),
            "deep_documents": len(self._deep._documents),
            "key_facts": len(self._fact_store.all_facts),
            "token_budget": self._budget.total,
        }

def run_stress_test():
    """执行 50 轮跳跃对话压测"""
    print("=" * 70)
    print("三层记忆聚合架构 —— 50 轮跳跃对话压测")
    print("=" * 70)

    aggregator = MemoryAggregator(
        token_budget=2000,
        flash_rounds=5,
        deep_top_k=5,
    )

    conversations = build_stress_test_conversations()
    test_points = {35, 42, 49, 50}
    test_results = {}

    for conv in conversations:
        rnd = conv["round"]
        msgs = conv["messages"]
        aggregator.store(msgs, round_number=rnd)

        if rnd in test_points:
            user_query = msgs[0]["content"]
            context = aggregator.retrieve(user_query)

            print(f"\n{'─' * 60}")
            print(f"📍 第 {rnd} 轮 - 冷知识回忆测试")
            print(f"用户问：{user_query}")
            print(f"{'─' * 60}")

            checks = {}
            if rnd == 35:
                checks["职业-数据分析"] = "数据分析" in context
                checks["工具-Python"] = "Python" in context or "python" in context
            elif rnd == 42:
                checks["指标-DAU-12万"] = "12万" in context or "12" in context
            elif rnd == 49:
                checks["职业-数据分析"] = "数据分析" in context
                checks["工具-Python"] = "Python" in context or "python" in context
                checks["指标-DAU"] = "DAU" in context or "dau" in context or "12万" in context or "12" in context
            elif rnd == 50:
                checks["职业信息"] = "数据分析" in context
                checks["DAU指标"] = "DAU" in context or "12万" in context or "12" in context

            for check_name, passed in checks.items():
                status = "✅ 通过" if passed else "❌ 未通过"
                print(f"  {check_name}: {status}")

            test_results[rnd] = checks

            print(f"\n聚合上下文预览（前 500 字符）：")
            print(context[:500])
            if len(context) > 500:
                print("...")

    # 汇总报告
    print(f"\n\n{'=' * 70}")
    print("压测汇总报告")
    print(f"{'=' * 70}")

    stats = aggregator.get_stats()
    print(f"\n聚合器状态：")
    print(f"  总轮次: {stats['total_rounds']}")
    print(f"  短期闪存消息数: {stats['flash_messages']}")
    print(f"  中期摘要段数: {stats['summary_segments']}")
    print(f"  长期深层文档数: {stats['deep_documents']}")
    print(f"  关键事实数: {stats['key_facts']}")
    print(f"  Token 预算: {stats['token_budget']}")

    print(f"\n关键事实库内容：")
    for fact in aggregator.fact_store.all_facts:
        print(f"  [{fact.category}] 第{fact.round_number}轮: {fact.content}")

    total_checks = 0
    passed_checks = 0
    print(f"\n各测试点详细结果：")
    print(f"{'─' * 50}")
    for rnd, checks in sorted(test_results.items()):
        for name, passed in checks.items():
            total_checks += 1
            if passed:
                passed_checks += 1
            status = "✅" if passed else "❌"
            print(f"  第{rnd}轮 | {name}: {status}")

    pass_rate = (passed_checks / total_checks * 100) if total_checks > 0 else 0
    print(f"\n{'─' * 50}")
    print(f"总计: {passed_checks}/{total_checks} 项通过 ({pass_rate:.0f}%)")

    if pass_rate == 100:
        print(f"\n🎉 压测通过！冷知识保持能力验证成功")
    elif pass_rate >= 80:
        print(f"\n⚠️  压测基本通过，部分冷知识召回需要优化")
    else:
        print(f"\n❌ 压测未通过，冷知识保持能力不足，需要排查")

    print(f"{'=' * 70}")
    return pass_rate


if __name__ == "__main__":
```

### 代码架构要点总结

| 组件 | 职责 | 设计模式 |
| --- | --- | --- |
| `MemoryLayer` | 抽象接口，定义 retrieve/store 协议 | 策略模式（Strategy） |
| `FlashMemory` | 短期闪存，最近 K 轮原始消息 | 具体策略 |
| `SummaryCache` | 中期摘要，滑窗压缩缓存 | 具体策略 |
| `DeepMatchMemory` | 长期深层，向量/关键词匹配 | 具体策略 |
| `FactStore` | 关键事实永久存储，冷知识保护 | 仓储模式（Repository） |
| `FactExtractor` | 从对话中提取关键事实 | 提取器（Extractor） |
| `TokenBudget` | Token 预算管理与动态分配 | 值对象（Value Object） |
| `MemoryAggregator` | 统一聚合器，一调即走 | 外观模式（Facade）+ 策略模式 |

### 交付一：三层记忆聚合架构验证

**架构清单**：

```
MemoryAggregator（统一聚合器）
    │
    ├── FlashMemory ──── 最近 5 轮原始消息
    │                     存储: 内存 list
    │                     检索: O(1) 切片
    │
    ├── SummaryCache ─── 滑窗摘要压缩
    │                     存储: 内存 dict
    │                     检索: 关键词匹配 + 按相关度排序
    │
    ├── DeepMatchMemory ─ 全量历史向量/关键词检索
    │                     存储: 内存文档列表（生产替换为向量DB）
    │                     检索: TF 余弦相似度 top-k
    │
    ├── FactStore ─────── 关键事实永久存储
    │                     存储: 内存字典（去重）
    │                     检索: 关键词命中排序
    │
    └── TokenBudget ───── 混合自适应预算分配
                          策略: 基础比例 × 置信度加权
```

**核心检查清单**：

| 检查项 | 要求 | 状态 |
| --- | --- | --- |
| 面向接口设计 | 三层均实现 `MemoryLayer` 抽象接口 | ✅ |
| 策略模式 | 各层可独立替换，不影响聚合器 | ✅ |
| 一调即走 | `aggregator.retrieve(query)` 一行代码获取完整上下文 | ✅ |
| Token 预算控制 | 输出上下文不超过预算总量 | ✅ |
| 动态预算分配 | 根据各层置信度自适应调整分配比例 | ✅ |
| 并行检索 | 中期和长期层使用线程池并行检索 | ✅ |
| 冷知识保护 | FactStore 永久存储 + FactExtractor 自动提取 | ✅ |
| 线程安全 | store 操作加锁保护 | ✅ |
