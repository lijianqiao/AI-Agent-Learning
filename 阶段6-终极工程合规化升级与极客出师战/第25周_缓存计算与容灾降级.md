## 第一部分学习：Prompt Cache 原理——相同前缀复用 KV-Cache

### 为什么 LLM 调用这么贵？

每次调用 LLM，模型都要对输入的每个 Token 做 Self-Attention 计算，生成 Key-Value 对（KV-Cache）。这些 KV 对就是模型"理解"你输入内容的中间产物。Token 越多，计算量越大，费用越高。

但你有没有想过——如果你连续发 100 个请求，每个请求的前 2000 个 Token 都一样（比如都是同一份 System Prompt + 角色设定 + 知识库上下文），那这 2000 个 Token 的 KV-Cache 不是白算了 99 次吗？

**Prompt Cache 就是解决这个问题的。**

### 生活类比：图书馆的"预翻好的参考书"

想象你是图书馆管理员，每天有 100 个学生来查同一本《高等数学》的第 1-50 页，然后各自查不同的习题。

**没有缓存**：每个学生来了都从书架上取书、翻到第 1 页、一页一页翻到第 50 页，再翻到自己要的习题。100 个学生 = 翻了 100 次前 50 页。

**有缓存**：你提前把第 1-50 页复印好放在桌上，学生来了直接拿复印件看，只需要自己翻后面的习题页。翻前 50 页的动作只做了 1 次。

Prompt Cache 就是那份**"预翻好的复印件"**——第一次请求时计算前缀部分的 KV-Cache 并缓存起来，后续请求如果前缀相同，直接复用，不再重复计算。

### KV-Cache 复用的技术原理

```
请求 1: [System Prompt (2000 tokens)] + [用户问题 A (50 tokens)]
         ├─── 计算 KV-Cache ───────┤  ├── 增量计算 ──┤
         │      缓存这部分！        │
         ▼
请求 2: [System Prompt (2000 tokens)] + [用户问题 B (80 tokens)]
         ├─── 命中缓存，跳过！─────┤  ├── 增量计算 ──┤
                                        ↑
                                    只需要计算这 80 个 Token 的 KV
```

**关键规则**：缓存匹配是**严格前缀匹配**——从第一个 Token 开始，连续相同的部分才能复用。中间插一个字符不同，后面全部作废。

### 各厂商 Prompt Cache 方案对比

| 维度 | OpenAI Prompt Caching | Anthropic Claude Cache | Google Gemini Context Caching | 自建 KV-Cache 复用 |
| --- | --- | --- | --- | --- |
| 触发方式 | 自动（前缀 ≥ 1024 tokens） | 手动标记 cache_control | 手动创建 CachedContent | 自建推理引擎配置 |
| 最小缓存长度 | 1024 tokens | 1024 tokens（Sonnet）/ 2048（Haiku） | 32,768 tokens | 自定义 |
| 价格折扣 | 缓存命中部分 50% off | 缓存命中部分 90% off | 缓存存储费 + 使用费，总体约 75% off | 仅服务器成本 |
| 缓存有效期 | 5-10 分钟（低流量时更短） | 5 分钟（每次命中刷新） | 手动设置 TTL，最长 48h | 自定义 |
| 适用场景 | 高频相同 System Prompt | 长文档分析、多轮对话 | 超长上下文反复查询 | 私有化部署 |
| 类比 | 自助餐厅自动保温 | 手动把菜放保温台 | 包下一个保温柜 | 自家厨房冰箱 |

### Anthropic Claude Cache 实战示例

Claude 的缓存机制最有代表性，因为它需要**显式标记**哪些内容要缓存，给了开发者完全的控制权。

```python
import anthropic

client = anthropic.Anthropic()

LONG_SYSTEM_CONTEXT = """
你是一位金融分析师 AI 助手。以下是你需要掌握的产品知识库：

【产品一：稳健增长基金】
... (假设这里有 3000 tokens 的产品说明) ...

【产品二：科技成长基金】
... (又 3000 tokens) ...

【合规要求】
... (又 2000 tokens) ...
"""

def query_with_cache(user_question: str):
    """利用 Prompt Cache 的查询函数"""
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system=[
            {
                "type": "text",
                "text": LONG_SYSTEM_CONTEXT,
                "cache_control": {"type": "ephemeral"}  # 标记缓存
            }
        ],
        messages=[
            {"role": "user", "content": user_question}
        ]
    )

    usage = response.usage
    print(f"输入 tokens: {usage.input_tokens}")
    print(f"缓存写入 tokens: {getattr(usage, 'cache_creation_input_tokens', 0)}")
    print(f"缓存命中 tokens: {getattr(usage, 'cache_read_input_tokens', 0)}")

    return response.content[0].text

# 第一次调用：缓存写入（正常价格）
print("=== 第一次查询（缓存写入）===")
answer1 = query_with_cache("稳健增长基金的年化收益率是多少？")

# 第二次调用：缓存命中（90% 折扣！）
print("\n=== 第二次查询（缓存命中）===")
answer2 = query_with_cache("科技成长基金适合什么风险偏好的客户？")

# 第三次调用：继续命中
print("\n=== 第三次查询（继续命中）===")
answer3 = query_with_cache("合规要求中对推荐话术有什么限制？")
```

### 缓存命中率优化策略

缓存能省多少钱，核心看**命中率**。命中率低 = 缓存形同虚设。

| 优化策略 | 原理 | 预期命中率提升 | 类比 |
| --- | --- | --- | --- |
| System Prompt 前置固定化 | 把不变的长文本放最前面，变化的用户输入放最后 | +30%~50% | 把教材放书桌固定位置，只换习题卷 |
| 多轮对话共享前缀 | 同一会话的历史消息保持不变，新消息追加在末尾 | +20%~40% | 笔记本只往后写，不撕前面的页 |
| 请求批量化 + 时间窗聚合 | 短时间内的请求尽量用相同前缀模板 | +15%~25% | 同一批学生统一发同一张考卷 |
| 知识库版本化 | 知识库内容变更时用版本号标记，避免频繁失效 | +10%~20% | 教材改版才换新复印件，不是每天换 |
| 前缀模板标准化 | 将 System Prompt 拆为固定模板 + 动态插槽 | +25%~35% | 用活页夹，只换中间的活页 |

## 第二部分学习：Redis 语义缓存设计——Embedding 相似度匹配历史问答

### 为什么 Prompt Cache 不够？

Prompt Cache 解决的是**同一前缀**的重复计算问题。但生产环境中还有一种更常见的浪费：**不同用户问的其实是同一个问题，只是措辞不同。**

- 用户 A："Python 怎么读取 JSON 文件？"
- 用户 B："用 Python 读 JSON 数据的方法"
- 用户 C："python json 文件读取"

这三个问题本质相同，但文字完全不同，Prompt Cache 无法命中。你需要的是**语义级别的缓存**。

### 生活类比：餐厅的"常见问题预制菜"

想象你开了一家餐厅：

**没有语义缓存**：每个客人点菜，厨师都从切菜开始现做。即使连续 10 个客人都点了"番茄炒蛋"，厨师也做了 10 次。

**有语义缓存**：厨师发现"番茄炒蛋"点得特别多，于是提前做好 5 份放保温台。客人点了直接上，不用等。而且不管客人说的是"西红柿炒鸡蛋"、"番茄蛋"还是"那个红红黄黄的菜"，服务员都能识别出来——这就是**语义匹配**。

### 语义缓存架构

```
用户请求: "Python 怎么读 JSON？"
        │
        ▼
┌───────────────────┐
│  Embedding 编码   │  ← 将问题转为向量
│  text → [0.12,    │
│   0.87, -0.34...] │
└────────┬──────────┘
         │
         ▼
┌───────────────────────────────────────┐
│           Redis 向量检索              │
│                                       │
│  HNSW 索引存储历史问答:               │
│  ┌──────────────────────────────────┐ │
│  │ key: "cache:q:abc123"           │ │
│  │ embedding: [0.13, 0.86, -0.33]  │ │
│  │ question: "python读取json文件"   │ │
│  │ answer: "使用 json.load()..."    │ │
│  │ created_at: 1709856000          │ │
│  │ hit_count: 47                   │ │
│  └──────────────────────────────────┘ │
│                                       │
│  余弦相似度 = 0.96 > 阈值 0.92       │
│  → 命中！直接返回缓存答案            │
└───────────────────────────────────────┘
         │
    命中？─┬─ 是 → 直接返回缓存答案（0ms LLM 调用）
         │
         └─ 否 → 调用 LLM → 返回答案 → 写入缓存
```

### Prompt Cache vs 语义缓存 vs 传统缓存对比

| 维度 | 传统 Key-Value 缓存 | Prompt Cache（KV-Cache） | 语义缓存（Embedding） |
| --- | --- | --- | --- |
| 匹配方式 | 精确字符串匹配 | 严格前缀 Token 匹配 | 向量余弦相似度匹配 |
| 缓存粒度 | 完整请求 → 完整响应 | 请求前缀 → KV 中间状态 | 问题语义 → 历史答案 |
| 能否处理措辞变化 | ❌ 差一个字就 miss | ❌ 前缀必须完全一致 | ✅ 语义相近即命中 |
| 存储位置 | Redis / Memcached | LLM 推理引擎内存 | Redis + 向量索引 |
| 省钱原理 | 完全跳过 LLM 调用 | 减少重复 Token 计算量 | 完全跳过 LLM 调用 |
| 适用场景 | 完全相同的重复请求 | 相同 System Prompt 的批量请求 | 语义相近的高频问题 |
| 类比 | 自动贩卖机（选什么出什么） | 图书馆的预翻好的参考书 | 餐厅的常见预制菜 |

### 实战代码：Redis 语义缓存系统

```python
import json
import time
import hashlib
import numpy as np
from typing import Optional, Tuple

# ============================================================
# 语义缓存核心实现
# ============================================================

class SemanticCache:
    """基于 Redis 向量检索的语义缓存系统"""

    def __init__(
        self,
        redis_client,
        embedding_fn,
        similarity_threshold: float = 0.92,
        ttl_seconds: int = 3600,
        max_cache_size: int = 10000,
        index_name: str = "sem_cache_idx",
        prefix: str = "sem_cache:"
    ):
        self.redis = redis_client
        self.embed = embedding_fn
        self.threshold = similarity_threshold
        self.ttl = ttl_seconds
        self.max_size = max_cache_size
        self.index_name = index_name
        self.prefix = prefix
        self._ensure_index()

    def _ensure_index(self):
        """确保 Redis 向量索引存在"""
        try:
            self.redis.ft(self.index_name).info()
        except Exception:
            from redis.commands.search.field import (
                VectorField, TextField, NumericField
            )
            from redis.commands.search.indexDefinition import (
                IndexDefinition, IndexType
            )
            schema = (
                VectorField(
                    "embedding",
                    "HNSW",
                    {
                        "TYPE": "FLOAT32",
                        "DIM": 1536,
                        "DISTANCE_METRIC": "COSINE"
                    }
                ),
                TextField("question"),
                TextField("answer"),
                NumericField("created_at"),
                NumericField("hit_count"),
            )
            self.redis.ft(self.index_name).create_index(
                schema,
                definition=IndexDefinition(
                    prefix=[self.prefix],
                    index_type=IndexType.HASH
                )
            )

    def _make_key(self, question: str) -> str:
        h = hashlib.md5(question.encode()).hexdigest()[:12]
        return f"{self.prefix}{h}"

    def lookup(self, question: str) -> Optional[Tuple[str, float]]:
        """语义查找：返回 (缓存答案, 相似度) 或 None"""
        from redis.commands.search.query import Query

        query_vec = self.embed(question)
        q = (
            Query("*=>[KNN 3 @embedding $vec AS score]")
            .sort_by("score")
            .return_fields("question", "answer", "score", "hit_count")
            .dialect(2)
        )
        params = {"vec": np.array(query_vec, dtype=np.float32).tobytes()}
        results = self.redis.ft(self.index_name).search(q, query_params=params)

        if not results.docs:
            return None

        best = results.docs[0]
        similarity = 1 - float(best.score)  # COSINE 距离转相似度

        if similarity >= self.threshold:
            self.redis.hincrby(best.id, "hit_count", 1)
            self.redis.expire(best.id, self.ttl)
            return (best.answer, similarity)

        return None

    def store(self, question: str, answer: str):
        """将问答对写入语义缓存"""
        key = self._make_key(question)
        vec = self.embed(question)
        self.redis.hset(key, mapping={
            "question": question,
            "answer": answer,
            "embedding": np.array(vec, dtype=np.float32).tobytes(),
            "created_at": int(time.time()),
            "hit_count": 0,
        })
        self.redis.expire(key, self.ttl)

    def get_stats(self) -> dict:
        """获取缓存统计信息"""
        info = self.redis.ft(self.index_name).info()
        return {
            "total_entries": int(info["num_docs"]),
            "index_size_mb": round(
                int(info.get("inverted_sz_mb", 0)), 2
            ),
        }


# ============================================================
# 带语义缓存的 LLM 调用封装
# ============================================================

class CachedLLMClient:
    """在 LLM 调用链路前插入语义缓存层"""

    def __init__(self, llm_fn, cache: SemanticCache):
        self.llm = llm_fn
        self.cache = cache
        self.stats = {"total": 0, "cache_hit": 0, "llm_call": 0}

    def query(self, question: str) -> dict:
        self.stats["total"] += 1

        # 第一关：语义缓存拦截
        cached = self.cache.lookup(question)
        if cached:
            answer, similarity = cached
            self.stats["cache_hit"] += 1
            return {
                "answer": answer,
                "source": "semantic_cache",
                "similarity": round(similarity, 4),
                "latency_ms": 2,  # 缓存命中延迟极低
            }

        # 未命中 → 调用 LLM
        start = time.time()
        answer = self.llm(question)
        latency = round((time.time() - start) * 1000, 1)
        self.stats["llm_call"] += 1

        # 写回缓存
        self.cache.store(question, answer)

        return {
            "answer": answer,
            "source": "llm",
            "similarity": None,
            "latency_ms": latency,
        }

    def hit_rate(self) -> float:
        if self.stats["total"] == 0:
            return 0.0
        return self.stats["cache_hit"] / self.stats["total"]

    def report(self) -> str:
        return (
            f"总请求: {self.stats['total']} | "
            f"缓存命中: {self.stats['cache_hit']} | "
            f"LLM 调用: {self.stats['llm_call']} | "
            f"命中率: {self.hit_rate():.1%}"
        )
```

### 缓存命中率优化：从 60% 到 95% 的进阶策略

| 优化手段 | 原理 | 预期命中率提升 | 实现复杂度 |
| --- | --- | --- | --- |
| 问题标准化预处理 | 去除停用词、统一大小写、去标点后再 Embedding | +10%~15% | ⭐ |
| 多粒度缓存 | 同时存"完整问题"和"提取后的核心意图"两层缓存 | +8%~12% | ⭐⭐ |
| 热度加权 TTL | 高频命中的缓存延长过期时间，冷门的快速淘汰 | +5%~8% | ⭐ |
| 答案新鲜度校验 | 定期用 LLM 验证缓存答案是否过时，过时则刷新 | 不直接提升命中率，但保证质量 | ⭐⭐⭐ |
| 分域缓存隔离 | 不同业务域用不同索引，避免跨域误匹配 | +3%~5%（减少误命中） | ⭐⭐ |
| 相似度阈值自适应 | 根据业务容忍度动态调整阈值（严格/宽松模式） | +5%~10% | ⭐⭐ |

## 第三部分学习：自动降级架构——健康探活与故障切换

### 为什么需要自动降级？

生产环境中，LLM API 的故障不是"如果发生"，而是"什么时候发生"：

- OpenAI 全球宕机（2024 年多次发生，每次数小时）
- 云厂商网络链路抖动（跨境调用尤其频繁）
- API 限流 429（高并发场景下常态化）
- 账户余额不足 / API Key 被封

如果你的业务完全依赖单一 LLM API，上面任何一种情况都会导致**业务停摆**。

### 生活类比：医院的应急供电系统

一家三甲医院不会只接一根市电线路：

- **正常状态**：用市电（主力 LLM API，如 GPT-4）
- **市电中断**：10 秒内自动切换到柴油发电机（备用 LLM API，如 Claude）
- **发电机也挂了**：UPS 电池组维持手术室等关键设备 30 分钟（本地部署的小模型 Llama3）
- **全部挂了**：手动手电筒 + 人工操作（静态兜底响应 / 人工接管）

自动降级架构就是给你的 AI 业务搭建同样的**多级供电保障**。

### 降级策略分级

不同故障等级需要不同的降级力度。降级过度会影响用户体验，降级不足会导致服务崩溃。

| 降级等级 | 触发条件 | 降级行为 | 用户感知 | 类比 |
| --- | --- | --- | --- | --- |
| L0 - 无降级 | 所有后端健康 | 正常调用主力 API | 无感知 | 市电正常，灯火通明 |
| L1 - 透明切换 | 主力 API 异常，备用 API 健康 | 自动切到备用 API | 可能略有延迟变化，功能完整 | 市电断了，发电机顶上 |
| L2 - 部分降级 | 主力+备用均异常，本地模型可用 | 切到本地小模型（质量下降） | 回答质量下降，但核心功能可用 | 只剩 UPS，关掉空调只保手术灯 |
| L3 - 静态兜底 | 所有模型不可用 | 返回预设的静态回复 / FAQ 匹配 | 明显感知到非智能回复 | 全停电，护士拿手电筒巡房 |
| L4 - 人工接管 | 静态兜底也无法满足 | 转人工客服 / 排队等待 | 完全人工服务 | 拉闸停业，人工手写病历 |

### 健康探活机制设计

```python
import time
import asyncio
import logging
from enum import Enum
from dataclasses import dataclass, field
from typing import Callable, Optional, Dict, List

logger = logging.getLogger(__name__)


class HealthStatus(Enum):
    HEALTHY = "healthy"
    DEGRADED = "degraded"      # 亚健康：响应变慢但能用
    UNHEALTHY = "unhealthy"    # 不可用


@dataclass
class BackendConfig:
    name: str
    priority: int              # 越小优先级越高（0=主力, 1=备用, 2=本地...）
    call_fn: Callable          # 实际调用函数
    health_check_fn: Callable  # 健康检查函数
    max_consecutive_failures: int = 3
    timeout_seconds: float = 10.0
    recovery_check_interval: float = 15.0  # 故障后多久探活一次


@dataclass
class BackendState:
    status: HealthStatus = HealthStatus.HEALTHY
    consecutive_failures: int = 0
    last_failure_time: float = 0
    last_success_time: float = 0
    total_calls: int = 0
    total_failures: int = 0
    avg_latency_ms: float = 0
    _latency_window: list = field(default_factory=list)

    def record_success(self, latency_ms: float):
        self.consecutive_failures = 0
        self.last_success_time = time.time()
        self.total_calls += 1
        self._latency_window.append(latency_ms)
        if len(self._latency_window) > 20:
            self._latency_window.pop(0)
        self.avg_latency_ms = sum(self._latency_window) / len(self._latency_window)
        self.status = HealthStatus.HEALTHY

    def record_failure(self, max_failures: int):
        self.consecutive_failures += 1
        self.last_failure_time = time.time()
        self.total_calls += 1
        self.total_failures += 1
        if self.consecutive_failures >= max_failures:
            self.status = HealthStatus.UNHEALTHY


class HealthProber:
    """多后端健康探活器"""

    def __init__(self, backends: List[BackendConfig]):
        self.backends = sorted(backends, key=lambda b: b.priority)
        self.states: Dict[str, BackendState] = {
            b.name: BackendState() for b in self.backends
        }

    def get_active_backend(self) -> Optional[BackendConfig]:
        """按优先级返回第一个健康的后端"""
        for backend in self.backends:
            state = self.states[backend.name]
            if state.status != HealthStatus.UNHEALTHY:
                return backend
        return None  # 全部不可用

    def get_degradation_level(self) -> int:
        """获取当前降级等级"""
        active = self.get_active_backend()
        if active is None:
            return 3  # L3 静态兜底
        return active.priority

    async def call_with_failover(self, question: str) -> dict:
        """带自动故障切换的调用"""
        for backend in self.backends:
            state = self.states[backend.name]

            if state.status == HealthStatus.UNHEALTHY:
                elapsed = time.time() - state.last_failure_time
                if elapsed < backend.recovery_check_interval:
                    continue  # 还在冷却期，跳过
                logger.info(f"[探活] 尝试恢复 {backend.name}...")

            try:
                start = time.time()
                result = await asyncio.wait_for(
                    backend.call_fn(question),
                    timeout=backend.timeout_seconds
                )
                latency = (time.time() - start) * 1000
                state.record_success(latency)

                return {
                    "answer": result,
                    "backend": backend.name,
                    "degradation_level": backend.priority,
                    "latency_ms": round(latency, 1),
                }

            except Exception as e:
                state.record_failure(backend.max_consecutive_failures)
                logger.warning(
                    f"[故障] {backend.name} 调用失败 "
                    f"({state.consecutive_failures}/{backend.max_consecutive_failures}): {e}"
                )
                continue

        return {
            "answer": "抱歉，系统正在维护中，请稍后再试。您也可以联系人工客服获取帮助。",
            "backend": "static_fallback",
            "degradation_level": 3,
            "latency_ms": 0,
        }

    def status_report(self) -> str:
        lines = ["========== 后端健康状态 =========="]
        for backend in self.backends:
            s = self.states[backend.name]
            icon = {"healthy": "🟢", "degraded": "🟡", "unhealthy": "🔴"}
            lines.append(
                f"  {icon[s.status.value]} {backend.name} (P{backend.priority}) | "
                f"状态: {s.status.value} | "
                f"连续失败: {s.consecutive_failures} | "
                f"平均延迟: {s.avg_latency_ms:.0f}ms | "
                f"总调用: {s.total_calls} | "
                f"总失败: {s.total_failures}"
            )
        return "\n".join(lines)
```

## 第四部分学习：故障演练方法论——Chaos Engineering

### 什么是混沌工程？

混沌工程（Chaos Engineering）是 Netflix 在 2011 年提出的理念：**在生产环境中主动注入故障，验证系统的韧性**。它的核心哲学是——

> 与其等着系统在半夜三点崩溃，不如自己选个白天把它"搞崩"，看看会发生什么。

### 生活类比：消防演习

你公司每年都会搞消防演习，不是因为"今天会着火"，而是要验证：

1. 烟雾报警器响不响？（**监控告警**是否灵敏）
2. 大家知不知道往哪跑？（**降级链路**是否畅通）
3. 消防通道堵没堵？（**故障切换**是否生效）
4. 从报警到全员撤离要多久？（**恢复时间 RTO** 是否达标）

混沌工程就是 IT 系统的消防演习。

### 故障注入方法论

| 故障类型 | 注入方式 | 验证目标 | 预期系统行为 |
| --- | --- | --- | --- |
| 网络超时 | 设置极短超时 / iptables 丢包 | 超时熔断是否生效 | 超时后自动切到备用后端 |
| API 5xx | Mock 返回 500/502/503 | 错误重试+降级是否生效 | 重试 N 次后切到下一级 |
| API 限流 429 | Mock 返回 429 + Retry-After | 退避重试是否合理 | 指数退避等待后重试 |
| 完全断网 | 防火墙阻断所有外部连接 | 本地模型兜底是否可用 | 自动切到本地 Llama3 |
| 延迟注入 | 人为增加 2-5 秒延迟 | 亚健康感知是否灵敏 | 标记为 DEGRADED，降低权重 |
| 缓存失效 | 清空 Redis 缓存 | 缓存击穿防护是否有效 | 互斥锁防止雪崩，逐步重建 |
| 多重故障 | 主力+备用同时挂掉 | 极端场景兜底能力 | 切到本地模型或静态回复 |

### 实战代码：故障注入框架

```python
import random
import asyncio
import time
import logging
from dataclasses import dataclass
from typing import Callable, Optional
from enum import Enum

logger = logging.getLogger(__name__)


class FaultType(Enum):
    NONE = "none"
    TIMEOUT = "timeout"
    ERROR_500 = "error_500"
    ERROR_429 = "error_429"
    SLOW = "slow_response"
    NETWORK_DOWN = "network_down"


@dataclass
class FaultConfig:
    """故障注入配置"""
    timeout_prob: float = 0.0        # 超时概率
    error_500_prob: float = 0.0      # 500 错误概率
    error_429_prob: float = 0.0      # 429 限流概率
    slow_prob: float = 0.0           # 慢响应概率
    slow_delay_range: tuple = (2.0, 5.0)
    network_down: bool = False       # 是否模拟完全断网


class ChaosMonkey:
    """混沌猴子：在 LLM 调用链路中注入可控故障"""

    def __init__(self, config: FaultConfig):
        self.config = config
        self.injection_log: list = []

    def _decide_fault(self) -> FaultType:
        if self.config.network_down:
            return FaultType.NETWORK_DOWN
        r = random.random()
        cumulative = 0
        for prob, fault in [
            (self.config.timeout_prob, FaultType.TIMEOUT),
            (self.config.error_500_prob, FaultType.ERROR_500),
            (self.config.error_429_prob, FaultType.ERROR_429),
            (self.config.slow_prob, FaultType.SLOW),
        ]:
            cumulative += prob
            if r < cumulative:
                return fault
        return FaultType.NONE

    async def wrap_call(
        self, call_fn: Callable, question: str
    ) -> str:
        """包装原始调用，注入故障"""
        fault = self._decide_fault()
        self.injection_log.append({
            "time": time.time(),
            "fault": fault.value,
            "question": question[:50],
        })

        if fault == FaultType.TIMEOUT:
            logger.warning("[ChaosMonkey] 注入超时故障")
            await asyncio.sleep(30)  # 超过任何合理的 timeout
            raise TimeoutError("ChaosMonkey: simulated timeout")

        if fault == FaultType.ERROR_500:
            logger.warning("[ChaosMonkey] 注入 500 错误")
            raise Exception("ChaosMonkey: HTTP 500 Internal Server Error")

        if fault == FaultType.ERROR_429:
            logger.warning("[ChaosMonkey] 注入 429 限流")
            raise Exception("ChaosMonkey: HTTP 429 Too Many Requests")

        if fault == FaultType.SLOW:
            delay = random.uniform(*self.config.slow_delay_range)
            logger.warning(f"[ChaosMonkey] 注入慢响应 {delay:.1f}s")
            await asyncio.sleep(delay)

        if fault == FaultType.NETWORK_DOWN:
            logger.warning("[ChaosMonkey] 注入完全断网")
            raise ConnectionError("ChaosMonkey: network unreachable")

        return await call_fn(question)

    def summary(self) -> dict:
        from collections import Counter
        faults = Counter(log["fault"] for log in self.injection_log)
        return {
            "total_calls": len(self.injection_log),
            "fault_distribution": dict(faults),
        }
```

```python
import time
import asyncio
import logging
from typing import Optional, Dict, Any
from dataclasses import dataclass, field
from enum import Enum

logger = logging.getLogger(__name__)
logging.basicConfig(level=logging.INFO)


# ============================================================
# 静态兜底响应库
# ============================================================

STATIC_FALLBACK_RESPONSES = {
    "greeting": "您好！系统正在维护中，我可以为您提供基础帮助。如需专业服务，请联系人工客服。",
    "unknown": "抱歉，当前 AI 服务暂时不可用。您的问题已记录，恢复后将优先处理。",
    "faq_refund": "退款通常在 3-7 个工作日内到账。如有特殊情况，请拨打客服热线 400-xxx-xxxx。",

}


def match_static_response(question: str) -> str:
    """简单的关键词匹配静态回复"""
    q = question.lower()
    if any(w in q for w in ["退款", "退钱", "退货"]):
        return STATIC_FALLBACK_RESPONSES["faq_refund"]
    if any(w in q for w in ["物流", "配送", "快递", "发货"]):
        return STATIC_FALLBACK_RESPONSES["faq_delivery"]
    if any(w in q for w in ["你好", "hi", "hello"]):
        return STATIC_FALLBACK_RESPONSES["greeting"]
    return STATIC_FALLBACK_RESPONSES["unknown"]


# ============================================================
# 模拟各级 LLM 后端
# ============================================================

async def mock_gpt4o(question: str) -> str:
    """模拟 GPT-4o 调用（主力）"""
    await asyncio.sleep(0.8)
    return f"[GPT-4o 高质量回答] 关于'{question[:20]}'的详细分析..."


async def mock_claude(question: str) -> str:
    """模拟 Claude 调用（备用）"""
    await asyncio.sleep(0.6)
    return f"[Claude 备用回答] 关于'{question[:20]}'的回答..."


async def mock_llama3_local(question: str) -> str:

    await asyncio.sleep(0.3)
    return f"[Llama3-本地] 关于'{question[:20]}'的基础回答..."


# ============================================================
# 简化版语义缓存（不依赖 Redis，用于演示）
# ============================================================

class InMemorySemanticCache:
    """内存版语义缓存（生产环境请替换为 Redis 版）"""

    def __init__(self, threshold: float = 0.85):
        self.store: Dict[str, Dict] = {}
        self.threshold = threshold
        self.hits = 0
        self.misses = 0

    @staticmethod
    def _simple_similarity(a: str, b: str) -> float:
        """简化版相似度（生产环境请替换为 Embedding 余弦相似度）"""
        set_a = set(a.lower().split())
        set_b = set(b.lower().split())
        if not set_a or not set_b:
            return 0.0
        intersection = set_a & set_b
        union = set_a | set_b
        return len(intersection) / len(union)

    def lookup(self, question: str) -> Optional[str]:
        best_score = 0
        best_answer = None
        for key, entry in self.store.items():
            score = self._simple_similarity(question, entry["question"])
            if score > best_score:
                best_score = score
                best_answer = entry["answer"]
        if best_score >= self.threshold:
            self.hits += 1
            return best_answer
        self.misses += 1
        return None

    def store_entry(self, question: str, answer: str):
        import hashlib
        key = hashlib.md5(question.encode()).hexdigest()[:12]
        self.store[key] = {
            "question": question,
            "answer": answer,
            "time": time.time(),
        }

    def hit_rate(self) -> float:
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0


# ============================================================
# 弹性 LLM 服务：缓存 + 探活 + 多级降级
# ============================================================

class DegradationLevel(Enum):
    L0_NORMAL = 0
    L1_BACKUP_API = 1
    L2_LOCAL_MODEL = 2
    L3_STATIC_FALLBACK = 3


@dataclass
class BackendNode:
    name: str
    level: DegradationLevel
    call_fn: Any
    timeout: float = 10.0
    max_failures: int = 3
    consecutive_failures: int = 0
    is_healthy: bool = True
    last_failure_time: float = 0
    recovery_interval: float = 15.0

    def mark_failure(self):
        self.consecutive_failures += 1
        self.last_failure_time = time.time()
        if self.consecutive_failures >= self.max_failures:
            self.is_healthy = False
            logger.warning(
                f"[熔断] {self.name} 连续失败 {self.consecutive_failures} 次，标记为不可用"
            )

    def mark_success(self):
        if not self.is_healthy:
            logger.info(f"[恢复] {self.name} 探活成功，恢复为可用状态")
        self.consecutive_failures = 0
        self.is_healthy = True

    def should_try_recovery(self) -> bool:
        if self.is_healthy:
            return True
        return (time.time() - self.last_failure_time) > self.recovery_interval


class ResilientLLMService:
    """生产级弹性 LLM 服务"""

    def __init__(self):
        self.cache = InMemorySemanticCache(threshold=0.85)
        self.backends = [
            BackendNode("GPT-4o", DegradationLevel.L0_NORMAL, mock_gpt4o, timeout=10),
            BackendNode("Claude", DegradationLevel.L1_BACKUP_API, mock_claude, timeout=8),
            BackendNode("Llama3-Local", DegradationLevel.L2_LOCAL_MODEL, mock_llama3_local, timeout=5),
        ]
        self.request_log: list = []

    async def query(self, question: str) -> Dict[str, Any]:
        start = time.time()

        # 第一关：语义缓存拦截
        cached = self.cache.lookup(question)
        if cached:
            result = {
                "answer": cached,
                "source": "semantic_cache",
                "degradation_level": "cache_hit",
                "latency_ms": round((time.time() - start) * 1000, 1),
            }
            self.request_log.append(result)
            return result

        # 第二关：多级后端调用（带故障切换）
        for backend in self.backends:
            if not backend.should_try_recovery():
                continue

            try:
                answer = await asyncio.wait_for(
                    backend.call_fn(question),
                    timeout=backend.timeout
                )
                backend.mark_success()

                # 写回缓存
                self.cache.store_entry(question, answer)

                result = {
                    "answer": answer,
                    "source": backend.name,
                    "degradation_level": backend.level.name,
                    "latency_ms": round((time.time() - start) * 1000, 1),
                }
                self.request_log.append(result)
                return result

            except (asyncio.TimeoutError, Exception) as e:
                backend.mark_failure()
                logger.warning(f"[降级] {backend.name} 失败: {e}")
                continue

        # 第三关：静态兜底
        answer = match_static_response(question)
        result = {
            "answer": answer,
            "source": "static_fallback",
            "degradation_level": DegradationLevel.L3_STATIC_FALLBACK.name,
            "latency_ms": round((time.time() - start) * 1000, 1),
        }
        self.request_log.append(result)
        return result

    def get_system_status(self) -> str:
        lines = [
            "=" * 60,
            "         系统健康状态看板",
            "=" * 60,
        ]
        for b in self.backends:
            icon = "🟢" if b.is_healthy else "🔴"
            lines.append(
                f"  {icon} {b.name:20s} | "
                f"健康: {'是' if b.is_healthy else '否':3s} | "
                f"连续失败: {b.consecutive_failures}"
            )
        lines.append(f"\n  缓存命中率: {self.cache.hit_rate():.1%}")
        lines.append(
            f"  总请求: {len(self.request_log)} | "
            f"缓存命中: {self.cache.hits} | "
            f"缓存未命中: {self.cache.misses}"
        )
        return "\n".join(lines)

# ============================================================

    print("\n📌 阶段三：注入故障 —— GPT-4o 连续超时")
    gpt4o = service.backends[0]
    original_fn = gpt4o.call_fn

    async def broken_gpt4o(q):
        raise TimeoutError("Connection timed out")

    gpt4o.call_fn = broken_gpt4o

    fault_start = time.time()
    for i in range(4):
        result = await service.query(f"第{i+1}个故障期间的新问题")
        print(f"  请求 {i+1}: [{result['source']}] 降级等级={result['degradation_level']}")

    failover_time = time.time() - fault_start
    print(f"  ⏱ 故障切换总耗时: {failover_time:.1f}s")

    # 阶段四：双重故障
    print("\n📌 阶段四：双重故障 —— GPT-4o + Claude 同时宕机")
    claude = service.backends[1]
    original_claude_fn = claude.call_fn

    async def broken_claude(q):
        raise ConnectionError("502 Bad Gateway")

    claude.call_fn = broken_claude
    claude.consecutive_failures = 0
    claude.is_healthy = True

    for i in range(4):
        result = await service.query(f"双重故障期间的问题{i+1}")
        print(f"  请求 {i+1}: [{result['source']}] 降级等级={result['degradation_level']}")

    # 阶段五：三重故障（全挂）
    print("\n📌 阶段五：极端场景 —— 所有模型不可用")
    llama = service.backends[2]
    original_llama_fn = llama.call_fn

    async def broken_llama(q):
        raise RuntimeError("GPU OOM")

    llama.call_fn = broken_llama
    llama.consecutive_failures = 0
    llama.is_healthy = True

    for i in range(2):
        result = await service.query("退款要多久到账？")
        print(f"  请求 {i+1}: [{result['source']}] {result['answer'][:60]}")

    # 阶段六：恢复
    print("\n📌 阶段六：主力 API 恢复")
    gpt4o.call_fn = original_fn
    gpt4o.is_healthy = False
    gpt4o.last_failure_time = time.time() - 20  # 模拟已过冷却期

    result = await service.query("恢复后的测试问题")
    print(f"  恢复结果: [{result['source']}] 降级等级={result['degradation_level']}")

    # 最终报告
    claude.call_fn = original_claude_fn
    llama.call_fn = original_llama_fn
    print("\n" + service.get_system_status())

    # 演练总结
    print("\n" + "=" * 60)
    print("         故障演练总结报告")
    print("=" * 60)
    sources = {}
    for log in service.request_log:
        src = log["source"]
        sources[src] = sources.get(src, 0) + 1
    print(f"\n  总请求数: {len(service.request_log)}")
    for src, cnt in sources.items():
        print(f"    {src}: {cnt} 次")
    print(f"\n  缓存命中率: {service.cache.hit_rate():.1%}")
    print(f"  系统崩溃次数: 0（所有请求均有响应）✅")
    print(f"  故障切换耗时: {failover_time:.1f}s（目标 < 30s）✅")


if __name__ == "__main__":
    asyncio.run(chaos_drill())
```
